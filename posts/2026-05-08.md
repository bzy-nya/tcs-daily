## 编辑按语

今天的 5 篇论文覆盖了 graph minor separator、attention coreset、dense random hypergraph 中的 online 算法阈值、QBF 的参数化宽度、以及 decremental forest 数据结构几个不同方向。它们没有一个共同主题；比较自然的读法是把它们当作五条各自独立的进展：一篇推进 $K_h$-minor-free 图 separator 的经典猜想线，一篇把 softmax attention 压缩问题放进 discrepancy/coreset 框架，一篇完整刻画 dense hypergraph online independent set 的相变。

另外两篇分别关注结构参数与动态数据结构。QBF 论文提出 bilateral treewidth，把 strategy expansion 与 Q-resolution 两种机制合进同一个 FPT 框架；forest 论文则说明 deletion-only 限制什么时候能把树上求和降到 $O(\log^* n)$ 甚至线性，什么时候又会重新撞上 partial sums 下界。

## A Separator for Minor-Free Graphs Beyond the Flow Barrier [arXiv:2605.05494](https://arxiv.org/abs/2605.05494)

这篇论文研究的是 $K_h$-minor-free 图的 balanced separator：给定一张 $n$ 点图 $G$，并且知道它不包含 $K_h$ 作为 minor，目标是删掉尽量少的顶点，使剩下每个连通分量都至多有 $2n/3$ 个顶点。作者证明任意 $K_h$-minor-free 图都有大小
$$
O\!\left(h\sqrt{\log h}\sqrt n\right)
$$
的 balanced separator，并给出 randomized polynomial-time 构造算法。这把最近 flow-duality 路线能达到的 $O(h\log h\sqrt n)$ 再降了一个 $\sqrt{\log h}$ 因子，朝 Alon-Seymour-Thomas 猜想的最优 $O(h\sqrt n)$ 迈了一步。

这里的对象先要拆开看。一个 balanced separator 是图算法里最常用的“递归入口”：如果每次能删掉小集合，把问题分成两个都明显变小的部分，很多动态规划、近似算法、子指数算法都会随之出现。平面图的 Lipton-Tarjan separator theorem 是最经典例子：任意 $n$ 点平面图都有 $O(\sqrt n)$ 大小的 separator。Minor-free 图是平面图的自然推广。所谓 $H$ 是 $G$ 的 minor，意思是可以在 $G$ 里删点、删边、收缩边得到 $H$；等价地，可以在 $G$ 中找到若干两两点不交的连通子图，把它们各自压成一个点以后形成 $H$。于是 $K_h$-minor-free 图就是“不含大小为 $h$ 的完全图 minor”的图类。它包含平面图、固定 genus 图、以及许多稀疏图类，是 graph minor theory 与算法设计交汇处的基本模型。

:::aside[Minor model]
论文里用的 minor model 是一组两两点不交的连通子图，称为 branch sets。若任意两个 branch sets 之间至少有一条边，则它们构成一个 clique minor model；branch set 的个数就是 clique minor 的大小。$G$ 是 $K_h$-minor-free，当且仅当不存在大小为 $h$ 的 clique minor model。
:::

为什么大家关心 $h$ 和 $n$ 的精确依赖？因为 separator theorem 在算法里通常会被递归调用，$h$ 上多一个 $\log h$ 或 $\sqrt h$ 不是装饰常数，而会直接变成 treewidth bound、分治复杂度或近似比里的参数损失。Alon, Seymour, and Thomas (1990) 首先证明 $K_h$-minor-free 图有
$$
O\!\left(h^{3/2}\sqrt n\right)
$$
大小的 balanced separator，并且已经指出他们相信正确答案应为 $O(h\sqrt n)$。这个猜想是 tight 的：常度 expander 在 $h=\Theta(\sqrt n)$ 的尺度下可以没有 $K_h$ minor，同时任意 balanced separator 都是线性大小，因此 $O(h\sqrt n)$ 不可能再整体改进。

随后 Plotkin, Rao, and Smith (1994) 给出 $O(h\sqrt{n\log n})$ 型 bound：它在 $h$ 上是线性的，但对固定 $h$ 会比 AST 的 $O(\sqrt n)$ 多一个 $\sqrt{\log n}$。Kawarabayashi and Reed (2010) 宣称过更接近猜想的结果，后来相关表述被澄清为 $O(h\sqrt n+f(h))$，其中 $f(h)$ 来自 graph minor structure theorem，增长极快；即便结合 Gorsky-Seweryn-Wiederrecht (2025) 对 structure theorem 的 polynomial bound [arXiv:2504.02532](https://arxiv.org/abs/2504.02532)，这个多项式依赖也远不适合当作一个 elementary separator theorem。另一条算法线来自 flow-cut duality：Biswal, Lee, and Rao (2010) 用 concurrent flow 和 sparsest cut 的对偶联系得到 separator；Spalding-Jamieson (2025) 结合 Conroy-Filtser 的 optimal padded decomposition，把 bound 推到 $O(h\log h\log\log h\sqrt n)$ [arXiv:2506.01228](https://arxiv.org/abs/2506.01228)。本文先给出一个更直接的 flow-based 证明，把这一项改成
$$
O\!\left(h\log h\sqrt n\right),
$$
然后指出这已经碰到了 flow 方法的自然边界。

这个“flow barrier”并不是一句泛泛的经验判断。Concurrent flow 可以理解为：对每一对顶点都送一单位流，允许流在所有路径之间分配，关心的是某个顶点被多少总流量穿过。Leighton-Rao 型 max-flow/min-cut 对偶告诉我们，若图没有小 sparse cut，就能找到低 congestion 的 concurrent flow；反过来，若 flow 拥塞太小，可以随机抽路径拼出一个 forbidden minor，从而和 $K_h$-minor-free 矛盾。问题在于 sparse cut 与 concurrent flow 之间存在 $\Theta(\log h)$ 的 gap：小的常度 expander 已经能在排除 $K_{\Theta(h)}$ minor 的同时实现 $\Omega(\log h)$ 的 flow-cut gap。换句话说，只靠这套对偶框架，$O(h\log h\sqrt n)$ 基本就是终点。

本文的主定理可以写成：
$$
\textbf{Theorem.}\quad
\text{对任意 }h\ge 3\text{ 和任意 }n\text{ 点 }K_h\text{-minor-free 图 }G,
\text{存在大小 }O(h\sqrt{\log h}\sqrt n)\text{ 的 balanced separator，}
$$
并且可以在 randomized polynomial time 内找到。

和前面的 bound 放在一起，量级差异是：

| 结果 | separator 大小 | 主要方法 |
| :-- | :-- | :-- |
| Alon-Seymour-Thomas (1990) | $O(h^{3/2}\sqrt n)$ | minor-growing template |
| Plotkin-Rao-Smith (1994) | $O(h\sqrt{n\log n})$ | shallow minor / layering |
| Spalding-Jamieson (2025) | $O(h\log h\log\log h\sqrt n)$ | flow-cut duality + padded decomposition |
| 本文 flow 版本 | $O(h\log h\sqrt n)$ | flow-cut duality，达到 barrier |
| 本文主结果 | $O(h\sqrt{\log h}\sqrt n)$ | AST template + low-diameter decomposition |
| AST 猜想 | $O(h\sqrt n)$ | open |

我觉得这篇最值得细看的地方，是它没有完全抛弃 AST 框架。AST 框架大致维护三样东西：一个当前已经构造出的 clique minor model $\mathcal K$，一个已经承诺放进 separator 的集合 $X$，以及删掉当前 separator 后最大的连通分量 $H$。只要 $H$ 还超过 $2n/3$，算法就继续缩小 $H$：有时把一个新的 branch set 加进 $\mathcal K$，有时扩张已有 branch set，有时把一层顶点加入 $X$。如果 $\mathcal K$ 的大小达到 $h$，就找到了 $K_h$ minor；在 $K_h$-minor-free 图里这不会发生，所以过程必须以 separator 结束。

AST 分析的一个关键权衡是这样的。取参数 $\ell$。每次若能新建 branch set，其大小大约是 $O(h\ell)$；因为最多有 $h$ 个 branch sets，最终贡献 $O(h^2\ell)$。另一种情形里，算法只能保证某个 branch set 在当前大分量 $H$ 中的邻居数变成 $O(n/\ell)$；separator 最后对每个 branch set 删除 $V(C)$ 或 $N_H(C)$ 中较小者，于是邻居项总贡献 $O(hn/\ell)$. 平衡两项得到
$$
O\!\left(\frac{hn}{\ell}+h^2\ell\right)
  = O(h^{3/2}\sqrt n).
$$
Kotlov (2000) 后来证明，这个 $O(n/\ell)$ 的 neighborhood bound 在 AST 原始设置里是 tight 的。也就是说，如果不改变进入这个 bound 的局部几何，单纯在 AST 的旧 Case I / Case II 里挤常数不会推出猜想。

本文的新入口是把 low-diameter decomposition 插进每轮迭代。Low-diameter decomposition 的直觉很简单：删掉少量顶点后，剩下每个连通分量在原图距离里都很小。这里用到 Conroy and Filtser (2025) 对 $K_h$-minor-free 图的 padded decomposition：它的 padding parameter 是 $O(\log h)$，由此推出一个 LDD 引理：
$$
\text{给定 }\Delta>0,\quad
\exists S\subseteq V,\quad |S|=O\!\left(\frac{n\log h}{\Delta}\right),
$$
使得 $G-S$ 的每个连通分量 weak diameter 至多 $\Delta$。

:::aside[Padded decomposition 到 LDD]
Padded decomposition 随机把图分成直径至多 $\Delta$ 的块，并保证每个小球 $B(x,\gamma\Delta)$ 以至少 $e^{-\beta\gamma}$ 的概率完整落在 $x$ 所在的块里。取 $\gamma=1/\Delta$，把那些半径 $1$ 小球没有被保住的顶点删掉，期望删点数是 $O(\beta n/\Delta)=O(n\log h/\Delta)$；剩余每个连通分量必须落在同一个 partition block 中，因此 weak diameter 至多 $\Delta$。
:::

算法令 $\Delta=\ell\log h$。如果 LDD 删点集 $S$ 已经把 $H$ 切成所有分量不超过 $2n/3$，那直接输出 separator；$|S|=O(n/\ell)$，在最终预算内。否则，$H-S$ 里有一个大分量，它的 weak diameter 至多 $\Delta$，于是存在一个顶点 $v$，使 $H$ 中至少 $2n/3$ 个顶点都在 $v$ 的距离 $\Delta$ 内。接下来围绕 $v$ 做 BFS layering：前 $\Delta$ 层叫 $U$，后面约 $(\log h+1)\ell$ 层叫 $M$，再往外叫 $B$；同时把 $B$ 顶部的 $\ell$ 层并进 $M^{+\ell}$ 作为缓冲区。

这个分层把 AST 的两难局面拆成了三个更细的情形。若每个 branch set 都在 $U\cup M^{+\ell}$ 里有邻居，就可以用从 $v$ 出发的短路径把这些邻居连起来，形成一个新的 branch set。路径长度现在是 $O(\ell\log h)$，所以新 branch set 大小是 $O(h\ell\log h)$。这比 AST 的 $O(h\ell)$ 多了一个 $\log h$，看起来亏了，但后面会换回更大的收益。

若有某个 branch set $C$ 的所有邻居都躲在更远的 $B\setminus M^{+\ell}$，而且 $|B|\le n/h$，那么 $B$ 顶部的 $\ell$ 层里必有一层 $Y$ 大小至多 $n/(h\ell)$. 算法把 $C$ 向外扩张，吃掉从 $N_H(C)$ 能在更深层区域到达的所有顶点。扩张之后，$C$ 在新的 $H$ 里的邻居只能落在那一层 $Y$，于是
$$
|N_H(C)|\le \frac{n}{h\ell}.
$$
这就是全文技术收益的核心：AST 原来只能把邻居数压到 $O(n/\ell)$，这里多省了一个 $h$。省下的 $h$ 来自“若远端区域 $B$ 总共只有 $n/h$ 大，那么在 $\ell$ 层里找薄层”的平均论证；而能把邻居全部逼进远端区域，则依赖前面 LDD 产生的中心球 $U$ 与中间缓冲层 $M$。

剩下的情况是 $|B|>n/h$。这时不能用薄层把某个 branch set 的邻居压小，因为远端区域太大。算法转而在中间层 $M$ 里找一层 $L_{i^*}$，满足
$$
|L_{i^*}|\le \frac{|L_{\ge i^*+1}|}{\ell}.
$$
这样的层存在是一个增长反证：如果每层都比后缀的 $1/\ell$ 大，那么经过约 $\ell\log h$ 层，后缀大小会从 $>n/h$ 放大到超过 $n$。找到后，把 $L_{i^*}$ 加入 $X$，并丢弃其外侧所有层。由于 LDD 已经保证至少 $2n/3$ 个顶点在前 $\Delta$ 层内，外侧被丢弃的部分不会成为新的最大分量；并且每个被丢弃顶点只会被这样 charge 一次，所以整个过程中 $|X|\le n/\ell$。

把三部分合起来，论文证明了参数化引理：
$$
\textbf{Lemma.}\quad
\text{任意 }K_h\text{-minor-free }n\text{ 点图都有大小 }
O\!\left(\frac n\ell+h^2\ell\log h\right)
\text{ 的 balanced separator。}
$$
这里 $n/\ell$ 来自 LDD 删点与中间层 charging，$h^2\ell\log h$ 来自最多 $h$ 个 branch sets、每个新建时大小 $O(h\ell\log h)$。取
$$
\ell=\frac{\sqrt n}{h\sqrt{\log h}}
$$
就得到
$$
O\!\left(h\sqrt{\log h}\sqrt n\right).
$$

这一步也解释了为什么本文不是简单把某个旧 bound 中的 $\log h$ 改成 $\sqrt{\log h}$。Flow-based 方法的 $\log h$ 来自 metric decomposition / flow-cut gap，本质上是全局对偶损失；本文的 $\sqrt{\log h}$ 来自一个完全不同的 trade-off：为了把 neighborhood bound 从 $n/\ell$ 降到 $n/(h\ell)$，作者愿意让新 branch set 的半径从 $\ell$ 增到 $\ell\log h$。两项平衡后，$\log h$ 只剩平方根。这也是我读这篇时最意外的一点：Kotlov 的 tightness 没有被“反驳”，而是被绕开了。原来的 neighborhood bound 在原来的局部模板里仍然 tight；本文通过 LDD 先制造一个大中心球，再把远端区域切成“足够小可薄层”与“足够大可 charge”两种状态，从而改变了 neighborhood bound 所处的几何条件。

论文最后还给了一个 flow-based $O(h\log h\sqrt n)$ 证明，作用不只是补一条结果线，而是把 barrier 说清楚。借助 Feige-Hajiaghayi-Lee 的 separator/embedding 框架和 Rabinovich 关于 padded decomposition 到 line embedding 的联系，$K_h$-minor-free 图的 $O(\log h)$ padding 给出一个二选一结论：要么找到大小 $O(n^2\log h/\gamma)$ 的 separator，要么在一个至少 $2n/3$ 大的 induced subgraph 上找到 congestion $\gamma$ 的 concurrent flow。取 $\gamma=\Theta(n^{3/2}/h)$，若得到 separator 就是 $O(h\log h\sqrt n)$；若得到 flow，则随机选择路径并用 Lovász Local Lemma 控制路径碰撞，可以构造出一个 almost minor embedding，进而推出 forbidden minor，矛盾。这里 $\log h$ 正是 padding / flow-cut gap，不能靠同一套对偶证明消掉。

这篇论文没有解决 AST 猜想，也没有声称运行时间最优；作者明确说重点是 separator 的存在性与大小。一个有意思的旁枝是，主算法每轮调用 LDD，但分析真正需要的是找到一个半径 $O(\ell\log h)$ 的大球。论文指出可以用随机采样加 BFS 替代除一次以外的大多数 LDD 调用，把实现瓶颈集中到 padded decomposition 的构造上。对我来说，这让结果显得更像一条可继续推进的组合路线，而不是某个沉重 structure theorem 的黑盒后果：如果将来要把 $\sqrt{\log h}$ 再拿掉，恐怕也要在“如何同时保持短 branch set 与 $1/h$ 级邻居压缩”这处权衡上动刀，而不是回到 flow-cut duality 里寻找不存在的免费午餐。

## Nearly Optimal Attention Coresets [arXiv:2605.05602](https://arxiv.org/abs/2605.05602)

这篇论文研究的是 softmax attention 的 coreset 压缩问题：给定一组 key-value cache $(K,V)=\{(k_i,v_i)\}_{i=1}^n$，其中 $k_i,v_i\in\mathbb{R}^d$ 且范数至多 $1$，目标是只保留原 cache 的一个小子集 $(K',V')$，让所有范数不超过 $\rho$ 的 query $q$ 都看到几乎相同的 attention 输出。Liberty、Andoni 和 Kleiner 证明存在大小
$$
O\!\left(\frac{\sqrt d\, e^{\rho+o(\rho)}}{\varepsilon}\right)
$$
的 $\varepsilon$-coreset，同时证明任意 coreset 都需要
$$
\Omega\!\left(\frac{\sqrt d\, e^\rho}{\varepsilon}\right)
$$
个点，因而除了一个 $\rho^{1/2+o(1)}$ 因子外已经最优。

这里的 attention 就是 Transformer 里每一步生成时读 KV-cache 的操作。给定 query $q$，每个 key $k_i$ 得到权重 $\exp(q^\top k_i)$，再用这些权重对 value $v_i$ 做归一化平均：
$$
\mathrm{Attn}(q,K,V)
=
\frac{A(q,K,V)}{B(q,K)},\qquad
A(q,K,V)=\sum_i e^{q^\top k_i}v_i,\quad
B(q,K)=\sum_i e^{q^\top k_i}.
$$
论文为了简化符号去掉了工程实现中常见的 $1/\sqrt d$ temperature；这不改变数学问题，只是把尺度吸收到 $\rho$ 里。Coreset 的要求是
$$
\left\|\mathrm{Attn}(q,K,V)-\mathrm{Attn}(q,K',V')\right\|_2\le \varepsilon
\quad\text{for all }\|q\|\le \rho.
$$
注意这里不是平均 query、也不是某个数据分布下的误差，而是对整个半径为 $\rho$ 的球做 uniform guarantee。

为什么必须限制 $\|q\|$？因为 attention 本身带有近似检索能力。如果允许 query 范数任意大，取 $q$ 朝着某个 key 的方向放大，softmax 权重会集中到这个 key 上，输出几乎就是对应的 $v_i$。这意味着一个支持所有 query 的压缩结构基本要能回答“第 $i$ 个 value 是什么”，也就谈不上真正丢弃大量 cache。本文的 $\rho$ 正是在量化这件事：query 越大，softmax 越像硬检索；query 被限制得越小，多个 token 的贡献越混合，压缩才有空间。

这个问题的应用背景很直接。长上下文 LLM 推理时，KV-cache 是主要内存瓶颈之一：每生成一个 token，都要保留过去 token 在每层每个 head 上的 key 和 value。工程上已经有很多压缩路线：量化每个向量、用 centroid 或 distilled token 替代原 cache、把 attention 换成 linear attention 或 state-space 模型、以及根据启发式分数丢 token。本文选择的是最 TCS 的版本：不改变模型，不学习新 token，只从原始 $(k_i,v_i)$ 中选一个子集，并给出最坏情形误差保证。

Coreset 本身是一类很老的算法思想：面对一个大数据集，我们希望保留少量代表点，使得某一整族查询函数的值都近似不变。典型例子包括 clustering、kernel density estimation、几何范围查询、矩阵 sketching。这里的“整族查询”就是所有 $\|q\|\le \rho$ 的 attention map。与采样不同，现代 coreset 构造常常依赖 discrepancy theory：给每个点染上 $+1/-1$，让一大类测试函数下的正负贡献彼此抵消，然后把其中一半点留下。这个 halving 操作不断迭代，就把 $n$ 个点压到目标规模。

这条脉络中最接近本文的是 Karnin 和 Liberty (2019) 的“discrepancy, coresets, and sketches”框架。他们证明，对于很多形如 $\sum_i f(q^\top k_i)$ 或 $\sum_i f(\|q-k_i\|)$ 的标量 analytic kernel sum，可以用 Banaszczyk (1998) 的 vector balancing theorem 得到近最优 coreset。更近期，Kochetkova et al. (2025) 把类似思想用于 streaming attention approximation，通过 exponential kernel 的 feature map 线性化和在线 discrepancy minimization 得到大小大致
$$
O\!\left(\frac{\sqrt d\,\mathrm{polylog}(n)\,e^{2\rho}}{\varepsilon}\right)
$$
的保证。本文把这个界改成
$$
O\!\left(\frac{\sqrt d\,e^{\rho+\frac12\log\rho+o(\log\rho)}}{\varepsilon}\right),
$$
去掉了 $\mathrm{polylog}(n)$，也把指数依赖从 $e^{2\rho}$ 降到接近 $e^\rho$。这不是常数优化：当 $\rho$ 是控制 attention 是否接近检索的核心参数时，指数上一倍的差距就是压缩是否可用的差距。

:::aside[Banaszczyk balancing]
Banaszczyk (1998) 的向量平衡定理可以粗略理解为：若 $\|x_i\|\le 1$，且对称凸体 $K$ 的 Gaussian measure 至少 $1/2$，则存在符号 $\sigma_i\in\{\pm1\}$ 使得 $\sum_i\sigma_i x_i\in 5K$。Coreset 证明里，$x_i$ 编码一个数据点对所有测试函数的贡献，$K$ 则编码“所有测试误差同时小”的条件。
:::

技术上，论文先不直接处理归一化后的 $\mathrm{Attn}=A/B$，而是分别控制 numerator $A(q,K,V)$ 和 denominator $B(q,K)$。核心 halving theorem 是：
$$
\textbf{Theorem 3.}\quad
\exists (K',V')\subseteq (K,V),\ |K'|\le |K|/2,\ \text{such that for all }\|q\|\le\rho,
$$
$$
\left\|A(q,K,V)-2A(q,K',V')\right\|_2
\le c\sqrt d\,e^\zeta,
\qquad
\zeta=\rho+\frac12\log\rho+o(\log\log\rho).
$$
同样的结论也适用于 $B(q,K)$，并且还能同时保证 key 的和不会漂移太多：
$$
\left\|\sum_{k_i\in K}k_i-2\sum_{k_i\in K'}k_i\right\|_2\le c\sqrt d.
$$
这第三个条件看似附属，其实是把一步 halving 变成多步压缩的关键。因为归一化分母 $B(q,K)=\sum_i e^{q^\top k_i}$ 需要有下界；若 key 的整体中心在迭代中漂走，Jensen 给出的 $B(q,K)\gtrsim n$ 就会失效。作者用一个 simultaneous balancing lemma 把 $A$、$B$、key-centering 三个目标合并到同一组符号选择里。

Halving 的标准变换如下。给每个点一个符号 $\sigma_i$，定义误差
$$
E(q)=\sum_i \sigma_i e^{q^\top k_i}v_i.
$$
若 $K'$ 是 $\sigma_i=+1$ 的那一半点，则
$$
A(q,K,V)+E(q)=2A(q,K',V').
$$
所以问题变成：能否选符号，让所有 $\|q\|\le\rho$ 下的向量误差 $\|E(q)\|$ 同时小。这里最自然的第一反应是把 $e^{q^\top k_i}$ 展开成 Taylor series：
$$
e^{q^\top k_i}=\sum_{m\ge 0}\frac{(q^\top k_i)^m}{m!}
=\sum_{m\ge 0}\frac{\langle q^{\otimes m},k_i^{\otimes m}\rangle}{m!}.
$$
于是每个点贡献了无限多个 tensor 层级 $k_i^{\otimes m}\otimes v_i$。要同时控制所有 query 和所有输出方向 $u$，需要控制
$$
\max_{\|\hat q\|=\|u\|=1}
\left\langle \hat q^{\otimes m}\otimes u,\,
\sum_i\sigma_i k_i^{\otimes m}\otimes v_i
\right\rangle.
$$
这就是本文和先前标量 kernel coreset 的主要差别：value 是向量，误差也必须在所有输出方向上小；但测试 tensor 不是任意 $(m+1)$ 阶 tensor，而是特殊的 rank-one 形状 $\hat q^{\otimes m}\otimes u$。

我觉得论文里最关键、也最节制的一步就在这里。作者没有把向量值问题粗暴地展平后套一个很松的 tensor spectral norm 界，而是为 Banaszczyk 定理专门设计 convex body，只约束这些特殊测试方向。具体地，他们把每个点编码成无限拼接向量
$$
x_i=\left(
\frac{v_i}{z(0)},
\frac{\mathrm{flat}(k_i\otimes v_i)}{z(1)},
\frac{\mathrm{flat}(k_i^{\otimes 2}\otimes v_i)}{z(2)},
\ldots
\right),
$$
其中 $z(m)=3(m+2)\log(m+2)$。这个权重让 $\|x_i\|\le 1$，从而可以调用 Banaszczyk。然后定义凸体 $K$，要求第 $m$ 层 Gaussian tensor $G_m$ 在范数
$$
\|G_m\|_\star
=
\max_{\|\hat q\|=\|u\|=1}
\langle \hat q^{\otimes m}\otimes u,G_m\rangle
$$
下不超过 $c d\log m$。由于 $\|\cdot\|_\star$ 只测试两个 $d$ 维单位向量 $(\hat q,u)$，而不是 $m+1$ 个独立方向，epsilon-net 的代价显著更低。这个结构性省法正是 $e^{\rho+o(\rho)}$ 而不是更差参数的来源之一。

拿到所有层级的 bound 后，把 Taylor series 加回去：
$$
\|E(q)\|
\le
\sum_{m\ge 0}
\frac{z(m)\rho^m}{m!}\,\|S_m\|_\star
\le
c\sqrt d
\sum_{m\ge 0}
\frac{\sqrt{m+2}\log^{3/2}(m+2)\rho^m}{m!}.
$$
最后这个和式由 Poisson 型集中控制，量级是
$$
c\sqrt d\,
e^{\rho+\frac12\log\rho+O(\log\log\rho)}
=c\sqrt d\,e^\zeta.
$$
这里的 $\frac12\log\rho$ 可以理解为 Taylor 层级在 $m\approx\rho$ 附近的宽度代价；它也是 upper bound 和 lower bound 之间剩下的主要缝隙。

从 halving 到最终 coreset 只差一个 telescoping 计算。假设当前 cache 大小为 $n$，且 key 仍 roughly centered，则 Jensen 给出 $B(q,K)\ge n/2$。若一次 halving 造成 $A$ 和 $B$ 的未归一化误差都至多 $c\sqrt d e^\zeta$，则归一化 attention 的误差至多
$$
O\!\left(\frac{\sqrt d\,e^\zeta}{n}\right).
$$
每次把 $n$ 减半，误差按几何级数累加；当剩余规模降到 $O(\sqrt d e^\zeta/\varepsilon)$ 时停止，总误差仍在 $\varepsilon$ 内。这给出主上界：
$$
\textbf{Theorem 4.}\quad
\exists (K',V'),\ |K'|=
O\!\left(\frac{\sqrt d\,e^\zeta}{\varepsilon}\right),
\quad
\sup_{\|q\|\le\rho}
\left\|\mathrm{Attn}(q,K,V)-\mathrm{Attn}(q,K',V')\right\|_2
\le \varepsilon.
$$

Lower bound 部分也很干净，走的是 communication complexity 里的 one-way Indexing。Indexing 是这样一个标准问题：Alice 有一个长 bit string，Bob 有一个位置，Alice 发一条消息后 Bob 要恢复该位置的 bit；若要常数成功率，通信量必须和 bit string 长度同阶。论文把一个 $m\times d$ 的 bit 矩阵嵌入到 attention cache 里。先取一组近乎正交的 spherical code vectors $u_1,\ldots,u_m$ 作为 keys；Bob 想读第 $(i,j)$ 个 bit 时，就发 query $q=\rho u_i$。因为
$$
e^{\rho\langle u_i,u_i\rangle}=e^\rho=L
$$
而其他 codeword 与 $u_i$ 的内积很小，目标 key 的 softmax 权重给出信号，其他 key 的贡献只是带随机符号的噪声。Chebyshev 控制噪声，Markov 把 $\ell_2$ 误差转成随机坐标上的小误差，于是一个太小的 sketch 或 coreset 就能违反 Indexing lower bound。

论文先证明任意 sketch 需要
$$
\Omega\!\left(
d\cdot
\min\left\{\frac{L}{\varepsilon},L^2,\exp\!\left(\frac{a d_k}{\log^2 L}\right)\right\}
\right)
\quad\text{bits},
\qquad L=e^\rho,
$$
然后对 coreset 证明
$$
\textbf{Theorem 7.}\quad
S=
\Omega\!\left(
d\cdot
\min\left\{\frac{L}{\varepsilon\sqrt d},L^2,
\exp\!\left(\frac{a d_k}{\log^2 L}\right)\right\}
\right).
$$
当 key dimension $d_k$ 足够大、不成为 spherical code 的瓶颈，并且参数落在第一项主导的区域，这就是
$$
S=\Omega\!\left(\frac{\sqrt d\,e^\rho}{\varepsilon}\right).
$$
这个 lower bound 和 upper bound 的形式非常对齐：$\sqrt d$ 来自向量输出维度与 $\ell_2$ 误差，$e^\rho$ 来自 bounded-query softmax 能放大的最大信号，$1/\varepsilon$ 则是 coreset 近似的线性精度成本。

这篇论文让我最意外的是它没有把 attention 当成一个需要全新模型的深度学习对象，而是把它拆回 TCS 熟悉的三个模块：analytic kernel expansion、vector balancing、Indexing lower bound。真正新的地方也因此很清楚：标量 kernel coreset 的 discrepancy machinery 要穿过 value 向量这一层，不能只保留“对所有 $q$ 的标量和近似”。作者靠特殊 tensor norm 把这一步做得足够紧，才让上界贴到 lower bound 的 $e^\rho$ 尺度上。

剩下的 $\rho^{1/2+o(1)}$ gap 不是特别大，但也不是看起来会被调常数抹掉的那种。论文最后指出，在 coreset 真正有意义的区间里，这个 gap 最坏可对应到约 $\log n$ 因子。我的理解是，若要完全去掉它，可能需要绕开“逐层 Taylor + Gaussian tensor norm + union over $m$”这条路线；而在这条路线内部，$m\approx\rho$ 的层级宽度似乎会反复把 $\sqrt\rho$ 带回来。

## Algorithmic Phase Transition for Large Independent Sets in Dense Hypergraphs [arXiv:2605.05618](https://arxiv.org/abs/2605.05618)

这篇论文研究的是 dense random hypergraph 里的 large independent set 问题：给定一个随机 $r$-uniform hypergraph，目标是找一个尽可能大的顶点集合，使得其中不包含任何一条完整的 hyperedge。作者考虑两个模型。第一个是 $H_r(n,p)$，顶点集为 $[n]$，每个 $r$ 元子集独立地以常数概率 $p$ 成为 hyperedge；第二个是 $r$-partite 模型 $H(r,n,p)$，顶点分成 $r$ 个大小为 $n$ 的部分 $V_1,\ldots,V_r$，每条跨越所有部分的 $r$-tuple 独立地以概率 $p$ 出现，并且要求输出的 independent set 按给定比例 $\gamma=(\gamma_1,\ldots,\gamma_r)$ 分布在各部分中。

主要结果是一个完整的 online 算法相变图景。令 $b=1/(1-p)$。在 $H_r(n,p)$ 中，最大 independent set 的规模 whp 是
$$
\alpha^{\mathrm{STAT}}_r=(r!\log_b n)^{1/(r-1)},
$$
而 online 算法能达到、且不能超过的阈值是
$$
\alpha^{\mathrm{COMP}}_r=((r-1)!\log_b n)^{1/(r-1)}.
$$
所以 dense $r$-uniform hypergraph 里的 online 近似因子正好是 $r^{1/(r-1)}$；当 $r=2$ 时，这就是经典 random graph $G(n,p)$ 的 factor-2 gap。对 $H(r,n,p)$ 的 $\gamma$-balanced 版本，最大规模是
$$
\alpha^{\mathrm{STAT}}_{r\text{-}\mathrm{bal}}
=\left(\frac{\log_b n}{\prod_{i=1}^r\gamma_i}\right)^{1/(r-1)},
$$
而 online 阈值是
$$
\alpha^{\mathrm{COMP}}_{r\text{-}\mathrm{bal}}
=\left(\frac{\gamma_{\max}\log_b n}{\prod_{i=1}^r\gamma_i}\right)^{1/(r-1)},\qquad
\gamma_{\max}=\max_i\gamma_i.
$$
对应 gap 是 $\gamma_{\max}^{-1/(r-1)}$。本文不仅给出达到这些 $\alpha^{\mathrm{COMP}}$ 的 online greedy-type 算法，还证明任何 online 算法都不能以非可忽略概率超过 $(1+\varepsilon)\alpha^{\mathrm{COMP}}$。

先把问题本身说清楚。Independent set 在图里是“两两之间没有边”的顶点集合；在 $r$-uniform hypergraph 里，禁止的不是一对顶点，而是 $r$ 个顶点同时组成一条 hyperedge。Dense 指的是 $p$ 是常数，所以一个顶点周围的局部邻域巨大，完全不像 sparse graph 那样有近似树状结构。随机模型里的优化问题通常有两个阈值：一个是 information-theoretic 或 statistical threshold，表示解在随机实例中是否存在；另一个是 computational threshold，表示自然算法能否找到。本文的重点正是证明，在 dense hypergraph 的 online 算法类里，这两个阈值之间的 gap 是精确的。

这里的 online 不是 online learning 里的 regret 模型，而是一个顺序揭示模型。算法一次选择一个尚未出现的顶点 $v_t$，看到所有由 $v_t$ 和此前已揭示顶点组成的相关 hyperedge 是否存在，然后必须立刻决定是否把 $v_t$ 放进输出集合；这个决定不能反悔。Karp (1976) 对 $G(n,1/2)$ 的 greedy 算法就在这个模型里：最大 independent set 约为 $2\log_2 n$，而简单 greedy whp 找到约 $\log_2 n$。Karp 问能否用有效算法找到 $(1+\varepsilon)\log_2 n$ 大小的 independent set，或者证明做不到。将近五十年后，这个 dense random graph 的 factor-2 barrier 仍然是 average-case complexity 的核心问题之一。

为什么要特别强调 online？因为 dense 和 sparse 在算法代理上差别很大。Sparse random graph $G(n,d/n)$ 的局部邻域近似 Galton-Watson tree，很多算法可以被描述为 local algorithm：每个顶点只看半径有限的邻域，然后决定是否加入解。Gamarnik and Sudan (2013) [arXiv:1304.1831](https://arxiv.org/abs/1304.1831) 首先用 Overlap Gap Property 证明 sparse random graph 中 local algorithms 不能突破某个大 independent set 阈值；Rahman and Virag (2017) 进一步证明 local algorithms 在 sparse independent set 上 half-optimal；Wein (2020) [arXiv:2010.06563](https://arxiv.org/abs/2010.06563) 把 sharp half-optimal lower bound 推到 low-degree polynomial algorithms。更近一些，Dhawan and Wang (2024) [arXiv:2404.03842](https://arxiv.org/abs/2404.03842) 研究 sparse random hypergraph，给出了与本文形式上非常相似的 LDP gap。

Low-degree polynomial method 的角色是：把一个算法的输出坐标近似成输入随机变量的低次数多项式，然后用多项式矩的计算来证明算法不可能达到某个阈值。它在 sparse 模型中很有用，因为局部树状结构能控制相关性和方差。但在 dense $G(n,p)$ 中，连深度 1 邻域都是 $\Theta(n)$ 级别，local limit 消失；更关键的是，dense greedy 算法本身是 unstable 的，而 LDP lower bound 通常正是利用稳定性。论文引用的 AIM workshop conjecture 甚至说，在 $G(n,1/2)$ 中，degree-$o(\log^2 n)$ polynomial 可能连 $0.9\log_2 n$ 的 independent set 都找不到。若这个观点正确，LDP 在 dense easy regime 里都不是合适的算法代理。本文于是把在线 greedy 及其推广当成 dense setting 中应当被精确刻画的算法类。

:::aside[OGP 的作用]
Overlap Gap Property 说的是：如果两个候选解都足够好，那么它们的 overlap 不会落在某个中间区间，要么非常相似，要么几乎不相交。算法下界通常证明：若某类算法能稳定地产生好解，那么沿着相关随机实例的插值路径运行算法，会连续地产生 overlap 从大到小变化的解，从而必然穿过 forbidden overlap 区间，矛盾。本文使用的是为 online 算法改造过的 OGP 思路，因为 online greedy 不满足传统 local/LDP 分析所需的稳定性。
:::

先看 $H_r(n,p)$ 的 statistical threshold。固定一个大小为 $\alpha$ 的顶点集合，它成为 independent set 的概率是
$$
(1-p)^{\binom{\alpha}{r}}=b^{-\binom{\alpha}{r}}.
$$
大小为 $\alpha$ 的候选集合有 $\binom{n}{\alpha}$ 个，所以一阶矩大约是
$$
\mathbb{E}Z_\alpha
\approx \exp\!\left(\alpha\log n-\frac{\alpha^r}{r!}\log b\right).
$$
让指数主项从正变负，得到 $\alpha^{r-1}\approx r!\log_b n$，这就是 $\alpha^{\mathrm{STAT}}_r$。论文用一阶矩给上界、二阶矩给下界，证明对任意固定 $\varepsilon>0$，
$$
\Pr[Z_\alpha>0]\to 0\quad \text{if }\alpha\ge (1+\varepsilon)\alpha^{\mathrm{STAT}}_r,
$$
且
$$
\Pr[Z_\alpha>0]\to 1\quad \text{if }\alpha\le (1-\varepsilon)\alpha^{\mathrm{STAT}}_r.
$$
这部分技术不是本文最难的地方，但它把后面的算法 gap 定标得很干净：统计上能存在到 $(r!\log_b n)^{1/(r-1)}$，算法上却只到 $((r-1)!\log_b n)^{1/(r-1)}$。

达到 computational threshold 的算法在 uniform 模型里就是直接 greedy：当前已经选了 $i-1$ 个顶点时，新顶点被接受当且仅当它不与其中任意 $r-1$ 个旧顶点形成 hyperedge。这样的坏 hyperedge 有 $\binom{i-1}{r-1}$ 个，所以接受概率是
$$
b^{-\binom{i-1}{r-1}}.
$$
从大小 $i-1$ 增长到大小 $i$ 的等待时间近似一个成功概率为 $b^{-\binom{i-1}{r-1}}$ 的几何随机变量。若目标 $k=(1-\varepsilon)\alpha^{\mathrm{COMP}}_r$，则
$$
\binom{k-1}{r-1}\lesssim (1-\varepsilon)^{r-1}\log_b n,
$$
于是最后阶段的接受概率仍大约为 $n^{-(1-\varepsilon)^{r-1}}$，在 $n$ 次机会里足够完成 $k$ 次接受。论文精确证明存在 online 算法以
$$
1-\exp(-n^{\Theta(\varepsilon)})
$$
的概率输出大小至少 $(1-\varepsilon)\alpha^{\mathrm{COMP}}_r$ 的 independent set。

Balanced multipartite 模型的算法更有意思。若只做 naive greedy，算法可能过早从某些 part 里拿太多点，最后无法满足 $\gamma$-balanced 约束。作者的 staged and bucketed greedy 把构造分成 $r$ 个阶段：假设 $\gamma_1\le\cdots\le\gamma_r$，算法维护一组已经 "locked" 的 part。每个阶段只从未锁定 part 接受仍保持 independent 的顶点；一旦某个未锁定 part 的数量达到当前目标 $\gamma_j\hat\alpha$，就把这个 part 锁住，进入下一阶段。这里最关键的不是实现复杂，而是分析顺序：先填小 bucket，再逐步锁定，使得每个阶段里形成阻塞所需的 hyperedge 必须跨过已经积累好的多个 part。这个设计正好把接受概率压到 $\gamma_{\max}$ 决定的阈值上，证明当
$$
\hat\alpha\le (1-\varepsilon)\alpha^{\mathrm{COMP}}_{r\text{-}\mathrm{bal}}
$$
时失败概率为 $\exp(-\Omega(n^{\varepsilon/(r-1)}))$。

我觉得这一步是本文最自然、也最容易被低估的算法贡献。Balancedness 是一个全局约束，不是每一步的局部可行性自动会照顾到的东西。Sparse bipartite/multipartite 文献中可以借助 local 或 LDP 结构来控制各 part 的选择比例；dense online 模型里没有这种工具。Staged locking 的作用是把“最后整体比例正确”拆成一系列局部容量事件，同时不破坏 greedy 的独立性检查。这个结构看上去简单，但恰好是 hypergraph 阶数、multipartite 结构和比例约束同时出现时需要的那种算法入口。

下界部分是本文真正把图景闭合的地方。定理可以这样表述：对任意固定 $\varepsilon>0$，不存在 online 算法在 $H_r(n,p)$ 中以概率
$$
\delta=\exp\!\left(-O(\varepsilon^2(\log_b n)^{r/(r-1)})\right)
$$
输出大小至少 $(1+\varepsilon)\alpha^{\mathrm{COMP}}_r$ 的 independent set；对 $H(r,n,p)$ 的 $\gamma$-balanced 版本同样不存在以这个量级概率输出大小至少 $(1+\varepsilon)\alpha^{\mathrm{COMP}}_{r\text{-}\mathrm{bal}}$ 的 online 算法。注意这不是只排除 whp 成功，而是排除非常小但仍非极端小的成功概率；论文称这是 strong hardness。这个概率尺度也基本最优，因为随机猜一个接近 computational threshold 的集合本身成为 independent set 的概率就是相近指数级。

证明的核心延续了 Gamarnik, Kizildag and Warnke (2025) 为 dense online graph 建立的思路，但 hypergraph 和 balanced constraint 迫使他们重新设计 stopping time。先固定一个 deterministic online algorithm；随机性可以通过 conditioning 处理。假设它能超过 computational threshold。作者构造 $m=\Theta(\varepsilon^{-2})$ 个相关随机 hypergraph：它们在某个停止时间 $T$ 之前对算法看到的信息完全一致，之后未揭示的 edge 独立重采样。因为算法是 deterministic，在这些实例上运行时，直到 $T$ 之前输出必须完全相同；但若每个实例最终都成功，它们又都要扩展成很大的 independent set。

这会产生所谓 forbidden tuple：$m$ 个大 independent sets，它们共享一个足够大的前缀交集，同时在各自独立重采样的尾部继续扩展。论文用组合计数证明，随机 hypergraph 中这样的 tuple whp 不存在。Uniform 情形里，停止时间可以设为算法输出刚达到
$$
(1-\mu)\alpha^{\mathrm{COMP}}_r
$$
的时刻。此时 $m$ 个输出共享大小约为这个量的 common core；要让每个输出最终长到 $(1+\varepsilon)\alpha^{\mathrm{COMP}}_r$，尾部需要补上许多顶点，但每个实例的补全部分都必须避开与 core 及自身形成的 hyperedge。把所有可能的 core、补全和停止时间 union bound 掉后，得到 forbidden tuple 的概率是
$$
\exp\!\left(-\Omega((\log_b n)^{r/(r-1)})\right).
$$

Balanced multipartite 下界更细。停止时间不能只看总大小，因为算法可能已经在某些 part 里拿了很多点、另一些 part 还没达到比例要求；这种状态不足以约束后续补全。作者改为在输出中已经包含一个大小为 $(1-\mu)\alpha^{\mathrm{COMP}}_{r\text{-}\mathrm{bal}}$ 的 $\gamma$-balanced 子结构时停止，也就是每个目标比例对应的 part 都已经达到相应数量。由于 part 的标签可以置换，还要记录见证这个 balanced prefix 的 permutation。之后的 forbidden tuple 计数必须同时处理共同前缀、每个输出的 balancedness、各实例自己的 part permutation，以及某个 "last reached" bucket 的等式约束。这里最巧的是 stopping time 本身：它不是为了让证明看起来像 graph case，而是专门把 global balancedness 转成可计数的刚性条件。没有这个停止规则，后面的 union bound 很难同时抓住所有 part 的缺口。

:::aside[本文定理的精确形式]
论文把“online 算法能达到/不能超过”形式化为 $(k,\delta)$-optimize：算法输出大小至少 $k$ 的概率至少为 $\delta$。Theorem 2.6 给出 $H_r(n,p)$ 中 $k=(1-\varepsilon)\alpha^{\mathrm{COMP}}_r$、$\delta=1-\exp(-n^{\Theta(\varepsilon)})$ 的 online 算法；Theorem 2.7 排除 $k=(1+\varepsilon)\alpha^{\mathrm{COMP}}_r$、$\delta=\exp(-O(\varepsilon^2(\log_b n)^{r/(r-1)}))$。Theorems 2.9 和 2.10 给出 multipartite balanced 版本，其中 $\alpha^{\mathrm{COMP}}_r$ 替换为 $\alpha^{\mathrm{COMP}}_{r\text{-}\mathrm{bal}}$。
:::

把本文和 sparse 结果放在一起看，最有意思的是“结论相似，机制不同”。Sparse random hypergraph 中，Dhawan and Wang (2024) 的 LDP lower bound 也给出 uniform 模型的 $r^{1/(r-1)}$ gap，以及 balanced 模型的 $\gamma_{\max}^{-1/(r-1)}$ gap；但那里算法代理是 low-degree/local，证明依赖 sparse 结构和稳定性。本文的 dense 模型里，这些技术入口基本关闭，online 算法又恰恰是不稳定的。最后得到同样形式的 gap，说明这个 gap 可能不只是某个证明技术的产物，而是 independent set landscape 在不同密度、不同算法类下反复出现的几何限制。

这也解释了论文摘要里说的 "universality" 应该怎样理解。它不是说 dense 和 sparse 的证明可以互相搬运；恰好相反，本文最重要的技术工作就是证明不能搬运时该怎么重建下界。Uniform greedy 的等待时间分析、multipartite 的 staged buckets、online OGP 的停止时间和 correlated instances，分别替代了 sparse 世界里的树状局部极限、低次多项式方差控制和稳定插值。最后相变位置却落在同一个简单比例上，这比单纯多一个 hypergraph generalization 更有信息量。

论文最后提到 limited future information 的模型也值得留意。Gamarnik, Kizildag and Warnke (2025) 以及 Dhawan, Kizildag and Maitra (2025) [arXiv:2508.20785](https://arxiv.org/abs/2508.20785) 在 dense graph/bipartite graph 中研究过允许算法查询少量未来 edge 的增强 online 模型，并证明这种额外信息可以越过普通 online threshold。本文自然猜测 hypergraph 和 multipartite balanced 版本也会如此。对我来说，这个方向的意义不是再调一个模型参数，而是它可能区分两种 barrier：一种来自顺序不可逆决策，一种来自解空间几何本身。本文把无未来信息的 online 边界钉得足够精确，下一步讨论“未来信息买来了什么”才有了清楚的参照系。

## Bilateral Treewidth for QBF: Where Strategies and Resolution Meet [arXiv:2605.06262](https://arxiv.org/abs/2605.06262)

这篇论文研究的是 quantified Boolean formula evaluation（QSat）的结构化算法：给定一个带量词前缀的 CNF 公式 $\Phi = Q.\varphi$，目标是判断存在玩家是否有 winning strategy。作者提出新的宽度参数 **bilateral treewidth**。主结果是：如果输入同时给出一个宽度为 $k$ 的 bilateral decomposition，即本文称作 trunk-aligned tree decomposition，那么 QSat 可在时间
$$
f(k)\cdot |\Phi|\cdot |T|
$$
内判定，其中 $f$ 只依赖于 $k$。这个参数严格强于此前两条互不包含的 prefix-sensitive width 路线：prefix pathwidth 和 dependency treewidth。更具体地，本文给出一族 parity QBF，使得已有两个参数都是 $\Omega(n)$，而 bilateral treewidth 恒为 $2$。

先把对象说清楚。普通 SAT 只问是否存在一个变量赋值使 CNF 为真；QBF 在变量前加上交替的 $\exists/\forall$ 量词，例如 $\exists x\,\forall u\,\exists z\,\varphi(x,u,z)$。它不再只是找一个静态赋值，而是一个两人博弈：存在玩家给 $\exists$ 变量赋值， universal 玩家给 $\forall$ 变量赋值，变量按前缀顺序出现；公式为真，当且仅当存在玩家能根据此前 universal 玩家已经做出的选择来应对，使得无论 universal 玩家怎么走，最后的 CNF 都为真。因此，一个存在变量 $x$ 的取值一般不是常数，而是一个函数，输入是排在 $x$ 左边、会影响 $x$ 的 universal 变量赋值。这就是 strategy。

这也解释了为什么 QBF 是 SAT 的自然强化而不只是语法扩展。QBF 能紧凑表达 planning、verification、synthesis 里的“对所有环境行为，都存在系统响应”式约束；代价是复杂度从 SAT 的 NP-complete 升到 PSPACE-complete。结构化算法的希望来自 SAT 的成功经验：如果把 CNF 的变量看成图的顶点，同一个 clause 中共现的变量连边，得到 primal graph；当这个图 treewidth 很小，SAT 可以用 tree decomposition 做动态规划，在 $f(\mathrm{tw})\cdot n^{O(1)}$ 时间求解。直觉上，treewidth 小意味着变量交互像树，局部信息可以沿树传递。

可是 QBF 上普通 treewidth 会失效。Atserias 和 Oliva (2014) 证明 bounded pathwidth、因此也 bounded treewidth 的 QBF 仍然 PSPACE-complete，回答了 Vardi 提出的 bounded-width QBF 是否 tractable 的问题。这一点非常关键：QBF 的困难不只在 clause-variable 交互图，还在量词前缀。Tree decomposition 只知道两个变量是否共现，不知道谁是 $\forall$、谁是 $\exists$，也不知道一个存在变量能否依赖某个 universal 变量。换句话说，SAT 的动态规划只需要记“边界变量取什么值”；QBF 还要记“某些存在变量作为策略函数如何响应此前的 universal play”。

:::aside[QBF 的依赖关系]
本文使用 dependency poset $D$ 来抽象“变量 $v$ 允许依赖哪些更早的变量”。形式上，$u\preceq_D v$ 表示 $v$ 依赖 $u$；$D$ 必须与前缀顺序兼容，并且任何与 $D$ 一致的变量重排都保持 QBF 真值。最保守的是 trivial dependency poset $D_{\mathrm{trv}}$，它让每个变量依赖所有排在它左边的变量。更精细的 dependency scheme 可以删去语义上不必要的依赖。
:::

过去十多年围绕这个失效点形成了两条路线。第一条是“从前往后”的 strategy / expansion 路线。Eiben, Ganian 和 Ordyniak (2020) 引入 prefix pathwidth：它要求 path decomposition 的忘记顺序尊重量词依赖，粗略说，一个变量被忘记时，它依赖的变量已经在 decomposition 的前缀部分出现过。算法上，这相当于沿路径推进时显式维护存在玩家对 universal 分支的局部策略；它和 proof complexity 里的 universal expansion 很接近。Universal expansion 的思想是把 universal 变量的所有可能赋值分支展开，然后给后续 existential 变量加 annotation，表示“这是在某个 universal play 下的副本”。

第二条是“从后往前”的 resolution / reduction 路线。Chen 和 Dalmau (2005) 的 respectful treewidth、Atserias 和 Oliva (2014) 的重述，以及 Eiben, Ganian 和 Ordyniak (2018) 的 dependency treewidth，都要求变量在 tree decomposition 中以某种反前缀、或反依赖的顺序被消去。对应算法不是显式枚举策略，而是构造 Q-resolution 型推导。Q-Res 的基本操作有两个：对 existential 变量做 resolution，把含 $x$ 和含 $\neg x$ 的 clause 合并来消去 $x$；对 universal 变量做 universal reduction，在它不被更晚的 existential 变量阻挡时从 clause 中删去它。Dependency treewidth 的好处是可以产出小的 Q(D)-Res proof，这在算法证书层面比只给 yes/no 更强。

问题在于，这两条路线并不互相包含。Proof complexity 里早就知道 universal expansion 和 Q-resolution 不模拟彼此；Beyersdorff, Chew 和 Janota (2015) 在 resolution-based QBF calculi 中系统研究了这种分离，后来的 IR-calc/IRM-calc 才尝试把两者放进同一个 proof system。结构参数这边也出现同样裂缝：prefix pathwidth 和 dependency treewidth 可以各自给 FPT，但 Eiben et al. (2018, 2020) 的结果表明它们不可比。本文的出发点很自然：既然 QBF solving 里 expansion 和 resolution 都是必要工具，宽度参数是否也应该允许同一个 decomposition 局部地选择“在这里做 strategy branching”，在别处做“Q-resolution 消元”？

Bilateral treewidth 的定义正是这个混合选择。给定 QBF $\Phi=Q.\varphi$、dependency poset $D$ 和 primal graph 的 nice tree decomposition $T=(T,\chi)$，作者指定一条从叶到根的路径 $\mathrm{Tr}_{<T}$，称为 trunk。一个变量 $u$ 在它的 forget node 处离开当前 bag。Trunk-aligned decomposition 要求每个变量 $u$ 满足下面二选一的条件：
$$
\begin{aligned}
\text{(P1)}\quad &\text{对所有依赖 }u\text{ 的变量 }v,\ v\notin \chi(\mathrm{forget}(u));\\
\text{(P2)}\quad &\mathrm{forget}(u)\in \mathrm{Tr}_{<T}\text{，且所有 }u\text{ 依赖的变量都出现在 }T_{\le \mathrm{forget}(u)}\text{ 中。}
\end{aligned}
$$
Bilateral treewidth $\mathrm{btw}(\Phi,D)$ 就是所有 trunk-aligned tree decomposition 的最小宽度。

这两个条件最好不要只当形式定义看。P1 说：当 $u$ 被忘记时，当前 bag 里没有还依赖 $u$ 的变量。于是如果 $u$ 是 existential，可以安全地对 $u$ 做 resolution；如果 $u$ 是 universal，可以安全地做 reduction，因为不会有“更晚的变量还需要看见 $u$”这种阻塞。P2 只允许发生在 trunk 上，它说：若不能用 P1 安全消去，那就必须已经把 $u$ 所依赖的变量都收进了当前子树。这样算法可以在 trunk 上做一次局部 strategy extension，把 $u$ 及其依赖变量的所有相关 partial strategies 展开。名字 “bilateral” 的含义就在这里：树的非 trunk 部分偏 resolution/reduction，trunk 部分允许 strategy branching；一个 decomposition 同时给两种 proof mechanism 留位置。

:::aside[与已有宽度的包含关系]
Dependency tree decomposition 的条件比 P1 更强，所以每个 dependency tree decomposition 都是 trunk-aligned decomposition。Prefix path decomposition 的条件对应 P2，而且整条 path 可作为 trunk，所以每个 prefix path decomposition 也是 trunk-aligned decomposition。因此固定 $D$ 时，$\mathrm{btw}(\Phi,D)$ 不大于 dependency treewidth，也不大于 prefix pathwidth。
:::

严格增强性由 parity 公式族给出。对 $n\ge 2$，本文使用
$$
\mathrm{QParity}_n :=
\exists x_1\cdots x_n\,\forall u\,\exists z_n\cdots z_1\,
\mathrm{eq}(x_1,z_1)\wedge \mathrm{eq}(u,z_n)\wedge
\bigwedge_{i\in[n-1]}\mathrm{xor}(z_{i+1},x_{i+1},z_i).
$$
这里 $\mathrm{eq}$ 用两个二元 clause 表达相等，$\mathrm{xor}$ 用四个三元 clause 表达奇偶约束。这个族在 QBF proof complexity 里不是偶然出现的：它正是用来分离 Q-Res 和 expansion-style 系统的标准例子之一。本文证明：
$$
\textbf{Theorem 6.}\quad
\forall n\ge 2,\ \exists \Phi\text{ with }2n+1\text{ variables such that }
\mathrm{dtw}(\Phi,D_{\mathrm{trv}})\ge n+1,\quad
\mathrm{ppw}(\Phi,D_{\mathrm{trv}})\ge n,\quad
\mathrm{btw}(\Phi,D_{\mathrm{trv}})=2.
$$
其中 $\Phi$ 就是 $\mathrm{QParity}_n$。

这个分离很有说明力。Dependency treewidth 高，是因为在 trivial dependency 下，消元顺序必须先消 $z$ 变量，再消 $u$，最后消 $x$ 变量；而 $z$ 变量连成链并共同把邻域传递给所有 $x_i$ 和 $u$，fill-in 会迫使某一步看到 $n+1$ 个右侧邻居。Prefix pathwidth 高，则通过 directed pathwidth 的 cops-and-robber 刻画证明：在相应有向图上，$n$ 个 cops 抓不住 robber，robber 可以在 $X_n\cup\{u\}$ 中借助未占用的 $z_i$ 逃跑。可是 bilateral decomposition 只要宽度 $2$：沿一条 trunk 交替放入 $x_i,z_i$，让 $x_i$ 用 P2 做 strategy extension，让 $z_i$ 用 P1 做 resolution，最后处理 $u$。这正是本文参数比“纯 expansion 路线”和“纯 resolution 路线”更强的最小例子。

算法部分的主定理是：
$$
\textbf{Theorem 10.}\quad
\text{存在可计算函数 }f\text{ 和算法，使得给定 QBF }\Phi=Q.\varphi\text{、dependency poset }D
\text{ 以及宽度 }k=\mathrm{btw}(\Phi,D)\text{ 的 trunk-aligned tree decomposition }T,
\text{ 可在 }f(k)\cdot |\Phi|\cdot |T|\text{ 时间判定 }\Phi\text{ 是否为真。}
$$
和很多 treewidth 算法一样，这个 theorem 假设 decomposition 已作为输入给出；如何 FPT 计算最优或近似的 bilateral decomposition 仍是开放问题。

算法维护的状态不是一个表，而是一个 proof-like derivation sequence。每一步有一个 prefix $Q^{(i)}$ 和一个集合 $F^{(i)}$；$F^{(i)}$ 的元素 $\pi$ 又是若干 matrix 的集合。可以把 $\pi$ 理解为“同一个 partial strategy 选择后，所有 universal plays 产生的公式集合”；而 $F^{(i)}$ 枚举所有还可能成功的 partial strategy 选择。于是状态天然是“sets of sets of formulas”。这看起来比 SAT 上的 bag table 繁琐，但它准确反映了 QBF 的博弈语义：存在玩家选策略，universal 玩家选分支。

消去变量时，算法按 decomposition 诱导的 forget 顺序 $v_1,\ldots,v_n$ 推进。对当前变量 $v_i$，有四种规则。若它已被此前的 strategy extension 一并消掉，就跳过。若 $v_i$ 是 existential 且所有依赖它的仍存变量都不在当前 bag 中，就对每个 matrix 全量 resolution 消去 $v_i$。若 $v_i$ 是 universal 且同样不被后续依赖阻塞，就对每个 matrix 做 reduction。剩下的情况只能发生在 trunk 上，于是对每个 $\pi$ 做 strategy extension：枚举 $v_i$ 依赖范围内的 partial existential strategies，并对这些策略下的所有 universal assignments 生成新的 matrix 集合。

这里最关键的正确性点，是 resolution/reduction 什么时候 sound。论文先证明两个局部引理：如果 existential 变量 $x$ 不与任何依赖 $x$ 的 universal 变量同 clause 出现，那么 exhaustive resolution over $x$ 保持 QBF 真值；如果 universal 变量 $u$ 不与任何依赖 $u$ 的 existential 变量同 clause 出现，那么 reduction of $u$ 保持真值。Trunk-aligned decomposition 保证当规则 R2/R3 被调用时，当前 matrix 中与 $v_i$ 共现的变量都在 $\chi(\mathrm{forget}(v_i))$ 内，而 R2/R3 的触发条件又保证这些邻居不是依赖阻塞者。这个“当前 matrix 的邻域仍被 bag 控住”是 Lemma 11，证明要处理 resolution 产生的新 clause；tree decomposition 的连通性和 forget 顺序共同防止新邻居跑出 bag。

Strategy extension 的引理则更接近博弈语义。设 $\pi=\{\psi_1,\ldots,\psi_\ell\}$ 共享 prefix $Q$，要扩展到变量 $v$。令 $B$ 是 $v$ 依赖的 universal 变量的所有赋值，$A$ 是 $v$ 依赖范围内 existential 变量的所有 partial strategies。$\mathrm{Ext}(\pi,v)$ 枚举 $A^\ell$ 中的策略元组；对每个 $\psi_j$ 和每个 $\beta\in B$，把 $\beta$ 与对应策略应用到 $\psi_j$ 上。Lemma 3 说，原来 $\pi$ 中所有 QBF 都为真，当且仅当存在某个扩展后的 $\pi'\in \mathrm{Ext}(\pi,v)$，使得其中所有剩余 QBF 都为真。这个“存在一个 $\pi'$，对其中所有 matrices 成立”的量词形状，正好是存在玩家选 strategy、universal 玩家选 play 的语义。

我觉得本文技术上最值得停下看的地方不是四条规则本身，而是状态数为什么不会爆。Strategy extension 看上去会把状态炸到不可控，因为每次都在枚举策略函数，且每个策略会生成多个 universal 分支。Lemma 14 给出核心界：
$$
|F^{(i)}|\le 2^{2^{2^{O(k)}}},\qquad
\forall \pi\in F^{(i)},\ |\pi|\le 2^{2^{O(k)}}.
$$
这就是结论里 $f(k)$ 三重指数的来源。

证明这个界的思路很细。作者对每个当前 clause 构造 certifying sequence：追溯它是由哪些原始 clause、resolution、reduction、partial assignment 得到的。设某个变量在 node $p$ 被忘记，$p$ 以下已经处理过的区域和 $p$ 以上/旁支尚未处理区域由 bag $\chi(p)$ 分开。Claim 15 证明了一个 separation property：如果一个当前 clause 含有尚未进入 $T_{\le p}$ 的外部变量，那么它的整个 certifying sequence 不会使用 $T_{<p}$ 中的变量。换句话说，跨越 separator 的 clause 不会暗中携带已经忘掉的子树信息。这个性质让作者用两类有限信息刻画 matrix：一类是只落在当前 bag 附近的局部 clause 集合，另一类是 bag 中由 strategy extension 造成的 assignment pattern。两者数量都只依赖于 $k$，于是每个 $\pi$ 和整个 $F^{(i)}$ 都被三重指数函数界住。

这个证明安排也解释了为什么 trunk 条件不是随便拼出来的。P1 给 resolution/reduction 的 soundness；P2 给 strategy extension 的局部性；tree decomposition 的 separator 性质负责把两者交替使用后产生的 clause 仍压在 bag 宽度里。若只看定义，bilateral treewidth 像是把 prefix pathwidth 和 dependency treewidth 生硬取并；但算法分析说明它不是简单 union，而是一个能让 proof objects 局部切换形态的同步条件。

和近期一些 parameterized complexity 论文相比，比如 [2604.25841](https://arxiv.org/abs/2604.25841) 讨论 multi-clique-width 下 fine-grained tightness，或 [2604.18746](https://arxiv.org/abs/2604.18746) 用多个宽度参数给 Capacitated Vertex Cover 画边界，本文的参数设计更带 proof-complexity 色彩。它不是在图参数库里再找一个更细的 width，而是从 QBF solver/proof system 的两种基本推理动作反推 decomposition 应该允许什么。这个角度让我觉得 Theorem 6 比主算法本身更能说明问题：parity formulas 之所以能逃过旧参数，正是因为它们需要在同一条结构分解上交替使用 strategy branching 和 resolution。

本文最后留下的计算 decomposition 问题也不是形式上的尾巴。普通 treewidth 有 Bodlaender (1993) 的固定参数算法和 Bodlaender et al. (2016) 的 $O(c^k n)$ 5-approximation；但 bilateral treewidth 的约束同时涉及 tree 的 trunk、变量 forget node、dependency poset 和 proof rule 可用性，现有 treewidth approximation 技术不能直接套。作者甚至指出，对 prefix pathwidth 和 dependency treewidth，类似的 decomposition 计算问题也还没有完全解决。于是目前的 theorem 更像是一个“如果你能识别这种结构，就能求解并产出小证明”的 meta-algorithm。对 QBF 来说，这已经是重要推进：它把原本分裂的 strategy expansion 和 Q-resolution 两套 FPT 解释，放进了同一个可证明更强的宽度框架里。

## Fast decremental tree sums in forests [arXiv:2605.06555](https://arxiv.org/abs/2605.06555)

这篇论文研究的是 deletion-only dynamic forest 上的求和问题：初始给定一片带点权的 rooted forest，之后只允许删除边 `cut(v)`，同时可能有点权更新 `update-weight(v,x)`，查询要么问某个连通分量的点权总和 `tree-sum(v)`，要么问以 $v$ 为根的子树点权总和 `subtree-sum(v)`。Berendsohn 和 Sokołowski 证明，较简单的 `tree-sum` 版本可以突破通用 dynamic forest 数据结构的 $O(\log n)$：带权情形可在 $O(n)$ 预处理后用总时间 $O((m+n)\log^* n)$ 处理 $m$ 次操作；0-1 权情形甚至是 $O(m+n)$。他们还给出一个 universal optimality 型结果：在 group model 中，算法对固定初始森林 $F$ 和 $m$ 次操作的运行时间为 $O(\mathrm{OPT}(F,m)+m+n)$，其中 $\mathrm{OPT}(F,m)$ 是任何数据结构在同一初始森林上不可避免要做的最优 group 加减次数。

同一篇论文也说明了为什么不能把这个好消息直接推广到 `subtree-sum`。带权 `subtree-sum` 即使没有点权更新、只允许 `cut`，也需要 $\Omega(n\log n)$ 时间处理 $\Theta(n)$ 次操作；0-1 权、无点权更新的版本则有匹配界
$$
\Theta\!\left(n\frac{\log n}{\log\log n}\right)
$$
处理 $\Theta(n)$ 次操作。也就是说，decremental restriction 对“整个连通分量的和”非常有用，但对“以当前根关系定义的子树和”只能帮到一部分。

先把背景说清楚。Dynamic graph data structure 关心的是图随时间变化时，能否避免每次从头计算。最基本的问题是 connectivity：边被插入或删除后，问两个点是否还连通。若图始终是一片森林，Sleator 和 Tarjan (1983) 的 link-cut trees 已经能在 $O(\log n)$ 时间内支持 link、cut、连通性以及很多路径/树聚合查询；后来 Pătraşcu 和 Demaine (2006) 通过 cell-probe lower bound 证明，在完全动态森林里这个对数界基本不能再突破，相关论文见 [arXiv:cs.DS/0502041](https://arxiv.org/abs/cs.DS/0502041)。更一般的 fully dynamic graph connectivity 直到今天也还围绕对数级别打转，例如 Huang et al. (2023) 的随机 Las Vegas 结构达到 $O(\log n(\log\log n)^2)$ amortized update time，[arXiv:1609.05867](https://arxiv.org/abs/1609.05867)。

突破 $O(\log n)$ 的自然方式，是限制更新方向。Incremental forest connectivity 只插入边，这就是 disjoint set union：Hopcroft 和 Ullman (1973) 得到 $O(\log^* n)$，Tarjan (1975) 用 union by rank 和 path compression 把分析推进到 $O(\alpha(n))$，其中 $\alpha(n)$ 是 inverse Ackermann function，增长比 $\log^* n$ 还慢。这个界后来由 Tarjan (1979)、Fredman 和 Saks (1989) 在若干模型下证明为最优。Deletion-only 方向则是 decremental connectivity：初始图给定，之后只删边。对森林而言，Alstrup, Secher 和 Spork (1997) 已经给出总时间 $O(n+m)$ 的最优算法；Łącki 和 Sankowski (2017) 还把相关思想推进到 planar graphs。

如果只问“连通吗”，decremental forest 已经线性了。本文真正卡住的是带权聚合：每个点有一个权重，边删除会把一棵树裂成两棵，`tree-sum(v)` 要返回 $v$ 所在当前连通分量的权重总和。一个朴素想法是沿用 decremental connectivity 的线性结构，再顺手维护每个 component 的总和；问题在于“顺手”并不免费。删除一条边后，原来的总和 $S$ 要分成 $S_1,S_2$，除非能快速知道其中一侧的和，否则只知道 $S$ 没法推出两边。通用 link-cut trees/top trees 可以维护这些聚合，但每次操作回到 $O(\log n)$。本文的第一个贡献就是证明 deletion-only 结构足以把这个代价降到 $O(\log^* n)$，并在 0-1 权下完全线性。

形式化地，作者维护 rooted forest $(F,w)$，权重来自交换群 $G$。操作包括
$$
\begin{aligned}
\mathrm{cut}(v)&:\text{删除 }v\text{ 与其父亲之间的边},\\
\mathrm{update\text{-}weight}(v,x)&: w(v)\leftarrow x,\\
\mathrm{tree\text{-}sum}(v)&: \sum_{u\in T(v)} w(u),\\
\mathrm{subtree\text{-}sum}(v)&: \sum_{u\in F_v} w(u),
\end{aligned}
$$
其中 $T(v)$ 是 $v$ 所在的当前连通分量，$F_v$ 是以 $v$ 为根的当前子树。`tree-sum` 不关心 rooted orientation，只关心 component；`subtree-sum` 则强依赖根方向。这一点后来正好解释了两者复杂度的分裂。

:::aside[Group model]
Group model 把点权看成不透明的交换群元素。数据结构只能对这些元素做加法和减法，不能比较、不能检查二进制表示，也不能利用权重是小整数这样的额外性质。这样定义的好处是，算法的代价真正反映“维护求和信息”需要多少次代数操作，而不是 Word RAM 上某个编码技巧。
:::

`tree-sum` 的第一个定理是：
$$
\textbf{Theorem 1.1.}\quad
\text{带权 rooted forest 可在总时间 }O((m+n)\log^* n)
\text{ 下支持 }\mathrm{cut},\mathrm{update\text{-}weight},\mathrm{tree\text{-}sum}.
$$
它和 prior best 的定量差别很直接：通用 dynamic forest 给 $O((m+n)\log n)$；decremental connectivity 本身给 $O(m+n)$ 但不处理不透明群权重；本文把中间的带权 component-sum 版本降到 $O((m+n)\log^* n)$。

技术入口是 micro-macro decomposition，不过作者为了 rooted trees 的形状把它整理成 cluster decomposition。一个 cluster 是树中的一个连通块，它有唯一的 upper boundary vertex，也就是块里最靠近原树根的顶点；它还至多有一个 lower boundary vertex，也就是块内某个有孩子跑到块外的顶点。这样的限制保证每个 cluster 与外界的交互只有常数个“端口”。Lemma 2.3 说：对任意二叉树 $T$ 和参数 $k$，可在线性时间把 $T$ 分成至多 $6n/k$ 个 cluster，每个大小至多 $k$，并构造对应的 cluster tree。

这个 decomposition 的直觉是：在每个小 cluster 内部处理局部变化，在 cluster tree 上处理大尺度连通分量。若把大小设为 $k=\log n$，cluster tree 只有 $O(n/\log n)$ 个端口；如果递归使用同一个思路，规模会从 $n$ 变成 $\log n$、$\log\log n$、$\log\log\log n$，于是出现 $\log^* n$。这和 union-find 里慢增长函数出现的方式有一点相似，但这里不是靠路径压缩，而是靠一层层把“难的树”压到越来越小的局部实例。

关键细节在于，cluster tree 上不能用一个所有操作都 $O(\log n)$ 的结构。一次在 cluster 内部的 cut，可能改变 cluster tree 某个 boundary vertex 代表的权重，于是会触发 `update-weight`；这种触发次数可以是 $\Theta(m)$，不是 $O(m/\log n)$。如果 cluster tree 结构的 `update-weight` 也要 $\Theta(\log n)$，递归节省会被吃掉。作者先设计一个不对称的基线结构：`tree-sum` 和 `update-weight` 都是 $O(1)$，所有 cut 的总成本是 $O(n\log n)$。它的 cut 做法很朴素但正合适：删边后并行遍历裂开的两侧，先遍历完较小侧，得到其总和后用原总和相减得到另一侧。每个点每次被遍历后所在 component 至少缩小一半，所以总遍历成本 $O(n\log n)$。

Lemma 3.2 是整套递归的抽象版：如果有一个小实例数据结构 $X$，总时间为 $f(n',m')$，那么对大树做大小 $\log n$ 的 cluster decomposition 后，总时间变成
$$
O(n+m)+\sum_i f(n_i,m_i),
$$
其中 $\sum_i n_i\le n$，$\sum_i m_i\le m$，且每个 $n_i\le \log n$。这条式子就是整篇上界的发动机。把基线 $O(n\log n)$ 放进去递归 $t$ 层，得到 Corollary 3.4：
$$
O\!\left(t(m+n)+n\log^{[t]}n\right),
$$
取 $t=\log^* n$ 即得 Theorem 1.1。这里 $\log^{[t]}n$ 是迭代 $t$ 次的对数。

0-1 权版本进一步利用 Word RAM 预计算：
$$
\textbf{Theorem 1.2.}\quad
\text{0-1-weighted rooted forest 可在 Word RAM 中用 }O(m+n)
\text{ 总时间支持 }\mathrm{cut},\mathrm{update\text{-}weight},\mathrm{tree\text{-}sum}.
$$
这里的 `update-weight` 只是把权重设为 $0$ 或 $1$。做法是只递归两层，把底层 cluster 缩到 $O(\log\log n)$ 大小。这样小的 labeled forest 状态数是 $k^{O(k)}$，远小于 $n$，可以预先为每个状态存下所有 `tree-sum` 答案以及每个可能 cut/update 后的新状态编码。运行时每个小 cluster 的状态装进一个 machine word，操作就是查表。这一步在 0-1 权下成立，是因为状态空间有限且可枚举；对一般群权重，权值不透明，不能这样压成有限状态。

本文最不寻常的结果是 universal optimality。作者定义 $\mathrm{OPT}(F,m)$ 为：固定初始森林 $F$ 后，所有数据结构中，处理任意 $m$ 次合法操作和任意初始权重时所需 group 加减次数的最小 worst-case 值。然后证明：
$$
\textbf{Theorem 5.1.}\quad
\text{存在单一数据结构，对任意 }F\text{ 和任意 }m\text{ 次操作，总时间为 }
O(\mathrm{OPT}(F,m)+m+n).
$$
论文引言中的 Theorem 1.3 是较粗的 $O(\mathrm{OPT}(n,m)+m+n)$ 版本；Theorem 5.1 更强，因为它允许最优对手把初始森林 $F$ 也 hard-code 进数据结构。换句话说，不是“对所有大小为 $n$ 的森林最坏情形最优”，而是“对每一片具体初始森林都只差常数因子”。Universal optimality 近几年在 shortest paths、sorting、convex hull 等静态/图算法问题里很活跃；把它放进 online data structure，尤其是 group model，技术上要处理的问题明显不同。

:::aside[Universal optimality 的比较对象]
这里的比较对象非常强：允许另一个数据结构知道初始森林 $F$ 和操作数 $m$，只是不知道初始点权和未来操作序列。本文算法本身并不知道这些 hard-coded 信息，却能达到 $O(\mathrm{OPT}(F,m)+m+n)$。
:::

证明路线延续 cluster decomposition，但底层不再用手写小结构，而是对大小 $k=\lfloor\log\log\log n\rfloor$ 的所有 ordered binary trees 预计算“最优数据结构”。这听起来像 Pettie 和 Ramachandran (2002) 的最优 MST 算法：把大实例分解到极小实例，枚举小实例上的最优 decision tree。但本文更麻烦，因为这是 online data structure，不是一次性静态问题；而且 group model 中算法看到的是一连串不透明权重加减，不能靠枚举权重大小关系来验证正确性。作者把相关验证放到 Appendix A，主文真正承担复杂度合成的是一个关于 $\mathrm{OPT}$ 的弱 super-additivity 证明。

最核心的中间结论是 Lemma 5.3。若把 $F$ 的顶点分成若干连通块 $T_1,\ldots,T_k$，并给每块分配操作数 $m_i$，则
$$
\sum_i \mathrm{OPT}(T_i,3m_i)\le O(\mathrm{OPT}(F,m)+n),
$$
其中 $m=\sum_i m_i$。这个式子正好允许把“每个小 cluster 上跑最优结构”的成本加起来，再由全局 $\mathrm{OPT}(F,m)$ 支付。

有意思的是，作者并不能直接证明 $\mathrm{OPT}$ 的真正 super-additivity。他们先引入 $\mathrm{OPT}_0$：初始所有权重都是 $0$ 的版本。这个版本容易拼接，因为子实例的初始权重没有跨块信息，能证明
$$
\mathrm{OPT}_0(F[U_1],m_1)+\mathrm{OPT}_0(F[U_2],m_2)
\le \mathrm{OPT}_0(F,m_1+m_2)
$$
对凸的顶点划分成立。然后再通过一系列 reductions 把普通 $\mathrm{OPT}$ 拉回 $\mathrm{OPT}_0$，并用一个“多实例合并”引理控制操作数放大：$\mathrm{OPT}(F,3m/2)\le 3\mathrm{OPT}(F,m)$。我觉得这是全文最有信息量的一步：universal optimality 的难点不在 cluster decomposition 本身，而在证明局部最优成本不会因为分解而凭空变大。

`subtree-sum` 的部分则给出了清晰的边界。若允许点权更新，`subtree-sum` 已经包含 classic partial-sum problem：把数组看成一条从右往左 rooted 的路径，前缀和就是某个节点的 subtree sum，数组更新就是点权更新。Pătraşcu 和 Demaine (2006) 给 partial sums 证明 $\Omega(n\log n)$ lower bound，于是有
$$
\textbf{Observation 1.4.}\quad
\mathrm{subtree\text{-}sum}+\mathrm{update\text{-}weight}
\text{ 在 Word RAM 上处理 }n\text{ 次操作需 }\Omega(n\log n).
$$

更强的是作者去掉 `update-weight` 后仍能证明：
$$
\textbf{Theorem 1.5.}\quad
\text{带权 forest 仅支持 }\mathrm{subtree\text{-}sum}\text{ 和 }\mathrm{cut}
\text{，处理 }\Theta(n)\text{ 次操作仍需 }\Omega(n\log n).
$$
这说明删除叶子本身就足以模拟数组值更新。构造有两棵树 $T^+$ 和 $T^-$，每棵都有一条长度 $n'$ 的 spine，第 $i$ 个 spine vertex 代表数组位置 $i$。在 $T^+$ 里给每个 spine vertex 挂 $8n'$ 个不同权重的叶子；在 $T^-$ 里挂 $n'$ 个单位权重叶子。一次数组更新 $A[i]\leftarrow A[i]+\Delta$ 被翻译成三次 cut：在 $T^+$ 的第 $i$ 个位置剪掉两个权重和为 $7n'-\Delta$ 的叶子，在 $T^-$ 剪掉一个对应 epoch 的单位叶子。这样表达式
$$
\mathrm{subtree\text{-}sum}(v^+_k)-7n'\cdot \mathrm{subtree\text{-}sum}(v^-_k)
$$
对 $k<i$ 不变，对 $k\ge i$ 增加 $\Delta$，于是 prefix sum 被完整模拟。为了保证每次都能找到还没剪掉的两个叶子，作者用一个 Las Vegas 随机采样子程序；每个 epoch 之前被剪掉的候选少于 $2n'$，而可用配对有线性多个，所以期望常数次采样成功。

0-1 权的 `subtree-sum` 没有这么硬，但也不是线性：
$$
\textbf{Theorem 1.6.}\quad
\text{0-1-weighted forest 支持 }\mathrm{subtree\text{-}sum}\text{ 和 }\mathrm{cut}
\text{ 的总时间为 }
O\!\left((m+n)\frac{\log n}{\log\log n}\right).
$$
匹配 lower bound 是
$$
\textbf{Theorem 1.7.}\quad
\text{unweighted forest 上同一问题处理 }n\text{ 次操作需 }
\Omega\!\left(n\frac{\log n}{\log\log n}\right).
$$
下界从 Fredman 和 Saks (1989) 的 partial-sum-parity lower bound 来：数组位 $A[i]$ 用挂在路径点 $v_i$ 下的一两个叶子表示，翻转一次 $A[i]$ 就 cut 掉一个叶子，$\mathrm{subtree\text{-}sum}(v_k)$ 的奇偶性就是前 $k$ 位的 parity。

上界仍是 cluster decomposition，但参数完全不同。`tree-sum` 可以用大小 $\log n$ 的 cluster 递归出 $\log^* n$；`subtree-sum` 需要在每层维护子树方向上的局部计数，作者改用大约 $k=\sqrt{\log n}$ 个 cluster tree 节点可装入一个 word 的设计，递归深度变成
$$
O(\log_k n)=O\!\left(\frac{\log n}{\log\log n}\right).
$$
小 cluster tree 内部结合 Dietz (1989) 的 delayed-updating 思想：数组 $A$ 存上次 flush 时的 subtree sums，数组 $B$ 存自上次 flush 后每个点被“递减”的次数，查询时用预计算表 $Q[B,U]$ 快速得到某个子树集合 $U$ 中的总递减量。这里最巧的地方是把 cut 后的“谁还在 $v$ 的子树里”编码成 bitset $C[v]$，而 $B$ 和 $C[v]$ 都在一个 word 内，查表就能把延迟更新一次性扣掉。

这篇论文给出的图景相当干净：decremental forest connectivity 的线性算法并没有自动解决带权聚合；但如果查询只问 connected component 的总和，cluster decomposition 可以把动态性压缩到极小局部实例，甚至达到 instance-wise optimal。若查询改成 rooted subtree，partial sums 的硬度会重新出现，区别只在于权重是否足够丰富。对我来说，最值得留下的不是 $O(\log^* n)$ 这个慢增长函数本身，而是 $\mathrm{OPT}(F,m)$ 那条线：它把“这片初始森林到底有多难维护”从 worst-case $n$ 中剥离出来，而 cluster decomposition 恰好提供了把这种 instance-specific 难度局部化再拼回去的语言。

## 未入选但值得关注

### An Improved Construction of Variety-Evasive Subspace Families [arXiv:2605.05621](https://arxiv.org/abs/2605.05621)
这篇做显式 variety-evasive subspace family 构造，并接近 Guo 下界；未入选主要是因为今天 pseudorandom/algebraic complexity 方向没有形成更强的组合，优先保留了 separator 与 phase-transition 两类结果。

### On the Parameterized Approximability of (Mergeable) Sum of Radii Clustering [arXiv:2605.06398](https://arxiv.org/abs/2605.06398)
论文给出 W[2]-hardness、排除 EPAS，并在 mergeable constraints 下改进 FPT 近似，参数化近似内容很扎实；未入选是因为它与 QBF 同属参数化复杂性，今天只保留一个参数化主题。

### The Pareto Frontier of Randomized Learning-Augmented Online Bidding [arXiv:2605.06106](https://arxiv.org/abs/2605.06106)
这篇刻画随机 learning-augmented online bidding 的 consistency-robustness frontier，问题定义清楚、边界也完整；未入选是因为模型范围较窄，优先级低于 dense hypergraph 的 online upper/lower threshold。

### Near-Tight Approximation Algorithms for Bottleneck Multiple Knapsack Problems [arXiv:2605.05233](https://arxiv.org/abs/2605.05233)
论文在 identical/arbitrary capacities 下给出近 tight 的近似算法与 hardness，结果成对出现、形式干净；未入选是因为它更像经典 knapsack 变体的界限补全，技术新意相对弱一些。

### A Constant-Factor Approximation for Continuous Dynamic Time Warping in 2D [arXiv:2605.05917](https://arxiv.org/abs/2605.05917)
Continuous DTW in 2D 的常数近似如果细节成立会很值得读，而且论文标注 ICALP 2026；未入选是因为本地缓存缺失摘要且无法联网核验技术内容，只能先列为 notable。
