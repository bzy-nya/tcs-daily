## 编辑按语

今天的论文覆盖了四个不同方向：子图枚举、参数化字符串算法、transformer 表达能力，以及无线 radio network 中的低能耗分布式算法。它们没有一个统一主题，放在一起更像是 TCS 的四个切面：一个在图结构和数据库 join 之间移动，一个把经典 FPT 问题的指数底数做到条件最优，一个用电路复杂性刻画现代模型架构，另一个把生成树结构转成可执行的通信 schedule。

其中两篇都和“正确的复杂度参数”有关，但参数来自完全不同的对象：Binary Closest String 的参数是最优 Hamming 半径 $d$，radio aggregation 的参数是最小最大度生成树的 $\Delta^\ast$。另外两篇则分别展示通用框架的边界：even-cycle listing 低于 submodular-width 屏障，padded transformer 的表达能力则稳定落回 uniform $\mathrm{AC}/\mathrm{TC}$ 层级。

## Listing Even Cycles Faster than the Submodular-Width Barrier [arXiv:2605.30564](https://arxiv.org/abs/2605.30564)

这篇论文研究的是无向图中的 even-cycle listing 问题：给定一张有 $m$ 条边的无向图 $G$ 和常数 $k\ge 2$，目标是输出 $G$ 中所有长度为 $2k$ 的简单环，也就是所有顶点两两不同、边按环相连的序列 $(v_1,\ldots,v_{2k})$。作者证明，对所有常数 $k$，可以在
$$
\tilde O\!\left(m^{(2k^2-k+1)/(k^2+1)}+t\right)
$$
时间内列出所有 $C_{2k}$，其中 $t$ 是输出环的个数。对 $k\ge 3$，这个指数严格优于 Alon-Yuster-Zwick (1997) 的 $\tilde O(m^{2-1/k}+t)$；对 $k=3$ 它给出 $\tilde O(m^{8/5}+t)$，匹配 Vassilevska Williams-Westover (2025) 的 6-cycle listing 指数，但去掉了很重的多对数因子和计算机辅助分类。

这个问题之所以不只是一个图算法里的参数优化，是因为它同时处在两个传统的交界处。一边是子图枚举：我们不只问图里有没有一个小模式，而是要把所有出现都列出来，运行时间必须是“预处理成本 + 输出大小”。另一边是数据库里的 conjunctive query evaluation：一张图就是一个二元关系 $E(u,v)$，找一个固定小图模式就是在关系上做 join。Cycle query 可以写成
$$
C_{2k}(v_1,\ldots,v_{2k}) \leftarrow
E(v_1,v_2)\wedge E(v_2,v_3)\wedge\cdots\wedge E(v_{2k},v_1)\wedge
\bigwedge_{i\ne j} v_i\ne v_j.
$$
这里最后的 $v_i\ne v_j$ 很重要：没有它，查询输出的是 closed walks，允许顶点重复；有它，输出才是真正的 simple cycles。Listing 版本的难点正是在这里，因为 closed walks 可能远多于 cycles。比如一棵树没有任何 simple cycle，但有大量偶长 closed walks；先列 walks 再过滤不是 output-sensitive 算法。

从数据库视角看，固定 conjunctive query 的求值复杂度通常由某种 width 参数控制。最经典的无环情形来自 Yannakakis (1981)：acyclic join 可以线性时间求值。后来 Grohe-Marx、Atserias-Grohe-Marx 等工作把这个图景推到 fractional cover 和 hypertree width。Marx (2013，预印本 [arXiv:0911.0801](https://arxiv.org/abs/0911.0801)) 引入 submodular width，给出了参数化意义上的 tractability 边界；Abo Khamis-Ngo-Suciu 的 PANDA 框架进一步把它变成一般 CQ 的求值算法，时间形如 $O(N^{\mathrm{subw}(Q)}+t)$。我在 [2026-03-17 日报](#2026-03-17) 里写过 Jaguar，它延续的正是这条“如何真正达到 submodular-width 指数”的线。

:::aside[submodular width 的位置]
对一个固定 CQ，submodular width 可以粗略理解为：允许用满足 Shannon-type 不等式的 set function 衡量变量集合的“信息量”，然后寻找 tree decomposition 中最大 bag 的最优上界。它比普通 treewidth 更会利用 degree/statistics 信息。本文不是重新定义这个参数，而是指出无向 even-cycle query 因为有 self-join 和对称边关系，可以低于这个通用屏障。
:::

Cycle query 是这套理论的一个标志性例子。Alon, Yuster, and Zwick (1997) 给出经典算法：在 $m$ 边图中，长度为 $2k$ 的 cycle detection/listing 可以做到 $\tilde O(m^{2-1/k})$ 预处理时间。这正好等于 $2k$-closed-walk query 的 submodular width 指数 $2-1/k$，再配合 color-coding 处理 distinctness 约束。所谓 color-coding，就是随机或伪随机地给顶点染上 $2k$ 种颜色，只保留颜色全不同的候选；重复 $2^{O(k)}\log n$ 次后，每个 simple cycle 都会在某次染色中变成“彩虹环”。论文用 Boolean tensor decomposition 的语言重述这一步：distinctness 谓词可以拆成若干 unary color filters，于是 cycle query 被化成若干个带 unary filters 的 closed-walk queries。

这个 AYZ 指数卡了接近 30 年。近年的一个有点微妙的事实是：Bringmann and Gorbachev (STOC 2025) 给出了 submodular-width barrier 的细粒度 lower-bound evidence，说明对一大类 colored subgraph listing 问题，通用屏障很可能就是最优。但本文处理的不是那个完全一般的 colored 版本。在 colored 版本里，$2k$ 个边谓词可以是独立的二元关系 $E_1,\ldots,E_{2k}$，它们分别连接相邻颜色类；而这里所有边谓词共享同一个 $E$，并且 $E$ 是无向对称关系。这两个限制不是技术细节：共享关系排除了很多 adversarial gadget，边的对称性则让 extremal graph theory 能插进来。

检测问题已经显示出这种差异。Yuster and Zwick (1997) 在稠密图中给出 $O(n^2)$ 的 even-cycle detection；Dahlgaard, Knudsen, and Stöckel (STOC 2017) 对稀疏图证明 $C_{2k}$ detection 可以在 $O(m^{2k/(k+1)})$ 时间内完成，优于 AYZ 的 $m^{2-1/k}$。Lincoln and Vyas (2020) 又给出证据说明，当 $k$ 为奇数时这个检测指数可能已经最优。Listing 则更硬：Jin and Xu (STOC 2023) 以及 Abboud-Khoury-Leibowitz-Safier (FSTTCS 2023, [arXiv:2211.10022](https://arxiv.org/abs/2211.10022)) 把 4-cycle listing 做到 $\tilde O(m^{4/3}+t)$，Vassilevska Williams and Westover (ITCS 2025, [arXiv:2411.07499](https://arxiv.org/abs/2411.07499)) 把 6-cycle listing 做到 $\tilde O(m^{8/5}+t)$。但对一般 $k\ge 4$，此前没有统一突破 AYZ listing bound 的算法。

本文的定量改进可以直接和 AYZ 比较。AYZ 对 $C_{2k}$ listing 的指数是
$$
2-\frac{1}{k}=\frac{2k-1}{k}.
$$
本文指数是
$$
\frac{2k^2-k+1}{k^2+1}.
$$
二者差距为
$$
\frac{2k-1}{k}-\frac{2k^2-k+1}{k^2+1}
=\frac{k-1}{k(k^2+1)}.
$$
所以对每个 $k>1$ 都有严格指数改进；不过 $k=2$ 时已有 $\tilde O(m^{4/3}+t)$ 的更强算法，本文公式给 $m^{7/5}$，真正新的区间是所有 $k\ge 3$，尤其是 $k\ge 4$。

算法的骨架可以先从 AYZ 的 heavy/light 分解理解。选一个阈值 $\Delta$，把度数超过 $\Delta$ 的顶点称为 heavy，其余称为 light。Heavy 顶点最多 $O(m/\Delta)$ 个；如果一个 closed walk 经过某个指定位置上的 heavy 顶点，那么对应 join 的中间关系规模可以压到 $O(m^2/\Delta)$。All-light 情况下，粗略地说一半 walk 有 $m$ 种起边选择，后续每步分支至多 $\Delta$，中间关系规模约为 $O(m\Delta^{k-1})$。令 $\Delta=m^{1/k}$ 平衡两项，得到 $m^{2-1/k}$。这就是 AYZ/submodular width 在 cycle 上出现的基本原因：它没有用到“如果图里环不多，那么结构必须稀疏”这类 extremal 信息。

Dahlgaard-Knudsen-Stöckel 的 detection 算法用了更细的观察：在一个 even cycle 上，可以选度数最高的顶点作锚点，然后把环切成两个长为 $k$ 的半 walk。按度数 bucket $V_p=\{v:2^{p-1}<\deg(v)\le 2^p\}$ 分层，考虑从 $V_p$ 中的锚点出发、其余顶点度数都不超过 $2^p$ 的 $k$-walk。本文沿用论文里的叫法，把这类对象称为 capped $k$-walk。若能证明 capped $k$-walk 总数小，那么 all-light 的 join bag 就小；这正是把图结构信息转化为 query plan 成本的入口。

为了控制 capped walks，Dahlgaard et al. 引入了 $\Phi$-norm。对向量 $x$，$\|x\|_\Phi$ 可以理解为“按阈值扫描坐标大小时看到的累积尾部分布面积”；对矩阵 $A$，$\|A\|_\Phi$ 是由这个向量范数诱导出的 operator norm。这个范数适合这里，是因为邻接矩阵作用在一个集合指示向量上时，$A1_S$ 的第 $v$ 个坐标就是 $v$ 到 $S$ 的邻居数；也就是说，$\|A1_S\|_\Phi$ 正在汇总“有多少顶点连向 $S$ 很多次”的尾分布。Dahlgaard et al. 的引理把 capped $k$-walk 数量 $\kappa_k(G)$ 控制为
$$
\kappa_k(G)\lesssim m\cdot \left(\max_p \|A_p\|_\Phi\right)^{k-1},
$$
其中 $A_p$ 是相应低度诱导子图的邻接矩阵。

本文最关键的新东西，是把“无 $C_{2k}$ 时 capped walks 少”升级成“有 $t$ 个 $C_{2k}$ 时 capped walks 可由 $t$ 付费”。这就需要 supersaturation。普通 Turán theorem 只告诉我们：边足够多就至少有一个 $C_{2k}$。例如 Bondy-Simonovits (1974) 证明 $C_{2k}$-free 图最多 $O(n^{1+1/k})$ 条边。Listing 算法需要更强的定量说法：如果一个局部子图边太多，它不只是含有一个环，而是含有很多环；这些很多环可以被记入输出大小 $t$，于是算法花掉的时间可以向输出收费。

:::aside[supersaturation]
Supersaturation 是 extremal combinatorics 里 Turán-type 结果的定量加强。Turan 型定理说“超过某个边数阈值必然出现一个 forbidden pattern”；supersaturation 进一步说“超过阈值后，pattern 的拷贝数至少按某个函数增长”。本文使用 Jiang-Yepremyan (2020) 的 even-cycle supersaturation：若 $N$ 点 $M$ 边图足够稠密，则含有 $\Omega((M/N)^{2k})$ 个 $C_{2k}$。
:::

论文的技术核心是一条 unbalanced supersaturation 推论。设 $G=(A\cup B,E)$ 是二分图，$|A|=a\ge b=|B|$，边数为 $m$，其中 $C_{2k}$ 数量为 $t$。作者从 Jiang-Yepremyan 的结果推出：只要 $m\gtrsim ab^{1/k}$，就有
$$
t\gtrsim \frac{m^{2k}}{a^k b^k}.
$$
证明里最巧的是处理不平衡二分图的方法：把大的那侧 $A$ 随机切成 $a/b$ 个大小约为 $b$ 的块，每块和 $B$ 形成一个相对平衡的二分图，在每块里调用 supersaturation，再计算一个原始 cycle 被某个块完整捕获的概率。这样就把平衡图上的 supersaturation 转成了适合 join bag 分析的 asymmetric 版本。

由此得到的 edge bound 是全文最有用的中间引理。对任意两个顶点集合 $A,B\subseteq V(G)$，若整张图里一共有 $t$ 个 $C_{2k}$，则
$$
|E(A,B)|\lesssim
|A||B|^{1/k}
+|B||A|^{1/k}
+t^{1/(2k)}\sqrt{|A||B|}.
$$
这个形式和后续 $\Phi$-norm 估计正好对齐：把 $A,B$ 换成某个度数 bucket $B_j$ 与测试集合 $S$ 时，第三项变成 $t^{1/(2k)}\sqrt{|B_j||S|}$。直觉很清楚：如果两个集合之间边太密，要么它已经违反 even-cycle-free 的 Turán 型稀疏性，要么它必须制造很多 $C_{2k}$；后者由全局输出数 $t$ 限制。

把这个 edge bound 喂给 $\Phi$-norm 分析后，作者得到
$$
\|A\|_\Phi
\lesssim
m^{1/(2k)}\Delta^{1/2-1/(2k)}
+t^{1/(2k)}\log \Delta,
$$
其中 $\Delta$ 是当前低度部分的最大度数。再代入 capped-walk 引理，经过 Young 不等式吸收混合项，得到
$$
\kappa_k(G)
\lesssim
m^{3/2-1/(2k)}\Delta^{(k-1)^2/(2k)}
+m^{2k(k-1)/(k+1)}(\log\Delta)^{2k(k-1)/(k+1)}
+t.
$$
这里我觉得最值得注意的是 $t$ 的出现方式：它没有作为一个松散误差项挂在最后，而是通过 supersaturation 进入矩阵范数，再进入 capped-walk 计数，最后恰好变成 output-sensitive 的 $+t$。这也是 listing 比 detection 更难的地方；detection 只需要证明结构足够稀疏，listing 必须证明“不稀疏的部分会产生足够多输出”。

最后的算法仍然是多 query-plan 的 AYZ 结构，只是 all-light 部分换成 capped-walk 账本。Heavy cycles 的成本是 $\tilde O(m^2/\Delta+t)$；light cycles 的成本由上面的 $\kappa_k(G)+t$ 控制。平衡
$$
\frac{m^2}{\Delta}
\quad\text{和}\quad
m^{3/2-1/(2k)}\Delta^{(k-1)^2/(2k)}
$$
得到
$$
\Delta=m^{(k+1)/(k^2+1)},
$$
从而主项指数变成
$$
\frac{2k^2-k+1}{k^2+1}.
$$
论文定理的形式是：
$$
\textbf{Theorem.}\quad
\text{给定一张 }m\text{ 边 }n\text{ 点无向图 }G\text{，可在 }
\tilde O\!\left(m^{(2k^2-k+1)/(k^2+1)}+t\right)
\text{ 时间内列出所有 }C_{2k}.
$$
更精确地，文中 Theorem 4.6 写成 $\lesssim m^{(2k^2-k+1)/(k^2+1)}\log n+t\log n$，常数和 $\lesssim$ 中隐藏的依赖只和固定的 $k$ 有关。

与 Vassilevska Williams-Westover 的 6-cycle 算法相比，本文对 $k=3$ 没有改进指数，但我更愿意把它看成“解释了为什么 $8/5$ 会出现”。之前的证明需要大量分层情况讨论，论文说约有 576 个 cases；这里则是一个统一的 asymmetric supersaturation lemma 加 $\Phi$-norm/capped-walk bookkeeping。对数据库实现也更干净：算法可以表达成 join 和 project 组成的 query rewritings，而不是嵌入 BFS traversal 作为特殊图过程。

这篇论文给 submodular-width barrier 留下的启示也比较具体。Bringmann-Gorbachev 的 lower bound 告诉我们，一般 colored pattern listing 不应指望打破通用宽度指数；本文则说明，一旦查询有大量 self-joins，并且输入 EDB 是同一个对称关系，通用 CQ 框架没有看见的 extremal structure 可能会降低真实复杂度。这里的“对称”不是装饰：无向图的 even-cycle supersaturation、最高度锚点、capped walks，全部都在用边关系 $E(u,v)=E(v,u)$。如果要把这套方法推到更一般的 CQ，真正要找的不是又一个 tree decomposition 参数，而是某种能替代 even-cycle supersaturation 的结构定理。

我读完后最大的感受是，本文并没有试图绕开 AYZ 框架；它保留 color-coding、heavy/light、multiple tree decompositions 这些老部件，然后只替换最核心的账本：从“中间关系最坏有多大”换成“中间关系若大，会强迫多少输出存在”。这一步很小，却正好击中了 listing 算法最该利用、而 submodular width 本身无法表达的信息。

## An Optimal Algorithm for Binary Closest String [arXiv:2605.31417](https://arxiv.org/abs/2605.31417)

这篇论文研究的是 Binary Closest String：给定一组等长二进制串 $X\subseteq\{0,1\}^n$，目标是找一个中心串 $y^\ast\in\{0,1\}^n$，使它到 $X$ 中最远字符串的 Hamming distance 尽量小。若最优最大距离为 $d$，Fischer 和 Habib 证明这个问题可以用随机算法在 $O^\ast(4^d)$ 时间内解决。这里 $O^\ast(\cdot)$ 隐藏输入规模 $|X|$ 与串长 $n$ 的多项式因子。

更精确地说，他们解决的是参数化版本：给定 $d$，判定是否存在 $y$ 使得
$$
\max_{x\in X}\mathrm{HD}(x,y)\le d.
$$
主定理是：
$$
\textbf{Theorem.}\quad \text{Binary Closest String can be solved in randomized time }O^\ast(4^d).
$$
这个 $4^d$ 不是随便一个漂亮常数。Abboud, Fischer, Goldenberg, Karthik C.S. 和 Safier (2023) 在 ESA 证明了一个 fine-grained lower bound：除非 SETH 失败，Binary Closest String 不存在 $O^\ast((4-\varepsilon)^d)$ 时间算法。也就是说，本文把此前 Chen, Ma 和 Wang (2016) 的 $O^\ast(5^d)$ 上界推进到条件最优。

Closest String 可以先从一个很朴素的聚类问题理解。我们有很多条观测序列，希望找一条“共识序列”，让所有输入都离它不太远。Hamming distance 只数两个串有多少位置不同，所以这个问题是在 Hamming cube 里找最小半径的 enclosing ball center。它不是纯理论玩具：通用 PCR primers 设计、motif finding、药物设计里都会出现类似“多个候选序列必须被一个代表序列同时覆盖”的需求。难点在于中心串 $y$ 不一定属于输入集合；它可以在每个坐标上自由选择，于是搜索空间是 $2^n$。

形式化地，Hamming distance 定义为
$$
\mathrm{HD}(x,y)=|\{j\in[n]:x[j]\ne y[j]\}|.
$$
Closest String 的输入是一组 $X\subseteq\Sigma^n$，输出一个 $y^\ast\in\Sigma^n$ 最小化 $\max_{x\in X}\mathrm{HD}(x,y^\ast)$。Binary Closest String 是字母表大小 $|\Sigma|=2$ 的特例。参数 $d$ 是最优半径；当 $d$ 很小而 $n$ 很大时，穷举 $2^n$ 个中心显然不合适，而 $f(d)\cdot \mathrm{poly}(|X|,n)$ 形式的 fixed-parameter algorithm 就有意义。

:::aside[参数化算法与 $O^\ast$ 记号]
参数化复杂性把输入规模和某个结构参数分开看。这里结构参数是最优距离 $d$：算法时间 $O^\ast(c^d)$ 表示指数部分只依赖 $d$，而对 $|X|$ 和 $n$ 只是多项式。这个记号适合比较不同算法的指数底数，例如 $5^d$ 与 $4^d$。
:::

这条线已经推进了二十多年。Closest String 的 NP-hardness 至少可追溯到 Frances 和 Litman (1997)，Lanctôt et al. (2003) 也系统研究了相关 string selection 问题；近似算法方面有 Ben-Dor et al. (1997)、Li, Ma 和 Wang (2002)、Ma 和 Sun (2009) 等经典工作。本文关注的不是近似比，而是精确参数化时间：Gramm, Niedermeier 和 Rossmanith (2003) 给出早期 $O^\ast(d^d)$ FPT 算法；之后 Binary Closest String 的上界一路降到 Ma 和 Sun (2009) 的 $O^\ast(16^d)$、Wang 和 Zhu (2009) 的 $O^\ast(9.513^d)$、Zhao 和 Zhang (2010) 的 $O^\ast(8^d)$、Chen, Ma 和 Wang (2012) 的 $O^\ast(6.731^d)$，最后到 Chen, Ma 和 Wang (2016) 的 $O^\ast(5^d)$。

这个历史脉络里最关键的是 2023 年的反向结果。Abboud et al. (2023) 问的是“Closest String 能不能比穷举更快”，他们构造的 SETH lower bound 蕴含 Binary Closest String 没有 $O^\ast((4-\varepsilon)^d)$ 算法。论文正文里还说明了参数换算：下界实例满足 $d=(1/2\pm o(1))n$，从而 $O^\ast((4-\varepsilon)^d)$ 会打破 $O^\ast((2-\varepsilon')^n)$ 级别的 lower bound；再用 padding 扩展到 $d\le n/2$ 的范围，而 $d>n/2$ 时直接穷举 $2^n$ 已经不比 $4^d$ 差。因此，$5^d$ 到 $4^d$ 的 gap 看起来只差一个常数底数，实际是在 upper bound 和 lower bound 之间最后一段缝。

:::aside[SETH lower bound 在这里怎么读]
SETH 是 fine-grained complexity 中常用的假设，粗略说它断言 $k$-SAT 在 $n$ 个变量上不能对所有 $k$ 都显著快于 $2^n$。本文引用的 Abboud et al. (2023) lower bound 不是说 $4^d$ 绝对不可突破，而是说若存在 $O^\ast((4-\varepsilon)^d)$ 算法，则会推出 SETH 失败。
:::

算法本身短到有点意外。给定阈值 $d$，先任选一个输入串作为当前候选 $y$。只要还没找到答案，就找一个离当前 $y$ 最远的输入串 $x$。如果 $\mathrm{HD}(x,y)\le d$，由于 $x$ 是最远的，说明所有输入串都在 $d$ 内，直接返回 $y$。否则，在所有 $x[j]\ne y[j]$ 的位置中均匀随机选一个 $j$，把 $y[j]$ 改成 $x[j]$。二进制情形下，这一步就是 flip 一个 bit。

写成伪代码几乎就是：
$$
\begin{aligned}
&\text{start with any }y\in X;\\
&\text{repeat: choose }x\in X\text{ maximizing }\mathrm{HD}(x,y);\\
&\text{if }\mathrm{HD}(x,y)\le d\text{ return }y;\\
&\text{pick random }j\text{ with }x[j]\ne y[j]\text{ and set }y[j]\leftarrow x[j].
\end{aligned}
$$
这个算法永远不会误报：它只有在真实检查到所有串都离 $y$ 至多 $d$ 时才返回。随机性只影响它多久能走到解附近；如果运行到设定步数还没返回，算法可以给出 one-sided Monte Carlo 的“未找到”结论。

为什么这样简单的 local search 能达到 $4^d$？固定一个真实解 $y^\ast$，把当前状态定义为
$$
i=\mathrm{HD}(y,y^\ast).
$$
初始时 $y\in X$，而 $y^\ast$ 距离每个输入串至多 $d$，所以 $i\le d$。每一步只改一个 bit，因此 $i$ 只会加一或减一。问题变成：这个随机游走什么时候击中状态 $0$？

关键引理给出了向正确方向走的概率下界。当前选择的最远串是 $x$，且还未终止，所以 $\mathrm{HD}(x,y)>d$。把 $x$ 与 $y$ 不同的位置分三类：

好的位置满足 $x[j]=y^\ast[j]\ne y[j]$，翻它会让 $y$ 更接近 $y^\ast$；坏的位置满足 $x[j]\ne y[j]=y^\ast[j]$，翻它会远离 $y^\ast$；无用的位置满足 $x[j]=y[j]\ne y^\ast[j]$，这类位置不会被算法选中，因为算法只在 $x$ 和 $y$ 不同的位置里抽样。设三类数量分别为 $G,B,U$。那么有三条关系：
$$
\mathrm{HD}(x,y)=G+B>d,\qquad
\mathrm{HD}(x,y^\ast)=B+U\le d,\qquad
\mathrm{HD}(y,y^\ast)=G+U=i.
$$
算法向状态 $i-1$ 移动的概率正是 $G/(G+B)$。把上面三条式子合起来，作者得到
$$
\frac{G}{G+B}
=\frac{(G+B)-(B+U)+(G+U)}{2(G+B)}
\ge \min\left\{\frac{i}{2d},\frac12\right\}.
$$
这就是全文最重要的局部结构：当当前串离某个最优解还有 $i$ 位差异时，即使我们不知道 $y^\ast$，只要去追最远的约束串，随机选到“修正真实错误”的概率至少是 $p_i=\min\{i/(2d),1/2\}$。

这个概率下界也解释了为什么分析不能只说“每步有常数概率变好”。离解很远时 $p_i$ 可以达到 $1/2$，但越靠近解，$i/(2d)$ 越小，最后几步反而最难。Schöning (1999) 的 $k$-SAT 随机算法也有类似精神：当前赋值违反某个 clause，就随机改 clause 里的一个变量；如果固定一个 satisfying assignment，随机步有一定概率减少到该 assignment 的 Hamming distance。本文的 Markov chain 比 Schöning 的 SAT 分析更不均匀，因为成功概率依赖状态 $i$，但最终乘起来刚好给出 $4^d$。

从这里到运行时间的分析很短。设 $D_i$ 是从状态 $i$ 下降到 $i-1$ 之前，期望需要经历多少次“下降步”。如果当前在状态 $i$，以概率至少 $p_i$ 直接下降；否则先上升到 $i+1$，再需要从 $i+1$ 回到 $i$，然后再从 $i$ 到 $i-1$。这给出递推
$$
D_i\le 1+\frac{1-p_i}{p_i}D_{i+1}.
$$
展开后会出现乘积
$$
\prod_{m=1}^{d}\frac{1-p_m}{p_m}
=\prod_{m=1}^{d}\frac{2d-m}{m}
=\binom{2d-1}{d}
\le 4^d.
$$
论文保守地带上状态空间中的多项式因子，得到击中状态 $0$ 的期望迭代次数至多 $4^d\cdot n^2$。用 Markov inequality 放大成功概率：运行 $O(4^d n^3)$ 次迭代后，失败概率指数小；每轮找最远串和更新距离都可多项式实现，所以总时间是 $O^\ast(4^d)$。论文还提到一个微优化：如果某步发现 $\max_{x\in X}\mathrm{HD}(x,y)>2d$，就把 $y$ 重置为任意输入串。由三角不等式，这不会增加到 $y^\ast$ 的距离，并可把多项式因子里的 $n^2$ 改成 $d^2$，得到期望时间 $O(n|X|+4^d d^2|X|)$ 级别的实现。

最值得玩味的是它和 Chen, Ma 和 Wang (2016) 的关系。那篇已经包含一个非常接近的 simple local search，时间是 $O^\ast((2e)^d)=O^\ast(5.437^d)$；同一篇论文的更复杂算法给出此前最优 $O^\ast(5^d)$。Fischer 和 Habib 的改动不是换一个更强的 branching rule，也不是发明新的 kernel，而是改变随机过程的使用方式：不要做很多条长度 $O(d)$ 的独立短游走，而是做一条长度 $O^\ast(4^d)$ 的长游走，并直接分析它的 hitting time。

这一步看起来像小修，但它改变了允许算法犯错的方式。短游走分析通常要求在有限窗口内连续积累足够多的好步，容易损失一个 $e^d$ 型因子；长游走则允许很多次向外偏离，只要总体上还会回到低状态。于是指数底数由“某类成功路径的概率”转为“带状态依赖 drift 的 hitting time”。我觉得这正是这篇论文简洁的地方：它没有试图在组合结构里再挖一个复杂分支，而是承认 local search 会反复走错，然后把走错的成本一次性算清楚。

Binary 这个限制在证明中是实打实用到的。好、坏、无用三分类依赖于每个坐标只有两个符号：当 $x[j]\ne y[j]$ 时，把 $y[j]$ 设成 $x[j]$ 要么命中 $y^\ast[j]$，要么离开 $y^\ast[j]$。若字母表更大，$x[j]$、$y[j]$、$y^\ast[j]$ 可以三者互不相同，局部一步的含义就不再是干净的“接近或远离”。这也是为什么本文虽然关闭了 Binary Closest String 的参数化复杂度，仍没有直接解决更大字母表或 Closest Substring。后者还要额外决定每个输入串中取哪段 substring，本质上多了一层 alignment。

从结果形态看，这篇论文属于那种少见的“最后一块常数因子”工作：问题足够经典，已有二十年上界链条；lower bound 已经把目标钉在 $4^d$；最终算法却只有六行。它提醒人不要低估分析层面的改进。有时最优算法不是更复杂的算法，而是对一个已有随机过程采用更合适的时间尺度。

## Revisiting Padded Transformer Expressivity: Which Architectural Choices Matter and Which Don't [arXiv:2605.30523](https://arxiv.org/abs/2605.30523)

这篇论文研究的是 padded transformer 的表达能力：给定一个输入串，先在后面接上多项式多个专用 filler symbols，再让 transformer 在这些额外位置上做并行中间计算，问这样的模型到底等价于哪些标准电路类。Svete, Merrill, Cotterell 和 Sabharwal 的主结果是一个精确分类：在足够表示容量下，常数精度的 L-uniform padded transformer 正好是 L-uniform $\mathrm{AC}^0$，增长精度的版本正好是 L-uniform $\mathrm{TC}^0$；如果把同一个 transformer block 循环运行 $\Theta(\log^d N)$ 次，则常数精度达到 FO-uniform $\mathrm{AC}^d$，增长精度达到 FO-uniform $\mathrm{TC}^d$。这里真正起作用的是 depth/looping 和 numeric precision；width、softmax vs. average hard attention、以及超过 $\Theta(\log N)$ 的精度增长，在这些条件下都不再改变表达能力。

先把问题放回最基本的模型里。Transformer expressivity 问的是：一个固定架构、固定数值精度的 transformer family 能识别哪些形式语言，或者能计算哪些布尔函数族。之所以要把它接到 circuit complexity，是因为电路给了一个清晰的并行计算标尺：$\mathrm{AC}^0$ 是多项式大小、常数深度、允许 unbounded fan-in 的 AND/OR/NOT 电路；$\mathrm{TC}^0$ 在此基础上允许 threshold gate，也就是一次判断很多输入里有多少个 1 超过阈值。Parity 不在 $\mathrm{AC}^0$ 但在 $\mathrm{TC}^0$，所以 $\mathrm{AC}/\mathrm{TC}$ 的差别可以理解为“能不能在常数并行深度里做全局计数”。更一般地，$\mathrm{AC}^d$ 和 $\mathrm{TC}^d$ 把深度放宽到 $O(\log^d N)$，对应更长但仍高度并行的计算。

这类刻画为什么需要小心？因为 transformer 不是一个单一数学对象。理论上你必须规定 attention 是 softmax 还是 hard attention，数值是实数、fixed-point 还是 floating point，位置编码如何给出，模型是否随输入长度变化，width 和 depth 是否增长。已有结果已经显示这些选择会显著改变结论。Hao et al. (2022)、Merrill et al. (2022) 等早期工作常用 average hard attention，因为它像“选中最大分数位置再平均”，更容易离散化成电路；Merrill and Sabharwal (2023) 证明 finite/log precision transformer 与 threshold circuits 有紧密关系；Li et al. (2024) 和 London and Kanade (2025) 则把 pause/padding tokens 与 $\mathrm{AC}^0,\mathrm{TC}^0$ 的模拟联系起来；Merrill and Sabharwal (2025) 进一步给出 padded fully uniform average-hard-attention transformers 的精确表达力。问题是这些结论分散在不同 idealization 里：有的要求 log width，有的要求 fully uniform，有的只处理 hard attention，有的只处理 softmax。本文的定位正是把这些局部图景合成一张稳定的分类表。

Padded transformer 的直觉很简单：普通 transformer 只有输入位置本身可用，如果输入长度是 $N$，它只有 $N$ 个 token slots；padded transformer 允许在输入后面接 $P(N)=\mathrm{poly}(N)$ 个空白符号，这些空白位置可以作为 scratch space。这样，一个 padding position 可以存某个电路 gate 的值，attention 可以把信息从输入或已计算 gate 拉到对应位置。于是 padding 在这里扮演的角色很像电路里的“多项式多个门”。这也是为什么 padded 版本比 unpadded 版本更容易和完整的 $\mathrm{AC}^0/\mathrm{TC}^0$ 类对齐：没有 padding 时，模型只有线性多个位置，且每个位置的表示空间有限，很难容纳任意多项式大小电路的中间状态。

本文的形式化对象是 transformer family $\{T_N\}_{N\in\mathbb N}$。每个长度 $N$ 有一个 transformer $T_N$，但不能任意指定，否则像 non-uniform circuits 一样可以把不可计算语言的答案硬编码进每个长度的模型。作者采用 L-uniform：存在 logspace Turing machine，在输入 $1^N$ 时输出 $T_N$ 的参数，并且能在输入 $(1^N,\mathrm{bin}(n))$ 时输出第 $n$ 个位置的位置编码。这个定义有一个细节很关键：位置编码本身也可以由 logspace 程序预先计算，所以它可以给模型提供“位置 $n$ 的二进制表示”“某个除法/取模结果”这类 transformer 在常数精度下未必能自行算出的信息。

:::aside[Uniformity]
电路族若没有 uniformity 条件，每个输入长度的电路可以任意给出，从而能识别许多不可计算语言。L-uniform 要求用 $O(\log N)$ 空间从 $1^N$ 构造第 $N$ 个对象；FO-uniform 更强，等价于电路连接关系可由带 $+,\times,\le$ 的一阶逻辑描述，也等价于 DLOGTIME uniformity。本文中 constant-depth padded transformers 对齐到 L-uniform 电路类，而 looped transformers 对齐到 FO-uniform 电路类。
:::

另一个核心量是 volume。宽度 $D(N)$ 是每个 token residual stream 的维度，数值精度 $b(N)$ 是 fixed-point 标量的 bit 数，作者定义
$$
V(N)=D(N)\cdot b(N).
$$
它衡量每个位置每一层能携带多少 bit 级信息。至少要有 $V(N)=\Omega(\log N)$，模型才有足够容量区分 $N$ 个位置；否则连 injective positional encoding 都放不下。本文把这个条件称为 sufficient volume。注意这里已经暗示了主结论的形状：如果 $b=\Theta(1)$，那就必须有 $D=\Omega(\log N)$；如果 $b=\Theta(\log N)$，constant width 已经足够。问题是这两种“凑够 $\log N$ bit”的方式是否等价。答案是否定的：precision 比 width 更能改变表达力。

第一组主定理是 constant-depth 版本。用 $\mathrm{LPT}^d_{b,D}$ 表示 polynomially padded、循环次数 $\Theta(\log^d N)$、精度类型为 $b$、宽度类型为 $D$ 的 transformer，其中 $c,l,p$ 分别表示 $\Theta(1),\Theta(\log N),\mathrm{poly}(N)$。当 $d=0$ 时就是常数深度。作者证明，在 sufficient volume 且 width 至多多项式的条件下，对任意宽度函数 $D(N)$，
$$
\mathrm{L\text{-}uniform}\ \mathrm{LPT}^0_{c,D}
=
\mathrm{L\text{-}uniform}\ \mathrm{AC}^0,
$$
以及
$$
\mathrm{L\text{-}uniform}\ \mathrm{LPT}^0_{l,D}
=
\mathrm{L\text{-}uniform}\ \mathrm{TC}^0.
$$
这里第二个式子里的 $l$ 可以理解为增长精度的代表，因为 polynomial precision 不会超过 $\mathrm{TC}^0$，而 logarithmic precision 已经能达到 $\mathrm{TC}^0$。和 London and Kanade (2025) 的 prior best 相比，已有结果覆盖了 constant precision + log width 对 $\mathrm{AC}^0$，以及 log precision + log width 对 $\mathrm{TC}^0$；本文补上的关键缺口是 log precision + constant width 也能模拟 L-uniform $\mathrm{TC}^0$，同时 polynomial width/precision 也不会越过对应电路类。

这个 theorem 的证明有两个方向。下界方向，也就是 transformer 模拟 circuit，最有意思的是 constant-width log-precision 怎么够用。London and Kanade 的构造用 log-width positional encodings 存位置的二进制信息；本文改用二维的 unit-length positional encoding，把“第 $n$ 个位置应该 attend 到目标位置 $n_{\mathrm{tgt}}$”编码成近似单位向量，例如通过 $1/n_{\mathrm{tgt}}$ 和 $\sqrt{1-1/n_{\mathrm{tgt}}^2}$ 这样的坐标制造足够大的分离间隙。这样 attention score 在目标位置和非目标位置之间有可控 gap，fixed-point rounding 又把这个 gap 变成离散选择。这里我觉得最值得注意的是：width 没有消失，它被 precision 和 positional encoding 的几何分离替代了；模型不是在 residual stream 里显式存 $\log N$ 个 bit，而是在一个高精度数值里存足够区分目标位置的信息。

上界方向则说明，即便给常数深度 transformer 多项式宽度，常数精度仍只能落在 $\mathrm{AC}^0$；给多项式宽度和多项式精度，也只能落在 $\mathrm{TC}^0$。直觉是常数层 transformer 只做常数轮 attention 和 position-wise feed-forward。每一轮 attention 的加权求和、比较、rounding、MLP 都可以展开成常数深度电路；常数精度下这些运算只能用 $\mathrm{AC}^0$ 控制，增长精度下阈值型判断进入 $\mathrm{TC}^0$。所以 width 只是在每个位置提供更多寄存器，不会让常数轮并行计算突然获得更深的 sequential structure。

:::aside[Fixed-point rounding 的作用]
本文采用 fixed-point arithmetic：每次算术操作后都 round 到 $F_b$ 中最近的可表示值。这个设定一方面限制精度，另一方面也提供了离散化 gadget。只要构造中让目标 score 与非目标 score 的差距大于 rounding error，rounding 就能把小的实数差距变成稳定的“选中/未选中”判断。作者也明确提醒，这些 upper bound 未必自动转移到 floating-point transformer。
:::

第二组主定理处理 looped transformers。Looping 指重复使用同一个 transformer block 多次；这和单纯增加 depth 不完全一样，因为 looped 模型不增加参数数量，只增加 inference-time computation。作者证明对任意 $d\ge 1$，
$$
\mathrm{L\text{-}uniform}\ \mathrm{LPT}^d_{c,l}
=
\mathrm{FO\text{-}uniform}\ \mathrm{AC}^d,
$$
以及
$$
\mathrm{L\text{-}uniform}\ \mathrm{LPT}^d_{l,c}
=
\mathrm{FO\text{-}uniform}\ \mathrm{TC}^d.
$$
这和 constant-depth 的 L-uniform 结论有一个微妙差别：looping 后右侧变成 FO-uniform，而不是 L-uniform。原因是一个“uniformity collapse”：如果单次 transformer update 可以由 L-uniform 常数深度电路模拟，那么重复 $\Theta(\log^d N)$ 次以后，可以把这些层级展开成一个 FO-uniform $\mathrm{AC}^d/\mathrm{TC}^d$ circuit。对于 $d\ge 1$，FO-uniform $\mathrm{AC}^d$ 本身已经能实现 logspace-uniform construction 中需要的 bookkeeping，因此外层 L-uniformity 不会继续提高右侧电路族的 uniformity 复杂度。

增长精度的 lower bound 可以借用 Merrill and Sabharwal (2025) 对 fully uniform padded AHATs 的构造，再通过本文的 softmax 模拟 hard attention 引理转到 SMAT。常数精度的 lower bound 更需要说明：作者用 reduction framework，把问题转成证明 looped constant-precision padded transformers 能计算 L reductions，并能评估 $\mathrm{AC}^d$ circuit evaluation 这个 complete problem。技术路线是先用 $\Theta(\log N)$ 次循环做 graph reachability，从而得到 $\mathrm{NL}$ 能力，进而覆盖 L reductions；再让 transformer 在 padding positions 中存放电路 encoding 和 gate values，按层推进 gate evaluation。每一轮找出输入已经可用的 gates，计算其值并传播，经过 $O(L+\log N)$ 次循环后输出 gate 被算出。这里 padding 的作用非常具体：它不是“多一些 pause tokens”这么抽象，而是给每个 gate、每条连接、每个中间标志一个可寻址的位置。

Attention 类型在本文里反而不是主角。作者先证明 logarithmic-precision L-uniform AHAT 可以被 logarithmic-precision L-uniform SMAT 精确模拟：softmax 的温度 $\tau(N)$ 被当成同样由 uniform machine 生成的模型参数，选择足够小的温度，就能在 fixed-point 语义下精确复现 average hard attention 的 argmax averaging。这个引理把 hard-attention 文献里的 lower bound 搬到 softmax 版本。反过来，upper bound 对两种 attention 都成立，所以在 sufficient-volume padded regime 下，softmax vs. hard attention 的表达力差异被抹平了。这一点和 unpadded 或低容量 transformer 的脆弱结论形成对照。

本文给出的结构判断可以压成一句话：在 polynomial padding 和 $V(N)=\Omega(\log N)$ 下，表达力主要由“有没有增长精度”和“有多少 sequential rounds”决定。增长精度把模型从 $\mathrm{AC}$ 推到 $\mathrm{TC}$，因为它能稳定表达 threshold/counting；looping 把常数深度推到 $\log^d N$ 深度，因为它引入顺序迭代；width 只要够索引位置就不再有质变，超过 $\Theta(\log N)$ 的精度也不再带来更高电路类。这也解释了论文标题里的 which matter：depth/precision matter，width/attention type mostly don't，前提是 padded 且 volume sufficient。

我读这篇时最意外的是它对 positional encoding 的态度。很多 transformer expressivity 讨论把 PE 当成技术细节，但这里 PE 几乎是 uniform computation 的接口：constant-precision transformer 自己不能算 division/modulo，却可以从 L-uniform PE 中读到这些位置相关信息；constant-width transformer 不能存二进制位置，却能通过高精度二维向量实现可寻址 attention。换句话说，本文的“robustness”不是说细节无关，而是说在明确的 uniform PE 和 fixed-point arithmetic 规则下，这些细节会被吸收到 $\mathrm{AC}/\mathrm{TC}$ 的边界里。

最后留下的不是“transformer 是否等价于电路”这种大问题，而是一个更窄也更硬的问题：insufficient volume，即 $V(N)=o(\log N)$，应该对应什么自然复杂性类？已有 unpadded fully uniform constant-precision AHAT/SMAT 可落到带线性序的 two-variable first-order logic 片段 $\mathrm{PFO}^2[<]$，但 padding 和更弱 uniformity 是否会把它推出这个片段仍不清楚。标准 $\mathrm{AC}^0/\mathrm{TC}^0$ 在这里太粗，因为模型连区分所有位置的空间都不够；真正要找的可能是某种 sub-$\mathrm{AC}^0$ 的、同时反映位置可寻址性和空间布局的电路/逻辑类。

## Energy-Efficient Aggregation and Minimum-Degree Spanning Trees in Radio Networks [arXiv:2605.30546](https://arxiv.org/abs/2605.30546)

这篇论文研究的是同步 multi-hop radio network 里的 aggregation：给定一张通信图 $G=(V,E)$，每个节点一开始持有一个值，目标是在某个节点处算出所有值的全局聚合，例如 sum、count、min 或 max。作者考虑的是无 collision detection 的 RADIO-CONGEST 模型：每一轮节点可以发送、监听或睡眠；一个监听节点只有在恰好一个邻居发送时才能收到消息，若多个邻居同时发送则什么也收不到，并且它无法区分“没人发送”和“发生碰撞”。主要结果是一个随机分布式算法，在 $O(n\mathrm{polylog}\,n)$ 轮内构造并执行 aggregation schedule，且每个节点醒着的轮数至多 $O(\Delta^\ast\mathrm{polylog}\,n)$，其中
$$
\Delta^\ast=\min\{\Delta(T):T\text{ 是 }G\text{ 的生成树}\}.
$$
也就是说，能量复杂度几乎只差 polylog 因子地匹配最小最大度生成树的最大度。

同一个算法还会产出一棵生成树 $T$，满足 $\Delta(T)=O(\Delta^\ast\log n)$，并且每条树边的两个端点都知道这条边属于树。这个附带结果很重要，因为在 radio network 中，“存在一棵好树”和“节点们以低能耗知道并能使用这棵树”不是一回事。论文的核心贡献正是在这个缝隙里：它不仅证明低能耗 aggregation schedule 存在，而且给出在 RADIO-CONGEST 里构造和执行该 schedule 的方法。

先把模型的动机说清楚。Radio network 是理论分布式算法里用来抽象无线通信的经典模型，最早可以追溯到 Chlamtac 和 Kutten (1985, 1987)。和 CONGEST/LOCAL 模型不同，radio 节点不是每条边上有独立信道；一个节点向外发送时，所有邻居都处在同一个广播介质里。如果两个邻居同时对同一个节点发送，就会碰撞。无 collision detection 的设定更苛刻：监听者只知道自己没收到消息，不知道原因。这抓住了无线网络里一个很基础的困难：局部并发不是免费的，协调谁在什么时候说话本身就是算法问题。

这篇论文关心的复杂度不是传统的“总轮数”一个指标，而是同时看 round complexity 和 energy complexity。Round complexity 是算法运行多少同步轮；energy complexity 是最坏节点醒着多少轮，醒着包括发送和监听。后者在传感器网络里很自然：设备可以睡眠，睡眠时几乎不耗电；但只要打开无线电，无论有没有听到消息都会消耗能量。尤其是 idle listening，即“开着收音机但没有收到有用消息”，在实际系统里是主要能量损失来源之一。因此一个算法即使轮数很长，只要节点大部分时间睡着，也可能是能量高效的。

Aggregation 是这个语境下最基本的任务之一。形式上，论文处理 decomposable aggregation function：存在一个结合运算 $\diamond$，使得对任意不交多重集 $X,Y$，
$$
f(X\uplus Y)=f(X)\diamond f(Y).
$$
这覆盖 sum、count、min、max 等常见操作。结合性让算法可以把全局任务拆成子任务：先算若干局部 aggregate，再把它们合并。最朴素的方式是选一棵以 $r$ 为根的生成树，然后做 convergecast：每个节点等孩子的 partial aggregate 到齐后，与自己的值合并，再发给父亲。

这时 $\Delta^\ast$ 自然出现。若沿一棵生成树 $T$ 做 aggregation，一个度数为 $\deg_T(v)$ 的节点至少要和许多树邻居交互；最大度越高，某个节点就越可能成为能量瓶颈。若事先有一棵最大度 $\Delta^\ast$ 的生成树，那么可以用 $O(n)$ 轮的后序 schedule 做 aggregation，并让每个节点只醒 $O(\Delta^\ast)$ 轮。更强的是，论文观察到 $\Delta^\ast$ 不只是树算法的分析产物，而是任意 aggregation schedule 的 universal lower bound：对任意 schedule，都存在某个节点至少醒 $\Delta^\ast$ 轮。

这个 lower bound 的证明很短，但定位很准。看一次 schedule 执行中所有成功传输，若 $u$ 的信息成功传给 $v$，就在有向图里加边 $u\to v$。最后某个节点 $r$ 得到全局 aggregate，所以每个节点的初值都必须能影响 $r$，也就是每个节点在这张有向图里都能到达 $r$。因此里面含有一棵指向 $r$ 的有向生成树。把方向忘掉后，这是一棵 $G$ 的生成树；其中最大度至少是 $\Delta^\ast$。最大度节点如果不是根，要听所有孩子并至少发一次给父亲；如果是根，要听所有孩子。无论哪种，它都至少醒 $\Delta^\ast$ 轮。

:::aside[为什么 schedule 被要求与输入值无关]
论文中的 aggregation schedule 预先指定每轮每个节点发送、监听或睡眠的模式，且这个模式不依赖具体输入值或具体聚合函数，只能在发送内容中用 $\diamond$ 合并已获得的信息。这个约束排除了用数值本身编码控制流的做法，也让 lower bound 可以只从“信息是否能影响最终结果”来推导。
:::

于是问题变成：既然 $\Delta^\ast$ 是正确的能量参数，能不能在 radio network 中以接近 $\Delta^\ast$ 的能量自动找到可执行 schedule？这里不能直接说“先算 MDST，再沿树聚合”。Minimum-degree spanning tree (MDST) 在集中式算法中已经不是平凡问题；它要求在所有生成树中最小化最大度，精确求解是 NP-hard。Fürer 和 Raghavachari (1994) 给出经典的多项式时间 additive-one 近似，即输出最大度至多 $\Delta^\ast+1$ 的生成树。分布式版本更晚才系统出现：Dinitz, Halldórsson, Izumi 和 Newport (2019) 给出 CONGEST 中的分布式 MDST 近似算法，[arXiv:1806.03365](https://arxiv.org/abs/1806.03365)，其中一个结果是 $O(\log n)$ 近似，轮复杂度约为 $\widetilde O(D+\sqrt n)$，另一个结果达到 $\widetilde O(\Delta^\ast+\log n)$ 最大度但轮复杂度有额外 polylog 因子。

Radio model 里的相关图构造有另一条线。BFS、broadcast、maximal matching 等任务已经有低能耗算法，例如 Censor-Hillel et al. (2018) 研究了 radio network 中的 broadcast，[arXiv:1705.07369](https://arxiv.org/abs/1705.07369)；Dani, Gupta, Hayes 和 Pettie (2021) 给出 maximal matching 的低能耗算法，[arXiv:2104.09096](https://arxiv.org/abs/2104.09096)。但这些结果不能直接替代本文要的对象。BFS 树可能有巨大最大度，沿它 aggregation 会让中心节点耗能很高；许多广播树构造只保证 child 知道 parent，不保证 parent 知道所有 child，而本文的树边需要两端都知道，才能作为后续 schedule 的可操作资源。

本文的两个定理可以这样精确表述。第一，aggregation：
$$
\textbf{Theorem 1.}\quad
\text{在 RADIO-CONGEST 模型中，存在随机算法以高概率构造一个 aggregation schedule，}
$$
$$
\text{其构造和执行均使用 }O(n\mathrm{polylog}\,n)\text{ 轮，能量为 }O(\Delta^\ast\mathrm{polylog}\,n).
$$
第二，MDST 近似：
$$
\textbf{Theorem 2.}\quad
\text{在 RADIO-CONGEST 模型中，存在随机算法以高概率构造生成树 }T,
$$
$$
\Delta(T)\in O(\Delta^\ast\log n),
\quad
\text{轮复杂度 }O(n\mathrm{polylog}\,n),
\quad
\text{能量 }O(\Delta^\ast\mathrm{polylog}\,n).
$$
这里 $O(n\mathrm{polylog}\,n)$ 轮也基本不可避免：即使在星图上，中心也必须和 $n-1$ 个叶子成功通信，所以任何完成 aggregation 或让树边两端都知道的算法都需要 $\Omega(n)$ 轮，甚至在无限消息大小的 RADIO-LOCAL 中也一样。

技术上，论文先把 Dinitz et al. 的 Borůvka-style MDST 框架改造成适合 radio network 的低能耗版本。Borůvka-style 的意思是：初始每个节点是一个 cluster；每一轮并行选择很多跨 cluster 边，把 cluster 合并；如果每轮能让 cluster 数减少常数比例，$O(\log n)$ 轮后就只剩一个 cluster，选过的边构成生成树。为了控制最大度，每轮不能让某个节点接太多新边。

作者把每轮合并写成一个 red-blue matching 问题。当前所有 cluster 随机染成 red 或 blue。目标是选跨色边，使每个 red cluster 最多连一条选边，而每个 blue 节点最多接 $d$ 条选边。这里 $d$ 是对 $\Delta^\ast$ 的猜测；若 $d\ge \Delta^\ast$，固定一棵最大度至多 $d$ 的最优树，再只看跨 cluster 且跨颜色的边，期望上仍有足够多可用边。因此只要每轮能找到接近 maximal 的这种 matching，就能合并常数比例的 clusters。

本文实际使用的是略微放松的对象：$d$-almost-maximal $(1,2d)$-component-node matching。它允许每个 blue 节点最多接 $2d$ 条边，但要求未匹配的 red cluster 周围的 blue 节点都已经至少接了 $d$ 条，反过来，未饱和的 blue 节点周围的 red cluster 都已经被匹配。这个从 $d$ 到 $2d$ 的 slack 是算法设计里很关键的宽度：它让 radio model 中的随机冲突和局部不同步有地方被吸收，同时仍足以保证 matching 大小达到任意可行 $(1,d)$ matching 的常数因子。论文证明若 $M$ 是 $d$-almost-maximal $(1,2d)$ matching，$M^\ast$ 是任意 $(1,d)$ matching，则
$$
|M|\ge \frac{|M^\ast|}{2}.
$$
再结合随机红蓝染色和最优树的跨 cluster 边，可以得到每轮以常数概率选到 $\Omega(|\mathcal V|)$ 条边，其中 $\mathcal V$ 是当前 cluster 集合。

:::aside[component-node matching]
给定一个 connected clustering $\mathcal V$，$(1,k)$-component-node matching 是一组 red cluster 到 blue 节点之间的原图边 $M$，满足每个 red cluster 至多 incident to 一条 $M$ 边，每个 blue 节点至多 incident to $k$ 条 $M$ 边。注意约束的一侧是 cluster，另一侧是原图节点；这正好对应“一个红 cluster 只用一条边挂到中心，但一个蓝节点可作为多个红 cluster 的星心连接点”。
:::

真正难的是怎么在 radio network 里低能耗地算这个 matching。普通图 matching 算法里，一轮随机试探往往只需要邻居间局部通信；但现在红色一侧是 cluster，不是单个节点。一个 cluster 要判断自己是否参与、是否有唯一候选、最后保留哪条边，都需要 cluster 内部协调。而 radio network 中做一次 cluster 内通信可能已经要 $O(n\mathrm{polylog}\,n)$ 轮。若逐轮模拟 Dani et al. 的 maximal matching 过程，外面再乘上很多随机试探轮，轮数会爆掉。

作者的处理是把许多 ordinary matching rounds 打包成一个 big-round。一个 big-round 内，节点只做跨 cluster 的随机试探，先不在 cluster 内同步；等这一批试探结束后，每个 red cluster 再做一次内部协调，只保留一条 incident matched edge，把多余的取消掉。这个“先允许 over-match，再 unmatch 修复”的思路是我读下来最关键的一步：它把昂贵的 cluster 内协调次数从 matching 试探轮数降到 polylog 级别，但没有把错误留给外层 Borůvka 框架。

这个 batching 也带来新的概率分析问题。取消阶段会制造依赖：某条边被试探成功，不代表它最后留下，因为同一个 red cluster 里可能有别的候选。论文把 red clusters 按一个 big-round 里期望产生的候选边数分成 large 和 small。Large cluster 很可能产生至少一条候选，因此很快会变 inactive；当剩下的活跃 red cluster 都 small 后，高度数 blue 节点在若干 big-round 内有足够概率获得留下来的匹配。这里需要二阶矩估计来处理候选边之间的非独立性。

第二个难点来自 blue 节点容量是 $2d$，不是 $1$。如果简单重复 maximal matching 过程 $d$ 次，能量和轮数都会多一个 $d$ 因子。论文按 $d$ 的大小分两种 regime。若 $d$ 只是 polylog，可以承受重复；若 $d$ 很大，作者提高 red-side sampling rate，但只提高到 $\Theta(d/\log n)$ 倍，而不是 $\Theta(d)$ 倍。这样每个 blue 节点在一个 big-round 中收到的候选数仍可用 Chernoff bound 控制在 $d$ 左右，不需要再设计 blue-side unmatching。这里 slack 再次发挥作用：目标容量是 $2d$，饱和阈值是 $d$，随机波动落在中间不会破坏不变量。

把这些组合起来，得到关键子程序：
$$
\textbf{Lemma 5.1.}\quad
\text{给定 clustering }\mathcal V,\text{ 可在 }O(n\mathrm{polylog}\,n)\text{ 轮、}
O(d\mathrm{polylog}\,n)\text{ 能量内}
$$
$$
\text{构造一个 }d\text{-almost-maximal }(1,2d)\text{-component-node matching}.
$$
然后 Decomposition$(d)$ 反复调用该子程序并 merge clusters。若 $d\ge \Delta^\ast$，cluster 数在 $O(\log n)$ 层后降到 $1$；由于每层每个节点最多新增 $2d$ 条树边，最终生成树最大度为 $O(d\log n)$。算法不知道 $\Delta^\ast$，于是对 $d=1,2,4,\ldots$ 做指数搜索，并用 loneliness testing 检查是否已经只剩一个 cluster；第一个成功的 $d$ 小于 $2\Delta^\ast$，于是得到 Theorem 2。

到这里还只是得到一棵近似 MDST。aggregation schedule 的构造更有意思，因为“沿树聚合”在 radio network 中并不自动可执行。若用后序遍历，节点们需要知道一个全局顺序；若按树层一层层聚合，树深可能是线性的，而且同层内 radio contention 可能很高。论文没有把最后那棵树当作唯一对象，而是回到构造树时产生的 hierarchy：每个高层 cluster 都是由上一层若干 cluster 通过一颗小星形合并而成，有一个 center child 和若干 red leaf children。这个 hierarchy 比裸生成树多了时间结构，也正是 schedule 可以递归构造的原因。

递归 schedule 的自然形态如下。对每个 cluster $C$，构造一个 schedule，使它先算出 $f(C)$，再把结果分发给 $C$ 内所有节点。若 $C$ 由中心 child $C_0$ 和 leaves $C_1,\ldots,C_s$ 合并而成，那么先让各个 leaf child 算出自己的 aggregate；再通过 matching 边把 leaf aggregates 传到 $C_0$；然后在 $C_0$ 内继续聚合；最后 down-cast 给整个 $C$。如果对大小为 $s$ 的 cluster 能保证 schedule 长度 $O(s\mathrm{polylog}\,n)$，root cluster 就给出全局 $O(n\mathrm{polylog}\,n)$ 轮。

问题在第一步：多个 leaf schedules 怎么安排？全顺序太慢；全并行会造成 radio contention；在每层都给每个 child 加随机 delay 又会让长度每层膨胀，$O(\log n)$ 层后多出不可接受的因子。论文为此引入 slot-based schedule。先把抽象 schedule 写成 slots，每个 slot 中可以放若干 transmission events，只要求任意按 slot 顺序执行都保持 aggregation 的偏序正确；之后再把每个 slot 展开成真实 radio rounds。这样，递归构造只需要控制每个 slot 的 congestion，即同一 slot 里有多少 events。

控制长度的办法是 large/small child 分流。对 cluster $C$，若某个 child 的 size 至少是 $C$ 的 $1/(\alpha\log n)$，它就是 large；large children 数量只有 $O(\log n)$，可以顺序执行。Small children 的 schedule 明显比 parent 短，于是可以放进一个公共 block 里，各自随机选择 delay。随机 delay 的额外长度只按最大小 child schedule 付费，而这个长度被选阈值压成 parent schedule 的小部分，所以不会在层级上滚雪球。

控制 congestion 的分析用一个递归定义的随机变量族 $\mathcal Z$。直觉上，一个 slot 的拥塞来自两种操作：要么是若干独立 child schedules 的贡献相加，要么是某个 child schedule 被随机 delay 后，当前 slot 随机命中它的某个本地 slot。于是论文定义 $\mathcal Z_0$ 为确定的 $0/1$ 变量；$\mathcal Z_d$ 中的变量可以由 $\mathcal Z_{d-1}$ 变量做 independent parallel sum，或随机选择其中一个得到，并始终要求期望至多 $1$。关键引理是：任意 $Z\in\mathcal Z$ 都以高概率为 $O(\log n)$。证明把 $Z$ 展开成叶子 $0/1$ 指示变量之和，并利用负相关变量的 Chernoff bound。

:::aside[slot 到真实 rounds]
若 slot-based schedule 有多项式长度、每个 slot 拥塞至多 $\kappa=O(\log n)$，论文把每个 slot 替换成 $R=\Theta(\kappa\log n)$ 个真实 rounds。每个 event 在这些 rounds 中以概率 $\Theta(1/\kappa)$ 参加。高概率下，每个 event 至少有一轮成为该 slot 的唯一参加者，因此成功执行；每个 event 只额外带来 $O(\log n)$ 次醒来。
:::

这个随机变量族的设计比表面上更精确。它不是为了套一个通用 concentration lemma 而人为包装，而是恰好对应 hierarchy schedule 中“顺序嵌入、随机 delay、并行叠加”三种操作。特别是 small children 的处理：固定 parent 的某个 slot $\sigma$，一个 small child 对 $\sigma$ 的贡献要么是 $0$，要么是它本地 schedule 的某个 slot 拥塞；随机 delay 正好把这变成“随机选择”。多个 small children 独立选择 delay，于是贡献相加。只要总期望被长度预算压到 $1$，就落回 $\mathcal Z$。

因此 Theorem 1 的证明不是“MDST 近似 + 一个常规 convergecast”的黑盒组合，而是把 MDST 算法的合并历史保留下来，作为 aggregation 的执行计划。这个区别很实质：最终生成树只告诉你哪些边被选中，层级分解还告诉你这些边是在什么规模的子问题中被选中、哪些 cluster 应该先完成、哪些可以随机错开并行。对 radio network 这样的碰撞模型来说，后者才是 schedule 级的信息。

我对这篇论文的主要判断是：$\Delta^\ast$ 作为能量参数本身很自然，但作者真正做成的是“参数正确 + 构造可执行”。尤其是 matching 部分和 aggregation 部分各自用了一次 slack：matching 用 $(1,2d)$ 与 $d$-almost-maximal 给随机局部试探留余量；schedule 用 large/small child 与 slot congestion 给递归并行留余量。两处 slack 都不是松分析，而是为了在无 collision detection 的广播介质中避免每一步都做昂贵全局协调。这个视角也解释了为什么结论里有 polylog 因子：它们大多是在把结构性存在证明转成低能耗 radio schedule 时，为碰撞隔离、随机延迟和局部同步支付的价格。

## 未入选但值得关注

### Aspects of Coherence in Dependence Logic [arXiv:2605.31269](https://arxiv.org/abs/2605.31269)
这篇论文研究 dependence logic 中 coherence 与 first-order rewritability 的关系，并给出 propositional dependence logic coherence 的 exponential hierarchy 完备性结果。未入选主要是因为主题更偏 finite model theory / logic，今天版面优先给算法与复杂性方向中更容易展开技术脉络的论文。

### Tree Containment Parameterized by Scanwidth [arXiv:2605.31071](https://arxiv.org/abs/2605.31071)
论文给出 scanwidth 参数下的 $O(4^{k+k\log k}n+nm^2)$ 算法，并配套 ETH 意义下的匹配下界，结果相当扎实。没有入选是因为 phylogenetic network 的背景较窄；和 Binary Closest String 相比，它对一般 TCS 读者需要更多领域铺垫。

### Diffusion-Robust Optimization over Graphs [arXiv:2605.30853](https://arxiv.org/abs/2605.30853)
这篇工作提出 diffusive uncertainty model，分析 shortest path 的 tractability transition，并把 TSP 的 adversarial 子问题化成显式公式。题目有新意，但从筛选信息看更像 robust optimization 框架论文，具体 theorem 形态不如入选论文适合在日报里深讲。

### How Many Slopes Does Polynomial Area Cost? [arXiv:2605.31098](https://arxiv.org/abs/2605.31098)
论文讨论 planar drawing 中 slopes、bends 和 area 的 trade-off，对 computational geometry 读者很有吸引力。未入选是因为摘要层面主要给系统性构造的高层描述，缺少足够具体的 bound 来支撑一篇深度解读。

### Incremental BPE Tokenization [arXiv:2605.30813](https://arxiv.org/abs/2605.30813)
这篇论文给出最坏情形 $O(n\log^2 t)$ 的 incremental BPE tokenization，并支持 eager streaming output，算法保证清楚且实用。它更靠近 LLM tokenizer 工程和系统实现，理论优先级低于今天入选的四篇。
