## 编辑按语

今天的 3 篇论文覆盖了三个不同方向：submodular maximization 的前处理与 containment guarantee、Online Orthogonal Vectors 的静态数据结构和 fine-grained lower bounds、以及 packed string 模型下 shortest unique/absent substring 的更快算法。它们之间没有直接技术联系，不需要强行归成一个主题。

如果从读法上看，三篇都在重新审视一个经典问题在更细模型里的边界：第一篇区分“保留好解所在 universe”和“直接优化”的差别；第二篇把 OnlineOV 放在确定性检索结构与 non-uniform lower bound 的交界处；第三篇说明 suffix-tree 线性时间在 packed word RAM 里并不是终点。

## Submodular Ground-Set Pruning: Monotone Tightness and a Non-Monotone Separation [arXiv:2605.04428](https://arxiv.org/abs/2605.04428)

这篇论文研究的是 submodular maximization 的 ground-set pruning 问题：给定一个很大的元素集合 $N$ 和一个非负 submodular 函数 $f:2^N\to \mathbb{R}_{\ge 0}$，先把 $N$ 缩成一个较小的核心集合 $P$，再要求 $P$ 里对每个下游预算 $k'\le k$ 都仍然含有一个接近最优的可行解。Alan Kuhnle 考虑的核心模型叫 containment pruning；主要结果是：monotone 情形下，无论 pruning budget $p\ge k$ 给多大，最优 containment factor 都正好是 $1-1/e$；non-monotone 情形下，可以用 $O(k/\varepsilon)$ 个元素做到 $1/2-\varepsilon$ containment，这个比例反而超过了已知 non-monotone maximization 算法的 $0.401$ 和 oracle hardness 的 $0.478$ 门槛。

形式化地，论文的基本问题是：给定 $N,f,k$ 和 $p\ge k$，输出 $P\subseteq N$，$|P|\le p$，使得
$$
\max_{T\subseteq P,\ |T|\le k'} f(T)\ge \alpha\cdot \mathrm{OPT}_{k'}
\quad \forall k'\le k,
$$
其中
$$
\mathrm{OPT}_{k'}=\max_{|T|\le k'} f(T).
$$
这里 $\alpha$ 叫 containment factor。注意这个定义没有要求 pruning algorithm 自己找出最终解；它只需要保留一个小 universe，让之后的 greedy、IP solver、local search 或其他更贵的 optimizer 在 $P$ 上有东西可找。这个区别是全文的中心：containment 不是 optimization 的弱写法，而是一个不同的前处理任务。

为什么这个问题值得单独研究？很多 subset selection 任务都长这样：从一堆传感器里选少数几个，从社交网络里选 seed users，从海量 passage 里选 LLM context，从特征库里选 predictive features，或者从图顶点里选一个小 cut witness。目标函数往往有 diminishing returns：已经覆盖过的信息，再加一个相似元素的边际收益会变小。这正是 submodular function 捕捉的结构。直觉上，$f$ 是 submodular 的意思是“越往大的集合里加同一个元素，它带来的新增价值越小”；形式化地，对 $A\subseteq B\subseteq N$ 和 $x\notin B$，
$$
f(A\cup\{x\})-f(A)\ge f(B\cup\{x\})-f(B).
$$
如果还满足 $A\subseteq B\Rightarrow f(A)\le f(B)$，就叫 monotone；coverage、facility location、influence maximization 的许多版本都落在这里。non-monotone 则允许加元素伤害目标值，例如 MaxCut 的集合函数 $f(S)=|\{(u,v)\in E:u\in S,v\notin S\}|$：往 $S$ 里再加一个顶点，可能把原来跨 cut 的边变成内部边，值会下降。

Submodular maximization 的经典脉络很清楚。对 monotone submodular function 加 cardinality constraint，Nemhauser 和 Wolsey (1978) 证明 greedy 给出 tight 的 $1-1/e$ 近似；后来这条界扩展到 matroid、knapsack、curvature 等许多约束版本。昨天日报里讨论的 GS-Poisson [2026-05-06 日报](#2026-05-06) 也是这条线的一部分：它在 monotone submodular maximization under matroid constraint 下，以 Poisson-process swap 动态匹配 tight 的 $1-1/e$ 保证。non-monotone 这边更难，因为边际收益可以变负，经典 greedy 分析不再直接工作。Feige, Mirrokni 和 Vondrák (2011) 研究 unconstrained non-monotone submodular maximization，Vondrák (2013) 的 symmetry gap 成为许多 hardness 的标准工具；Buchbinder 和 Feldman (2024) 给出 down-closed constraints 下 $0.401$ 近似 [arXiv:2311.01129](https://arxiv.org/abs/2311.01129)，Qi (2022) 给出 $0.478$ hardness [arXiv:2205.15874](https://arxiv.org/abs/2205.15874)，改进了 Gharan 和 Vondrák (2011) 的 $0.491$ hardness [arXiv:1007.1632](https://arxiv.org/abs/1007.1632)。

Pruning 的研究脉络比 optimization 年轻得多。实践上，大家早就在做 candidate reduction：先筛掉明显差的点、近重复的 passage、低分 feature，再把小实例交给更贵的求解器。问题是这类 heuristic 通常只能说“跑得快、实验好”，不能保证 pruned set 里仍然有接近最优的解。Nath 和 Kuhnle (2025) 把 containment pruning 明确提出并给了 QuickPrune，对 monotone weakly submodular knapsack 给出 $\alpha\ge 1/24-\varepsilon$ [arXiv:2410.17945](https://arxiv.org/abs/2410.17945)。本文接着问两个更基础的问题：在最标准的 monotone cardinality setting 里，多给 pruning budget 是否能超过 $1-1/e$？在 non-monotone setting 里，是否存在真正的 constant-factor budget-range containment？

:::aside[Weakly DR 与 curvature]
论文的 monotone lower bound 实际覆盖 $\gamma$-weakly diminishing returns functions：它把完全 submodularity 的 diminishing-return 不等式放松一个参数 $\gamma\in(0,1]$，对应 tight barrier $1-e^{-\gamma}$。Total curvature $\kappa$ 衡量元素在已有大集合里保留多少边际价值，$\kappa=0$ 接近 modular，$\kappa=1$ 是一般最坏情形。Sviridenko, Vondrák 和 Ward (2017) 证明 bounded-curvature monotone maximization 的 tight ratio 是 $1-\kappa/e$ [arXiv:1311.4728](https://arxiv.org/abs/1311.4728)。
:::

Monotone 部分的主定理可以精确写成：
$$
\textbf{Theorem 3.1.}\quad
\forall \gamma\in(0,1],\ \forall \varepsilon>0,\ \forall \text{ polynomial } C(\cdot),
$$
存在 $k_0=k_0(\gamma,\varepsilon,C)$，使得对所有 $k\ge k_0$ 和所有 $p\ge k$，任何至多做 $C(|N|)$ 次 value queries、输出 $|P|\le p$ 的随机算法，都不能对所有 monotone $\gamma$-weakly DR 函数保证
$$
\mathbb{E}\max_{S\subseteq P,\ |S|\le k} f(S) >
(1-e^{-\gamma}+\varepsilon)\cdot \mathrm{OPT}_k
$$
在 $|N|=\lceil 60p/\varepsilon\rceil$ 的 ground set 上成立。取 $\gamma=1$ 就得到一般 monotone submodular 情形的 upper bound $\alpha\le 1-1/e$。另一方面，标准 greedy 的 prefix property 已经给出 simultaneous $1-1/e$ containment：运行 $k$ 步 greedy 得到 $P$，则任意 $k'\le k$ 的前 $k'$ 个 greedy 元素都是一个 $1-1/e$ 近似。因此 monotone cardinality pruning 被完全钉住：$p=k$ 已经最优，$p=10k$、$p=k^2$ 甚至更大都不能改善 containment factor。

这点初看不太显然。一个自然但错误的想法是：optimization 要输出一个好解，pruning 只要保留好解，多给一些位置应该能更容易超过 $1-1/e$。论文的 lower bound 解释了为什么这个希望在 value-oracle worst case 里落空。作者使用 Harshaw, Feldman, Ward 和 Karbasi (2019) 的 planted-optimum construction [arXiv:1904.09354](https://arxiv.org/abs/1904.09354)：随机藏一个大小为 $k$ 的最优集合 $\widetilde T$，只要查询集合和 $\widetilde T$ 的重叠没有超过阈值，oracle 返回值就和一个没有 planted optimum 的 null instance 一样。算法看不到足够重叠，就无法知道哪些元素属于 $\widetilde T$。

本文的新动作是把 hard instance 的 ground set 规模设为 $|N|\propto p$，也就是随 pruning budget 一起放大。这样算法最后输出 $|P|\le p$ 后，它和随机 planted optimum 的期望重叠是
$$
\mathbb{E}[|P\cap \widetilde T|]\le \frac{pk}{|N|}=O(\varepsilon k),
$$
这里 $p$ 在分子和分母里抵消了。也就是说，多给算法 $p$ 个保留位置，同时也把隐藏空间放大到同一比例；在 oracle transcript 看起来像 null instance 的条件下，算法的输出对 $\widetilde T$ 仍然近似盲选。接着用两类 hypergeometric tail bounds 控制查询阶段和输出阶段的重叠，保证 $P$ 中任意 $k$ 元子集都看不见足够多 planted elements，于是值被 null instance 的 $1-e^{-\gamma}+O(\varepsilon)$ 上界卡住。

这个 lower bound 最值得注意的是它没有走 submodular hardness 里常见的 symmetry gap 路线。Symmetry gap 的 hard instance 通常对很多均衡子集都给近似好值，这对 pruning 反而不合适：如果好元素在很多 orbit-balanced slice 里都存在，随机保留 $O(k)$ 个元素可能已经能 contain 一个不错解。Planted-set construction 则把“好解”真的藏成一个具体集合，containment algorithm 必须保留足够多 planted elements 才能赢。这正好匹配 pruning 的困难点。

bounded curvature 的 corollary 也很干净：
$$
\textbf{Corollary 3.3.}\quad
\forall \kappa\in(0,1],\ \forall p\ge k,\ \forall \varepsilon>0,
$$
没有 polynomial-query pruning algorithm 能对所有 total curvature 至多 $\kappa$ 的 monotone submodular functions 保证
$$
\alpha>1-\kappa/e+\varepsilon.
$$
由于 Sviridenko, Vondrák 和 Ward (2017) 的算法达到 $1-\kappa/e$，这说明低 curvature 能改善 optimization，也能改善 greedy containment，但不能让 pruning 在同一 curvature class 内进一步越过对应 tight barrier。论文还在 appendix 给出一个 near-linear threshold-greedy 版本：用 $O(k/\varepsilon)$ 个元素和 $\widetilde O(n/\varepsilon^2)$ 次 value queries 同时达到 $1-1/e-\varepsilon$，相比普通 greedy 的 $O(nk)$ query cost 更适合大 ground set。

non-monotone 部分是这篇论文真正把 containment 和 optimization 分开的地方。主定理是：
$$
\textbf{Theorem 4.1.}\quad
\text{对任意非负 submodular } f \text{ 和 } k\le n,\ \forall \varepsilon>0,
$$
存在 $G\subseteq N$，$|G|=O(k/\varepsilon)$，使得对所有 $k'\le k$，
$$
\max_{T\subseteq G,\ |T|\le k'} f(T)
\ge (1/2-\varepsilon)\cdot \mathrm{OPT}_{k'}.
$$
而且 $G$ 可由 $O(nk/\varepsilon)$ 次 oracle queries 构造。这个 $1/2-\varepsilon$ 是 containment guarantee，不是说算法自己能在原 ground set 上输出 $1/2$ 近似解。正因为只要“保留一个可找的好解”，它绕开了 non-monotone maximization 的 $0.401$ best-known algorithm 和 $0.478$ hardness；这就是标题里 separation 的含义。

算法叫 sequential disjoint greedy。取 $\ell=\lceil 1/\varepsilon\rceil$，依次运行 $\ell$ 次 greedy，每次从剩余 ground set 中选 $k$ 个元素，得到两两 disjoint 的 $G_1,\ldots,G_\ell$，最后输出
$$
G=\bigcup_{i=1}^{\ell}G_i.
$$
证明固定任意预算 $k'\le k$ 和一个最优解 $O$。把 $O$ 分成已经被保留的部分 $\dot O=O\cap G$ 和漏掉的部分 $\widehat O=O\setminus G$，记 $b=|\widehat O|$。对每个 greedy run $i$，取它的前 $b$ 个元素 $A_i$。因为这些 $A_i$ 彼此 disjoint，submodularity 控制了它们对 $O$ 的总“破坏”：
$$
\sum_i \bigl(f(O)-f(O\cup A_i)\bigr)\le f(O).
$$
所以存在某个 $i^\ast$，使得
$$
f(O\cup A_{i^\ast})\ge (1-1/\ell)f(O).
$$

接下来是证明里最关键也最朴素的一步。漏掉的元素 $\widehat O$ 没有出现在任何 $G_i$ 里，因此在第 $i^\ast$ 次 greedy run 的每一步，它们都仍然是候选元素。Greedy 选择的 $A=A_{i^\ast}$ 中第 $j$ 个元素，其 marginal gain 至少不小于某个漏掉元素的 marginal gain。把这条逐步 domination telescope 起来，得到
$$
2f(A)-f(\emptyset)\ge f(\widehat O\cup A).
$$
这个式子就是 $1/2$ 的来源之一：non-monotone 下 marginal chain 只能给出这种二倍关系，而不像 monotone 分析里能滚出 $1-e^{-1}$。

最后用 submodularity 的 lattice inequality 把 $O\cup A$ 拆成两个真正位于 pruned set 中的候选解：$A$ 和 $\dot O\cup A$。二者大小都不超过 $k'$，且都在 $G$ 里。由前面的 averaging 和 domination，
$$
\max\{f(A), f(\dot O\cup A)\}
\ge \frac{1-1/\ell}{2}\cdot f(O).
$$
因为 $k'$ 是任意的，同一个 $G$ 同时服务所有预算。这里很值得注意的是，证明从未要求 downstream optimizer 知道哪个 $i^\ast$、哪个 $A$ 或哪个 $\dot O$；containment theorem 只证明这些 witness 存在于 $G$ 中。

:::aside[Containment 不是 bicriteria optimization]
Bicriteria submodular maximization 通常允许算法输出超过预算的集合，例如 Feldman 和 Kuhnle (2025) 输出约 $1.44k$ 个元素来包含一个 $(1/2-\varepsilon)$ 级别的 witness [arXiv:2507.10248](https://arxiv.org/abs/2507.10248)。本文的 cardinality containment 最终 witness 仍满足 $|T|\le k'$；超预算的是 pruned universe $G$，不是 downstream solution。
:::

论文还给了一个结构不同的 one-pass window proof。算法在每一步维护 top-$\omega k$ marginal elements 的 window，从窗口里随机选一个加入当前构造，同时把所有出现过的 window 累积进 pruned set $P_k$。定理为
$$
\textbf{Theorem 4.3.}\quad
\mathbb{E}\max_{T\subseteq P_k,\ |T|\le k'} f(T)
\ge \frac{e^{-1/\omega}}{2}\cdot \mathrm{OPT}_{k'}
\quad \forall k'\le k,
$$
渐近意义下 $|P_k|\le k+\omega k^2$，query complexity 为 $O(nk)$。$\omega\to\infty$ 时同样趋近 $1/2$。这个算法保留的元素更多，但它说明 $1/2$ 不是 sequential disjoint greedy 的偶然产物：一条靠多次 disjoint passes，一条靠单次 wide exploration，最后都碰到同一个二分因子。

Knapsack extension 则把“元素个数预算”换成“成本预算”。每个元素 $u$ 有成本 $c_u$，可行解要求总成本不超过 $B$。自然的 greedy 替代物是 density-greedy：每步选 marginal value per cost 最大的元素。论文证明，对 $\ell=\lceil 1/\varepsilon\rceil$ 次 sequential disjoint density-greedy，输出 $P$ 满足 $c(P)\le 3\ell B=O(B/\varepsilon)$，且对每个 $B'\le B$ 存在 $Q\subseteq P$，
$$
c(Q)\le B',\qquad
f(Q)\ge (1/2-\varepsilon)\mathrm{OPT}_{B'}-\Delta_{B'}.
$$
这里 $\Delta_{B'}$ 是 boundary-fragment loss：density-greedy prefix 可能刚好跨过预算边界，最后一个元素不能被完整放入 $B'$，证明需要把它概念性地 split 成可放入和不可放入的 fragment。这个边界项不是纯粹的证明瑕疵，因为跨界元素本身可能比 $B'$ 还贵，无法简单用“最佳可行 singleton”吸收；在 small-item 条件下，它可以被吞掉，恢复干净的 $1/2-\varepsilon$。

实验部分不是主贡献，但它解释了为什么 containment 是合适的建模语言。MaxCut 是标准 non-monotone submodular objective，作者把一个 $G(300,4500)$ 实例 prune 到 $p=50$ 个顶点后，让 exact IP solver 在 reduced instance 上运行，从 $120$ 秒降到 $0.19$ 秒，约 $620\times$ speedup，同时保留 $\alpha=1.000$。LLM context selection 实验里，HotpotQA 中 $38.7\%$ 的问题表现出 non-monotone context effects：加 passage 会伤害答案质量；在 MuSiQue pilot 上，用非单调 proxy $\widehat f(S)=\mathrm{FL}(S)-\theta(|S|)$ 做 relevance-gated pruning，使 $k=10$ 时 gold-passage recall 提高 $9.2$ 个百分点。这些实验当然只是 proof-of-concept，但它们和 theorem 的关系比较紧：pruning 的价值不是替代 optimizer，而是把 full instance 缩成 downstream solver 能承受的规模。

我读这篇论文时最意外的是 monotone 与 non-monotone 的对照方向。通常我们会觉得 monotone 是“容易侧”，non-monotone 是“困难侧”；但在 containment pruning 里，monotone 先被经典 $1-1/e$ barrier 完全锁死，而 non-monotone 反而出现了相对 optimization 的分离。原因不是 non-monotone 函数突然更好优化，而是 containment 的目标改变了：它允许算法保留多条互相 disjoint 的搜索轨迹，把“哪条轨迹真的有用”留给存在性证明和 downstream optimizer。这个视角也解释了剩下的核心 gap：non-monotone containment 的真实阈值现在只知道在 $[1/2,\,1-1/e]$ 之间；如果未来有人突破 $1/2$，大概率也不会是把现有 non-monotone optimizer 黑盒塞进 pruning，而是要继续利用“保留 universe 比找出 solution 更弱”这一点。

## Online Orthogonal Vectors Revisited [arXiv:2605.04798](https://arxiv.org/abs/2605.04798)

这篇论文研究的是 Online Orthogonal Vectors（OnlineOV）问题：预处理阶段给定 $n$ 个布尔向量 $x_1,\ldots,x_n \in \{0,1\}^d$，数据结构用 $S$ bits 存下关于它们的信息；查询阶段给定一个新向量 $q\in\{0,1\}^d$，要在时间 $T$ 内判断是否存在某个输入向量 $x_i$ 与 $q$ 正交，也就是 $\langle x_i,q\rangle=0$。作者给出新的确定性数据结构：当 $d=c\log n$ 时，空间和预处理时间可以做到 $O(n^{1+\delta})$，查询时间
$$
T = n^{1-\Omega(1/(c\log c))}
$$
其中 $\delta \ge \Omega(\log c/c)$；这匹配了 Chan (2017) 随机算法的量级，但算法本身完全确定性。更一般地，对任意维度 $d$ 和 $\varepsilon<1/2$，他们得到
$$
T \le n^{1-\varepsilon}d,\qquad
S \le 2^{O(\varepsilon d\log(1/\varepsilon))}n^{1-\varepsilon}d,\qquad
T_p \le 2^{O(\varepsilon d\log(1/\varepsilon))}n^{1+3\varepsilon}d.
$$
这在 $d=n^{o(1)}$ 的中等维度区间给出自 Charikar, Indyk, and Panigrahy (2002) 以来的首个确定性改进。

同一篇论文还证明了强下界：在 Non-Uniform SETH 下，即使预处理计算能力不受限制，只要想要 $n^{1-\gamma}$ 的次线性查询时间，就不能用任意固定多项式空间解决某些 OnlineOV 变体。形式上，对任意常数 $c,\alpha>0$ 和 $\delta,\gamma\in(0,1)$，存在常数稀疏度 $w$，使得没有 batch 数据结构能在空间 $n^c$、每个查询时间 $n^{1-\gamma}$ 内回答 $n^\alpha$ 个 $w$-Sparse-OnlineOV$_{n,d}$ 查询，其中 $d=n^\delta$。这个下界还传递到 Partial Match、Subset Query、Orthogonal Range Search、DNF Evaluation 和 Approximate Nearest Neighbor 等经典检索问题。

先把这个问题放回更基础的图景里。普通 Orthogonal Vectors（OV）问的是：给两组 $n$ 个布尔向量 $A,B\subseteq\{0,1\}^d$，是否存在 $a\in A,b\in B$ 满足 $\langle a,b\rangle=0$。如果把布尔向量看成集合的特征向量，正交就是两个集合不相交。这个问题之所以在 fine-grained complexity 里重要，是因为 Williams (2005) 把 SETH 与 OV 联系起来：若对 $d=c\log n$ 的 OV 有真正次二次时间算法，就会推翻 Orthogonal Vectors Conjecture，而 OVC 已经成为给大量多项式时间问题证明精细下界的标准中间假设。这个脉络和最近几篇日报里基于 SETH/OVH 建立 tight lower bounds 的工作相通，例如 Fréchet distance 的高维网格图下界见 [2026-04-28 日报](#2026-04-28)，bin packing 和 scheduling 的 pseudopolynomial 下界见 [2026-03-16 日报](#2026-03-16)。

OnlineOV 是把 OV 的一侧固定下来，预先建索引，之后不断回答另一侧的查询。这个转化非常自然：信息检索里，你有一批文档或集合，要回答一个模式是否命中；数据库里，你有记录集合，要判断某个查询条件是否与某条记录兼容；近邻搜索里，你希望预处理点集后快速回答查询点。最朴素的两种方案代表了典型的数据结构张力：要么把所有 $2^d$ 个查询的答案都预先存下来，查询几乎免费但空间指数；要么只存输入向量，查询时扫描全部 $n$ 个向量，空间近线性但查询线性。OnlineOV 的问题正是：在 $d=c\log n$ 或 $d=n^\varepsilon$ 这样的维度下，能不能同时显著低于这两个平凡边界？

这也是为什么它和 Partial Match 几乎是同一个世界。Partial Match 预处理 $n$ 个 $\{0,1\}^d$ 字符串，查询是 $\{0,1,*\}^d$ 模式，星号表示 wildcard，目标是判断是否有字符串在非 wildcard 坐标上完全匹配。把“查询为 1 的坐标必须在输入向量中为 0”理解成一组匹配约束，就能在 OnlineOV 和 Partial Match 之间做低开销规约。论文还列出到 Subset Query、Containment Query、DNF Evaluation、Orthogonal Range Search、Online Inner Product 和 Approximate Nearest Neighbor 的规约；所以 OnlineOV 不是一个孤立 toy problem，而是很多静态检索问题的共同核心。

:::aside[OnlineOV 与 Partial Match 的等价味道]
OnlineOV 查询 $q$ 要找 $x_i$ 使得所有 $q_j=1$ 的位置都有 $x_{i,j}=0$。Partial Match 查询模式 $y\in\{0,1,*\}^d$ 要求非 wildcard 坐标精确匹配。把每个布尔坐标拆成两个坐标，可以把“必须为 0”的约束编码成匹配约束；反向也可以把匹配 0/1 的要求写成若干正交约束。因此很多 upper/lower bounds 会在这些问题之间同步移动。
:::

历史上，这个问题从 Knuth (1973)、Rivest (1974, 1976) 就已经以 associative retrieval / partial-match retrieval 的形式出现。Charikar, Indyk, and Panigrahy (2002) 给出长期最好的确定性 tradeoff：一个算法达到
$$
S=n\cdot 2^{O(d\log^2 d\sqrt{t/\log n})},\qquad T=n/2^t,
$$
另一个达到
$$
S=ndt,\qquad T=O(nd/t).
$$
在 $d=c\log n$ 的低维区间，Chan (2017) 后来用 randomized 方法做到近线性空间 $S=n^{1.1}$ 和
$$
T=n^{1-1/(c\log c)}
$$
量级的查询时间；Abboud, Jin, and Williams (2025) 也给出相近的随机算法。确定性这边则更弱。Goldstein, Lewenstein, and Porat (2017) 曾提出一个硬度猜想：对每个 $\varepsilon>0$，存在 $c>1$，使得 OnlineOV$_{n,c\log n}$ 不可能同时有 $S=n^{c-1}$ 和 $T=n^{1-\varepsilon}$。随机算法已经反驳了这个猜想，但只是在 oblivious adversary 意义下：查询不能依赖预处理阶段的随机选择。本文的第一个贡献是把这个反驳变成确定性结论，所以它真正关闭的是 adversarial query 情形下的猜想空间。

算法的入口其实很朴素。假设输入向量是随机的，每个坐标独立地以概率 $1/2$ 为 0。对任意一组坐标 $C$，令
$$
Y_C=\{x\in X:x|_C=0\}.
$$
如果查询 $q$ 在 $C$ 上全为 1，那么任何与 $q$ 正交的输入向量都必须属于 $Y_C$。随机输入下，只要 $|C|=t$ 稍微大一点，$Y_C$ 的期望大小就是 $n/2^t$；用 Chernoff bound 加 union bound，可以保证所有这样的 $C$ 都没有太大的候选列表。于是平均情形算法非常直接：预处理时，对所有小 Hamming weight 的查询直接存答案；对所有 $|C|=t$ 的坐标集合存候选列表 $Y_C$。查询时，如果 $q$ 很稀疏，就查 bitmap；否则取 $q$ 中任意 $t$ 个 1 坐标，扫描对应的 $Y_C$。

这个平均情形算法给出 Corollary 3.5：当 $d=c\log n$ 且 $\delta>\Omega(\log c/c)$ 时，在随机输入 $B_{1/2}$ 上有
$$
T\le n^{1-\Omega(1/\log c)},\qquad S,T_p\le O(n^{1+\delta}).
$$
注意这里查询指数损失是 $1/\log c$，比最坏情形主定理里的 $1/(c\log c)$ 更强。这个差距不是分析小瑕疵，而是论文技术路线的真实代价：从随机输入到任意输入，需要递归处理所有“不随机”的结构块。

最坏情形算法的关键概念是 $(m,t)$-pseudorandomness。它不是密码学意义上的伪随机，而是一个非常具体的组合条件：对每个 $|C|=t$，候选集合 $Y_C$ 的大小都不超过 $m$。若输入 $X$ 满足这个性质，刚才的平均情形数据结构就直接工作。若不满足，说明存在一大批向量在同一组 $t$ 个坐标上全为 0；这就是可利用的结构，因为处理这些向量时可以把这 $t$ 个坐标删掉，维度下降。

论文把这个观察做成一个贪心分解。给定 $X$，不断寻找某个 $C$ 使得 $Y_C$ 里还有至少 $m$ 个向量；取出其中 $m$ 个作为一个 structured part $X_j$，并记录对应的坐标集 $C_j$。这个过程结束后，剩余部分 $X'$ 自动是 $(m,t)$-pseudorandom；每个被取出的 $X_j$ 都满足所有向量在 $C_j$ 上为 0。因此预处理可以存三类信息：稀疏查询 bitmap；伪随机剩余部分 $X'$ 的所有候选列表；以及每个 structured part 删除 $C_j$ 后的递归数据结构。

这里最巧的一步是把“坏情况”不是当成需要规避的 adversarial obstruction，而是当成递归降维的燃料。若某个坐标集合造成候选列表太大，普通列表扫描会失败；但正因为候选列表大，这些向量共同拥有很多 0，问题规模在维度上就变小了。算法分析的平衡点是设递归参数 $i$，令
$$
m=n^{1-1/i},\qquad t=d/i.
$$
每层最多产生 $n^{1/i}$ 个 structured parts，每个 part 的向量数降到 $n^{1-1/i}$，维度降到 $d-d/i$，递归深度也从 $i$ 变成 $i-1$。Theorem 3.7 给出精确归纳界：
$$
T \le 2idn^{1-1/i},\qquad
S \le \left(\sum_{j\le d/i}\binom{d}{j}\right) idn^{1-1/i},\qquad
T_p \le \left(\sum_{j\le d/i}\binom{d}{j}\right) idn.
$$
把 $i$ 选成 $\Theta(c\log c/\delta)$，再用二项式系数估计，就得到 $d=c\log n$ 下的近线性空间和 $n^{1-\Omega(1/(c\log c))}$ 查询时间。把 $i$ 改成 $\log n/(\varepsilon\log n+\log\log n)$，就得到任意维度版本的 Corollary 3.8。

这个算法和 Chan (2017) 的随机算法在低维参数上相近，但气质很不一样。随机算法靠随机划分或采样绕开 worst-case 结构；本文则显式把输入拆成 pseudorandom part 和 structured parts，然后对后者递归降维。它也没有调用 fast matrix multiplication，而是完全 combinatorial。对我来说，这比“又省了一个随机性”更有信息量：它解释了 OnlineOV 为什么在这些参数下不该被简单地看成 curse of dimensionality 的硬例。真正难的输入必须避免大块共同 0；但一旦它避免了这种结构，它又会像随机输入一样给出短候选列表。

由 OnlineOV 到其他检索问题的扩展基本继承这一点。Corollary 3.10 说，对 PartialMatch$_{n,c\log n}$、DNF-Evaluation$_{n,c\log n}$、SubsetQuery$_{n,c\log n}$、ContainmentQuery$_{n,c\log n}$ 和 $(1+\varepsilon)$-OrthogonalRangeSearch 的相应维度版本，都有
$$
T=n^{1-\Omega(1/(c\log c))},\qquad S,T_p=O(n^{1+\delta}).
$$
由于这些规约是确定性的，算法的确定性也保留下来。这一点在数据结构语境里很重要：随机数据结构常常需要限定 adversary 或给失败概率解释，而确定性结构可以直接用于 adaptive query 序列。

下界部分回答的是另一类问题：如果我们不限制预处理时间，只关心最后存下来的空间和查询时间，能证明什么？这正是静态数据结构 lower bound 的核心难点。通常的 SETH-based 数据结构下界会说：如果预处理也很快，那么一个快查询数据结构可以组合成 SAT 的快算法。但若预处理无界，这个论证失效，因为预处理可能已经用天文时间把输入相关的信息压进数据结构里。无条件地证明显式数据结构问题在 near-linear space 下需要 $n^\varepsilon$ 查询时间，目前远超现有技术；经典 cell-probe 下界在很多显式问题上仍只能给到对数级别。

Non-uniform 假设给了一个绕路。Non-uniform algorithm 可以为每个输入长度拿到一段 advice；如果某个数据结构实例与具体 SAT 公式无关，那么它的存储内容就可以作为 advice。Non-Uniform SETH（NUSETH）断言：对每个 $\varepsilon>0$，存在 $k$，使得没有 non-uniform RAM algorithm 能用 $2^{(1-\varepsilon)n}$ 时间和 $2^{(1-\varepsilon)n}$ bits advice 解 $k$-SAT。这里的重点不是把 SETH “加强一下”这么简单，而是要构造一个**固定的** OnlineOV 实例，让所有 $k$-SAT 公式只体现在查询向量里；否则每个公式都需要不同预处理，无法变成同一段 advice。

:::aside[为什么需要 non-uniform]
在 uniform 算法里，预处理必须由算法实际算出来；在 non-uniform 算法里，长度为 $n$ 的所有输入可以共享一段 advice。若某个 OnlineOV 输入 $D_n$ 只依赖 $n$ 和 $k$，不依赖具体公式 $\varphi$，那么对 $D_n$ 的数据结构内容 $\sigma_n$ 可以直接作为解 SAT 的 advice。本文 lower bound 的核心就是构造这样的 hardest OnlineOV instance。
:::

Abboud and Williams (2021) 已经使用过一个 universal table 思路：行对应变量赋值，列对应所有可能的 $k$-CNF 子句；某行某列为 0 表示该赋值满足该子句。给定公式 $\varphi$，查询向量标记它包含哪些子句，于是存在正交行当且仅当 $\varphi$ 可满足。本文把这个想法推进到稀疏输入和更平滑的维度 tradeoff。Lemma 4.2 构造了一个 $(w,\delta)$-hardest OnlineOV instance $D_{N,w}$：它有 $N$ 个 $w$-sparse 向量，维度 $O(N^\delta)$，其中
$$
w={k/\delta \choose k}=O_{k,\delta}(1),
$$
并且对任意 $k$-CNF 公式 $\varphi$，可以在线性于维度的时间构造查询向量 $a_\varphi$，满足
$$
\varphi\text{ satisfiable}\quad\Longleftrightarrow\quad \exists v\in D_{N,w}: v\perp a_\varphi.
$$

构造方式可以这样理解：把 SAT 变量分成 $\theta=k/\delta$ 个块，每个坐标描述“选出 $k$ 个块以及这些块上的一个局部赋值”。每个完整赋值对应一个稀疏向量，只在与它一致的局部赋值坐标上放 1；而公式查询向量在某个局部赋值不能满足相应子句时放 1。于是完整赋值向量与公式查询正交，恰好表示这个赋值没有违反任何子句。这个编码比 universal table 更细，它用局部赋值坐标换来了输入向量的常数稀疏性，维度则变成 $N^\delta$。

有了 hardest instance，下界证明就很短。假设存在空间 $N^c$、每查询时间 $N^{1-\gamma}$ 的 batch 数据结构。对一个 $m$ 变量 SAT 公式，先固定前 $(1-\beta)m$ 个变量的所有赋值，得到 $2^{(1-\beta)m}$ 个只剩 $\beta m$ 个变量的子公式。令 $N=2^{\beta m}$，用固定的 $D_{N,w}$ 作为预处理输入，数据结构内容作为 advice；每个子公式变成一个 OnlineOV 查询。选
$$
\beta=\min(1/(2c),1/(\alpha+1)),\qquad
\varepsilon=\min(1/2,\beta(1-\delta),\beta\gamma),
$$
则 advice 长度、生成所有查询的时间、以及回答所有查询的总时间都压到 $2^{(1-\varepsilon)m}$ 以内，违反 NUSETH。这就是 Theorem 4.3。

这个下界的力度在于它允许 computationally unbounded preprocessing，并且对任意固定多项式空间 $n^c$ 都能给出障碍；付出的代价是使用 NUSETH，而不是普通 SETH。论文还说明，若不要求输入稀疏，可以把维度降到 $\mathrm{polylog}(n)$，恢复 Abboud and Williams (2021) 类型的低维下界；若保持稀疏，则得到 $d=n^\delta$ 的强空间下界。通过问题间规约，Corollary 1.7 把同样的 $n^c$ vs. $n^{1-\gamma}$ 障碍传到 Partial Match、Subset Query、Containment Query、Orthogonal Range Search、DNF Evaluation、OnlineIP 和 $(1+\varepsilon)$-Approximate Nearest Neighbor。

论文最后还用 Non-Uniform Hamiltonian Path Conjecture 给 3-SUM with preprocessing 证明下界。这里模型是：预处理三个大小为 $N$ 的整数集合 $X_1,X_2,X_3$，查询给出各自子集 $X'_1,X'_2,X'_3$，问是否存在 $x_1+x_2=x_3$。已有研究做到 $S=T=O(N^{3/2})$，但没有强无条件下界。本文证明若 directed Hamiltonian Path 没有 $2^{(1-\varepsilon)n}$ 时间、同等 advice 的 non-uniform 算法，则 3-SUM with preprocessing 不存在 $(N^{1.087},N^{1.087})$ 数据结构；更一般地，对 $k$-SUM，若 $\delta<1/H(1/k)$，不存在 $(N^\delta,N^\delta)$ 数据结构。这个结果和 OnlineOV 主线不是同一个规约，但传递了同一个观点：当预处理无界时，能把“固定输入的压缩内容”解释成 non-uniform advice，是目前少数能绕过 cell-probe lower bound 瓶颈的方式。

我觉得这篇论文最完整的地方，是 upper bound 和 lower bound 并没有互相打架，而是在描述同一条边界的两面。算法说：OnlineOV 的 worst-case 输入可以被拆成随机状部分和降维结构部分，因此低维和中等维度里仍有确定性次线性查询空间；下界说：如果你希望在无界预处理下用任意固定多项式空间换来强次线性查询，那么这已经接近 non-uniform SAT/HamPath 的指数时间壁垒。也就是说，本文不是只给一个更快数据结构，也不是只给一个条件下界；它把 OnlineOV 重新放回了静态检索、fine-grained reductions 和 non-uniform complexity 的交叉点上。

## Faster Algorithms for Shortest Unique or Absent Substrings [arXiv:2605.04826](https://arxiv.org/abs/2605.04826)

这篇论文研究两个经典字符串问题：给定长度为 $n$ 的字符串 $S$，求一个 shortest unique substring（SUS）或 shortest absent substring（SAS）。Unique substring 指在 $S$ 中恰好出现一次的子串，SUS 是其中长度最短者；absent substring 指完全不在 $S$ 中出现的串，SAS 是其中长度最短者。作者考虑的是 word RAM 上的 packed string 模型：字母表是整数区间 $[0,\sigma)$，一个机器字能装下 $\Theta(\log_\sigma n)$ 个字符，所以输入本身只占 $O(n\log\sigma/\log n)$ 个 word。主结果是，在这个模型中可以用
$$
O\!\left(\frac{n\log\sigma}{\sqrt{\log n}}\right)
$$
时间求出一个 SUS，也可以用同样时间求出一个 SAS，突破了 suffix tree 给出的 folklore $O(n)$ 时间。

这个结果的入口不是“有没有线性算法”——线性算法早就有——而是“线性相对于什么度量”。在普通 RAM 分析里，字符串有 $n$ 个字符，读一遍就是 $\Theta(n)$；但在小字母表上，比如二进制串或 DNA 字母表，一个 word 可以同时携带许多字符。此时仅仅读入 packed 输入只需 $O(n\log\sigma/\log n)$ 时间。如果某个算法仍然逐字符扫完整个字符串，它在 word RAM 意义下就可能不是最优的。过去十多年，很多字符串任务都在这个角度被重新审视：pattern matching、BWT construction、LZ77 factorization、longest common substring 等都有 sublinear-in-$n$ 的 packed-string 算法。本文把 SUS/SAS 也放进这条线，并说明这两个“在 suffix tree 上几乎一眼可见”的对象，在 packed setting 里仍然有非平凡结构可挖。

先看经典解法为什么是 $O(n)$。Suffix tree 是把 $S$ 的所有后缀压缩成一棵 trie；一条从根到某个位置的路径标签对应 $S$ 的某个子串。对 SUS 来说，如果一个叶子 $v$ 表示某个后缀，而 $u$ 是它的父亲，那么从该后缀起点出发，长度 $\mathrm{sd}(u)+1$ 的前缀就是这个位置上最短的 unique substring：到 $u$ 为止还和别的后缀共享，往前多走一个字符后只剩这一个叶子。枚举所有叶子取最短即可。对 SAS 来说，如果某个 suffix-tree 节点 $u$ 没有以字符 $c$ 开头的出边，那么 $\mathrm{str}(u)c$ 就是不出现的串；取 string depth 最小的这种缺边即可。这个描述也解释了为什么这两个问题看起来“应该线性”：suffix tree 本身线性大小，答案也藏在线性个节点/边附近。

但 packed setting 里不能直接建完整 suffix tree。完整 suffix tree 有 $\Theta(n)$ 个节点，光把它写出来就可能比输入大一个 $\Theta(\log n/\log\sigma)$ 因子。本文的算法因此不是“更快地建 suffix tree”，而是只检查可能成为最短答案的结构，并把候选分成四类：很短的串、中等长度的非周期串、很长的非周期串、周期串。这个 decomposition 是全文的主线。这里的 period 是字符串算法里的标准概念：串 $P$ 的周期 $p$ 意味着 $P[i]=P[i+p]$ 对所有合法 $i$ 成立；如果一个长串有很小周期，它基本由短 pattern 重复而成，行为和一般串完全不同。SUS 的难点正好在这里：非周期串适合用局部 anchor 抽样，周期串则必须用 runs 的结构单独处理。

:::aside[模型与复杂度尺度]
论文使用 unit-cost word RAM，word size 为 $w=\Theta(\log n)$。Packed representation 把 $\Theta(\log_\sigma n)$ 个字符放入一个 word，因此长度为 $m$ 的串占 $O(1+m/\log_\sigma n)=O(1+m\log\sigma/\log n)$ 个 word。本文的 $O(n\log\sigma/\sqrt{\log n})$ 时间在二进制字母表上是 $O(n/\sqrt{\log n})$，比逐字符线性时间快一个 $\sqrt{\log n}$ 因子。
:::

研究脉络上，本文最直接接在两类工作后面。第一类是 packed string algorithms。Charalampopoulos et al. (2021) 的 longest common substring 算法 [arXiv:2105.03106](https://arxiv.org/abs/2105.03106) 已经展示了 wavelet tree 和 LCP 信息如何在 packed setting 中把某些“比较所有相邻后缀”的任务压到 $O(n\log\sigma/\sqrt{\log n})$。Kempa and Kociumaka (2019) 的 string synchronizing sets 提供了更深的工具：对非周期区域，可以用 $O(n/\tau)$ 个 anchor 代表所有长度 $\Theta(\tau)$ 的局部上下文，而且相同上下文会作出相同选择。第二类是 word RAM 细粒度下界。Kempa and Kociumaka (2025) [arXiv:2503.21049](https://arxiv.org/abs/2503.21049) 研究了一批最快算法处在 $O(n\sqrt{\log n})$ word-time 尺度的问题，并对 BWT、LZ77、longest common substring 等给出条件最优性框架。本文指出 SUS/SAS 在二进制情形也落到同一数量级：按输入 word 数 $N=\Theta(n/\log n)$ 来看，$O(n/\sqrt{\log n})=O(N\sqrt{\log n})$。但它们的“硬”味道不一样：那些下界 reduction 往往围绕“找某个模式是否出现”，而 SUS/SAS 分别是在找“只出现一次”和“完全不出现”的最短 witness。

主定理可以精确写成：
$$
\textbf{Theorem 1.1.}\quad
\text{给定 packed string } S\in[0,\sigma)^n,\text{ 可以在 }
O\!\left(\frac{n\log\sigma}{\sqrt{\log n}}\right)
\text{ 时间内计算一个 SUS。}
$$
以及
$$
\textbf{Theorem 1.2.}\quad
\text{给定 packed string } S\in[0,\sigma)^n,\text{ 可以在 }
O\!\left(\frac{n\log\sigma}{\sqrt{\log n}}\right)
\text{ 时间内计算一个 SAS。}
$$
prior best 是 suffix tree folklore 的 $O(n)$；对常数字母表，这就是从 $O(n)$ 到 $O(n/\sqrt{\log n})$。论文还证明了一个有意思的 reduction：任意字母表上的 packed SUS instance 可在 $O(n/\log_\sigma n)$ 时间化为长度 $O(n\log\sigma)$ 的二进制 SUS instance，且不增加渐近运行时间。这说明从 packed SUS 的角度看，binary alphabet 是 hardest case；不是因为二进制串更复杂，而是因为它的输入压缩得最狠，留给算法的时间预算最少。

算法的第一段处理很短的候选串。令 $\ell=\lfloor \frac15\log_\sigma n\rfloor$，所有长度至多 $\ell$ 的信息可以被装进常数个 word。作者把 $S$ 用长度约 $2\ell$ 的重叠片段覆盖，每个长度 $\ell$ 的子串至少落在某个片段内；由于不同片段最多只有 $O(\sigma^{2\ell})=O(n^{2/5})$ 种，先 bucket sort，再对每种片段最多保留两个副本，就足以保留“某个短串是否唯一”的信息。之后在这些短片段拼出的辅助串上建小 suffix tree，求出最短唯一者。关键点是“保留两个副本”：SUS 只关心出现次数是 $1$ 还是至少 $2$，第三次及之后不再带来新信息。这个 case 的时间是 $O(n/\log_\sigma n)$，已经接近读入 packed 输入。

中等长度非周期串是总复杂度的瓶颈。这里候选长度在
$$
\left[\frac15\log_\sigma n,\ 2\sqrt{\log n}\right]
$$
附近，周期大于 $\frac1{45}\log_\sigma n$。作者把问题化为 Shortest Unique String Pair：给一组二元字符串对 $(U_i,V_i)$，要找最短的前缀长度 $(\ell_1,\ell_2)$，使得某一对在第一分量或第二分量上与所有其他对分离。直觉上，候选 SUS 被某个 anchor 分成左半和右半；它唯一，当且仅当没有其他 anchor 同时匹配足够长的左半和右半。这个表述和 longest common substring 里“两组字符串 LCP 最大化”的结构很像，但目标从最大化公共前缀和，变成最小化足以排除所有竞争者的两侧长度。

如果候选串的开头已经落在一个短周期 run 里，所有相关二元对形成 prefix family，可以线性处理。否则，synchronizing set 会在每个非周期候选的相同相对位置放置 anchor；取 anchor 左侧最多 $\tau$ 个字符反转成第一分量，右侧最多 $\beta=2\sqrt{\log n}$ 个字符作第二分量，就得到一个 $(\alpha,\beta)$-family。作者复用 Charalampopoulos et al. (2021) 的 wavelet-tree construction：按第二分量排序，在第一分量的 trie skeleton 上建 wavelet tree，并为每个节点维护相邻第二分量的 LCP list。一个节点代表“左侧共同前缀长度已经固定”，LCP list 则告诉我们右侧还要多长才能成为 shortest unique prefix。于是每个节点线性扫描一次，整体得到
$$
O\!\left(\frac{n\log\sigma}{\sqrt{\log n}}\right).
$$
这里最值得注意的是目标函数的翻译：论文不是把 LCS 算法当黑盒，而是保留它的 wavelet-tree 数据流，只把“找最大 LCP”替换成“在每个局部列表里找 shortest unique prefix”。这个替换看起来小，但它正好把 unique 的全局条件压进了相邻 LCP。

长非周期串用的是 synchronizing sets 的本来面目。取 $\tau=\lfloor \frac13\log^4 n\rfloor$。如果一个长度至少 $3\tau$、周期大于 $\tau/3$ 的模式出现两次，那么由 synchronizing set 的 consistency，两个 occurrence 内相同 offset 的 anchor 必须同步出现。于是每个长非周期候选都可以围绕某个 anchor 分解：右边是 $S[i..n)$ 的前缀，左边是 $S[0..i)^R$ 的前缀。作者为所有 anchor 建两棵 compacted tries $T$ 和 $T^R$，一棵存 anchor 后缀，一棵存反转前缀。某个候选串的出现次数，就变成两棵 trie 中两个节点的 leaf-label 集合交集大小；唯一性就是
$$
|L(u)\cap L(v)|=1.
$$

直接在两棵 trie 上找这样的节点对仍然太大。论文下一步把它化为 Two Trees SUS，再用 heavy-light decomposition 解决。Heavy-light decomposition 的作用是把每条 root-to-leaf path 切成 $O(\log N)$ 条 heavy paths；每个 leaf label 只会参与 $O(\log^2 N)$ 对 heavy paths。固定一对 heavy paths $(h_1,h_2)$ 后，每个共同 leaf $\ell$ 给出一个点
$$
(d_{h_1}(\ell), d_{h_2}(\ell)),
$$
其中 $d_h(\ell)$ 表示 $\ell$ 在该 heavy path 上能下降到的最大深度。找两个节点使得交集大小为 $1$，就等价于在这些点里找一个二维格点，它被恰好一个点支配，并使坐标和最小。这就是论文反复使用的 Minimum Skyline Point。

这个几何子问题很简单，但放在这里非常好用。给定点集 $P\subseteq\mathbb Z_{\ge0}^2$，点 $(x,y)$ 被 $(a,b)$ 支配表示 $x\le a$ 且 $y\le b$。Primary skyline 是那些被恰好一个输入点支配的格点；Minimum Skyline Point 要找 primary skyline 上最小化 $x+y$ 的点。若点按一个坐标排序，从右往左扫，只需维护当前见过的最大和第二大 $y$ 值，就能在线性时间找答案。于是 long aperiodic case 的 Two Trees SUS 可在 $O(N\log^3 N)$ 时间解出；这里 $N=O(n/\log^4 n)$，所以仍被 $O(n/\log_\sigma n)$ 吸收。

:::aside[Primary skyline 的严格形式]
对 multiset $P$，一个格点 $q$ 在 primary skyline 上，当且仅当 $q$ 被 $P$ 中恰好一个点支配。若有重复点，例如 $P=\{(7,3),(7,3)\}$，则 $(7,3)$ 支配的整片区域都被至少两个点支配，primary skyline 为空。这也是为什么论文把 $P$ 当 multiset 而不是 set：重复 occurrence 会破坏唯一性。
:::

周期串是最需要字符串结构的一段。Run 是一个不能再向左右扩展的最大周期片段，长度至少为周期的两倍；它描述了 $S$ 中所有长周期区域。经典 runs theorem 告诉我们，一个长度为 $n$ 的串只有 $O(n)$ 个 runs，而本文需要更强的 packed 版本：对阈值 $\tau$，长度至少 $3\tau-1$ 且周期至多 $\tau/3$ 的 $\tau$-runs 只有 $O(n/\tau)$ 个，并且可以在 packed 时间内分组。分组依据是 Lyndon root。Lyndon word 是在所有非平凡 rotation 中字典序最小的 primitive word；每个 run 可以写成某个 Lyndon root 的若干次重复，再加左右残余扩展。对较大的 $\tau=\Theta(\log^4 n)$，论文调用最近的 sparse-Lyndon representation 来保持同样的分组能力。

周期 case 的漂亮之处，是把 run containment 的繁琐对齐条件再次变成 skyline。固定同一个 Lyndon root $\lambda$，设 $r=|\lambda|$，每个 run 的表示是 $(\lambda,e,\alpha,\beta)$：中间有 $e$ 个 $\lambda$，左侧有长度 $\alpha$ 的 suffix extension，右侧有长度 $\beta$ 的 prefix extension。若要在这组 runs 内找 SUS，只需考虑 exponent 为 $e_{\max}, e_{\max}-1, e_{\max}-2$ 的候选；更短的候选会在最大 run 中出现多次，不可能唯一。作者定义映射 $\mathfrak h$，把每个 run 变成至多三个点；再定义映射 $\mathfrak g$，把 $[0,2r-1]^2$ 中的格点还原成候选 run。Lemma 3.16 说：
$$
p\text{ 位于 }\mathfrak h(R)\text{ 的 primary skyline}
\quad\Longleftrightarrow\quad
\mathfrak g(p)\text{ 是 }R\text{ 中的 unique substring}.
$$
因此，对同一 Lyndon root 的所有 runs，求 SUS 又变成一个 Minimum Skyline Point instance，点数至多 $3|R_\lambda|$。这里我一开始以为会出现一套很专门的 periodic-case DP，结果作者把 exponent 差 $0,1,2$ 的所有 alignment 情况编码进二维象限；做完这步以后，周期串和长非周期串共享同一个几何核心。

SAS 的算法建立在 SUS 框架上，但要先换一个问题。Shortest Exclusive Substring 给两串 $S_1,S_2$，目标是在 $S_1$ 中找一个最短子串，使它不在 $S_2$ 中出现。它和 SUS 的关系是：SUS 要“被一个 occurrence 覆盖且不被第二个 occurrence 覆盖”，exclusive 要“被 $S_1$ 的 occurrence 覆盖且不被 $S_2$ 覆盖”。所以几何子问题从 Minimum Skyline Point 变成 Minimum Exclusive Point：找一个点被 $P_1$ 至少一个点支配、但不被 $P_2$ 中任何点支配，并最小化 $x+y$。同样按 $x$ 扫描，只需维护 $P_1$ 和 $P_2$ 已处理点的最大 $y$ 值，线性可解。四个 case 的框架因此可以平移到 Shortest Exclusive Substring，得到
$$
\textbf{Theorem 5.1.}\quad
\text{Shortest Exclusive Substring 可在 }
O\!\left(\frac{n\log\sigma}{\sqrt{\log n}}\right)
\text{ 时间内解决。}
$$

最后一步用 de Bruijn sequence 把 SAS 化成 exclusive。Order $k$ 的 de Bruijn sequence 是一个长度 $\sigma^k+k-1$ 的串，其中每个长度 $k$ 的字符串恰好出现一次。令
$$
k=\lfloor\log_\sigma n\rfloor+1.
$$
因为 $S$ 只有 $n-k+1$ 个长度 $k$ 子串，而总共有 $\sigma^k>n$ 个长度 $k$ 串，任意 SAS 长度至多 $k$。作者先构造 order $k-1$ 的 packed de Bruijn sequence $B_{k-1}$，用 Shortest Exclusive Substring 在 $(B_{k-1},S)$ 上检查是否存在长度至多 $k-1$ 的 absent substring；若没有，再处理长度 $k$。当完整 order $k$ sequence 太长时，只需生成长度 $n+1$ 的前缀：这个前缀已经包含 $n-k+2$ 个互异长度 $k$ 子串，比 $S$ 的长度 $k$ 子串数多一个，必有一个 absent witness。

:::aside[de Bruijn sequence 的 packed 构造]
论文使用 Fredricksen-Maiorana 的 Lyndon-word characterization：按字典序连接所有长度整除 $k$ 的 Lyndon words，可得到 order $k$ 的 de Bruijn sequence。再用 Duval (1988) 的 Lyndon word 生成算法，并观察 $k\le \lceil\log_\sigma n\rceil$，每个当前 word 可放在 $O(1)$ 个机器字里维护。由此可在 $O(\sigma^k/k)$ 时间构造 packed sequence，或在 $O(\ell/\log_\sigma n+1)$ 时间构造长度 $\ell$ 的前缀。
:::

这篇论文最有意思的地方，是它没有一个单一“银弹”数据结构。短串靠 tabulation，中等非周期靠 wavelet-tree data flow，长非周期靠 synchronizing anchors 和 heavy-light，周期串靠 runs/Lyndon roots；但这些部件最后都服务于同一个判断：一个局部 witness 是否只被一个全局 occurrence 支持。Minimum Skyline Point 把这个判断压成二维 dominance，是全文最稳的抽象层。也正因为 SUS/SAS 寻找的是“不重复”或“不出现”，它们和 Kempa-Kociumaka hardness hierarchy 中那些“找 occurrence”的问题不完全同型；如果以后要证明 $O(n/\sqrt{\log n})$ 在二进制 packed setting 下条件最优，恐怕需要更贴近 uniqueness/absence 的 reduction，而不是直接搬 dictionary matching 那套框架。

## 未入选但值得关注

### Average Attention Transformers and Arithmetic Circuits [arXiv:2605.04683](https://arxiv.org/abs/2605.04683)
这篇把 average attention transformer 的表达能力和 arithmetic circuits 联系起来，有清楚的模型刻画价值。未入选主要是因为它更偏 transformer 表达能力，而今天版面优先给了 proof/data-structure/geometry 等更接近 TCS 主线的结果。

### Local Homophily on Bicolored Graphs is $\mathbf{P}$-complete [arXiv:2605.05047](https://arxiv.org/abs/2605.05047)
论文证明 bicolored graph 上的 local homophily 动态达到 $\mathbf{P}$-complete，属于干净的局部图动力系统复杂性分类。未入选是因为从摘要看主要是 circuit simulation 型证明，深讲空间不如入选论文大。

### On the Complexity of Minimum Riesz s-Energy Subset Selection in Euclidean and Ultrametric Spaces [arXiv:2605.04715](https://arxiv.org/abs/2605.04715)
这篇给出 Euclidean 空间中的 hardness 与 ultrametric 空间中的 DP，形成清楚的复杂性图景。未入选是因为它更像自然优化目标的分类补全，重要性略低于今天被筛选结果标为更核心的 convex containment 方向。

### Faster Iterative $\phi$ Queries on the Positional BWT [arXiv:2605.04244](https://arxiv.org/abs/2605.04244)
论文改进 positional BWT 上 iterative $\phi$ query 的空间时间 tradeoff，方向具体且可能实用。今天字符串方向已经选了 SUS/SAS，后者的问题背景更通用，所以没有重复展开。

### Constructing Suffixient Arrays Revisited [arXiv:2605.04258](https://arxiv.org/abs/2605.04258)
这篇把 suffixient array 的 one-pass 构造降到线性时间，是整洁的字符串索引改进。未入选主要是因为它与 2605.04826 同属字符串索引，而后者在 packed 模型下的算法结构更丰富。

### Optimally Covering Large Triangles with Homothetic Unit Triangles [arXiv:2605.04111](https://arxiv.org/abs/2605.04111)
论文完整解决了 American Mathematical Monthly 中关于用 homothetic unit triangles 覆盖大三角形的问题，数学上很漂亮。未入选是因为它更偏离散几何题型，TCS 技术含量不如今天筛选中更靠前的 zonotope containment。
