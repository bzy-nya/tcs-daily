## 编辑按语

今天入选的三篇论文覆盖了近似算法、在线算法和动态算法三个不同方向，没有一条强行贯穿的统一技术主线。它们分别推进了 ATSP 常数近似、结构化 set system 上的 weighted online hitting set、以及动态维护 conjunctive queries 的参数化下界。

如果看研究姿态，前两篇都在已有框架上做“结构抽象 + 精确计费”的改进：一篇把分析阈值往前推，一篇把在线损失吸收到可控参数；第三篇则把原有 0/1 dichotomy 扩展成指数层级刻画，让 free-connex 查询的维护难度可以按参数连续定位。

## Better approximation guarantee for Asymmetric TSP [arXiv:2603.14334v1](https://arxiv.org/abs/2603.14334v1)

ATSP 这条线之所以一直被盯着，不只是“经典 NP-hard”这么简单。它对应的是一类很现实的路由抽象：从 $u$ 到 $v$ 的代价和从 $v$ 到 $u$ 的代价不一样（单行道、风向、拥堵时段、载重限制都会造成这种不对称），我们要找一条闭合行程覆盖所有点且总成本最低。对称 TSP 里很多好用的几何直觉在这里都会失效，所以 ATSP 常被看成“metric routing 里最顽固的核心问题”之一。

论文里采用的 tour 定义是一个连通且 Eulerian 的多重边集 $F$：允许重边，要求每个点入度等于出度，这样才能拼成一个封闭行程。形式化输入是强连通有向图 $G=(V,E)$ 和非负边权 $c:E\to \mathbb{R}_{\ge 0}$，目标最小化 $c(F)$。这一定义看起来比“哈密顿环”宽，但它正是近似算法和 LP 分析最自然的对象。

这条线的脉络其实很清楚：长期只有 $O(\log n)$ 或 $O(\log n/\log\log n)$ 近似；Svensson, Tarnawski, Végh (2020) 首次把 ATSP 拉进常数近似时代，比例 $506$（预印本 [arXiv:1708.04215](https://arxiv.org/abs/1708.04215)）；Traub 和 Vygen (2022) 降到 $22+\varepsilon$（[arXiv:1912.00670](https://arxiv.org/abs/1912.00670)）；Traub 和 Vygen (2025) 再降到 $17+\varepsilon$。所以这篇的价值不是“从 1.5 到 1.4 这种微调”，而是在已经很难压常数的阶段，继续把主常数明显往下推到 $15$ 以下。

本文的主定理写成最紧凑的形式是
$$
\forall\alpha>3+2\sqrt{2},\quad \mathrm{cost}(\mathrm{ALG})\le (9+\alpha)\cdot \mathrm{LP},
$$
其中 $\mathrm{LP}$ 是自然 cut-based 松弛的最优值。这立刻给出 ATSP 近似比和 LP integrality ratio 上界都小于
$$
9+(3+2\sqrt{2})=12+2\sqrt{2}<14.83.
$$
对 unweighted digraph（论文称 Asymmetric Graph TSP），同一分析还给出边数界
$$
|F|\le \alpha\cdot \mathrm{LP}\qquad(\alpha>3+2\sqrt{2}),
$$
也就是把该特例从先前的 $<8$ 级别继续压到 $<5.83$。

:::aside[自然 LP 的严格写法]
本文比较的 LP 是
$$
\begin{aligned}
\min\ &\sum_{e\in E} c(e)x_e\\
\text{s.t. }&x(\delta(U))\ge 2,\ \emptyset\neq U\subset V,\\
&x(\delta^+(v))=x(\delta^-(v)),\ \forall v\in V,\\
&x_e\ge 0,\ \forall e\in E.
\end{aligned}
$$
integrality ratio 指最优整数 tour 与该 LP 最优值之比的最坏情形上界。
:::

如果只看 theorem statement，会以为作者做了新的大框架；但读完整篇会发现，真正“狠”的地方是对 Svensson 算法第 2 步的一次很小但非常到位的改造。原算法每轮做三件事：先求一个 subtour cover $F$，再尝试找到更好的 initialization 来重启，最后把一部分边并入当前解。Traub-Vygen 2022/2025 的大部分改进都建立在这条主框架上；这篇没有推翻它，而是精准找到“什么时候应当更早重启”的新触发条件。

为了让这个改造可解释，先说一个核心中间对象。论文在 strongly laminar 实例上工作：边权被分解成一组层次化 cut 权重 $y_L$，并固定一个 backbone（可理解为“先覆盖所有大层级块的骨架”）。在这个结构下，整体 ATSP 通过 vertebrate pairs 归约为“补边问题”：在 backbone 外补一个多重边集 $H$，把全图补成 tour，同时控制 $c(H)$。这一步是近年来常数改进的主通道，因为它把“全局连接”拆成了“骨架 + 局部修复”。

作者的新招发生在 subtour cover 被分解成 circuits 之后。设 $\widetilde W_i$ 是当前 initialization 的第 $i$ 个连通分量，直观上每个 circuit $C$ 都有“在 $\widetilde W_i$ 内部绕的部分”和“跑到外部再回来的部分”。如果外部部分足够大（论文不等式 (6)），那么把 $C$ 并进去会显著增加 slack，就能构造更好的 initialization 并立刻重启。这一步在形式上由 Lemma 5/6/7 接起来，本质是把“坏循环”反过来当成“改良种子”。

关键在于：即便没有触发重启，作者也不会空手而归。因为“没触发”本身就强迫每个 circuit 满足互补不等式 (11)，再结合 subtour cover 的入度控制
$$
|F\cap \delta^-(v)|\le 2x(\delta^-(v)),
$$
可以推出
$$
c(F_i)\le \frac{2(1-\varepsilon)}{\gamma(1-\varepsilon)-1}\,\ell(\widetilde W_i)
$$
（Theorem 8 的式 (10)）。这条界就是后续常数改进的齿轮：它把“每层要付的 cover 成本”压得比 2025 版更低。

接下来在 Lemma 10，作者把所有会被真正加入解的 $F$-edges 总成本做了全局封顶。和旧分析一样，解的成本分成三桶：初始边、Step 3 里按 slack 购买的 $X$-edges、以及来自 subtour cover 的 $F$-edges。前两桶基本沿用旧框架，改进几乎全部来自第三桶的新上界。再把参数设成
$$
\gamma=1+\sqrt{2},\qquad \varepsilon=\min\left\{\frac16,\frac{\alpha-3-2\sqrt{2}}{10}\right\},
$$
即可满足收尾不等式，得到 $\alpha>3+2\sqrt2$ 的最终保证。

我读这篇最强的感受是：它不是“发明了更复杂的结构”，而是把重启机制的判定边界往前挪了一点点，然后把“重启或不重启”两侧的信息都吃干榨净。很多近似算法论文喜欢在主流程上加新模块，这篇反而像在原引擎里换一个阀门，结果主常数就从 $17+\varepsilon$ 掉到 $<15$。这种改进通常比大改框架更可信，因为它和已有 proof skeleton 高度兼容。

论文最后还给了两个延伸结果。其一是 Asymmetric Path TSP：借 Traub-Vygen 2025 的黑盒转换（$(\kappa,\eta)$ 映到 $\max\{3\kappa+\eta+2,4\kappa+7\}$），代入本文的 $\kappa=2,\eta<7$，得到近似比与 LP ratio 上界都到 $15$（此前是 $17$）。其二是 unweighted path 版本降到 $<11$。这些不是“顺带一提”的小修补，而是说明这次常数下降已经进入了可传导的 reduction 链。

文末 remark 还有一句很值得记：结合 Traub-Vygen 2025 书里若干练习中的额外优化，整体 ATSP 保证可到 $<14.7$。作者自己也明确说“我们还没完全理解这套算法的潜力”。这不是客套话，因为 Section 5.1 已经指出一个具体缝隙：当 $F_i$ 里出现很多成对循环/连通块时，现有分析没有完全榨干其结构；如果有人能把这部分转成新的可计费不等式，下一次改进很可能还在同一条骨架上发生。

## Improved Online Hitting Set Algorithms for Structured and Geometric Set Systems [arXiv:2603.14293v1](https://arxiv.org/abs/2603.14293v1)

这篇文章讨论的其实是一个很“工程化”的在线决策困境：约束一条条到来，你每次都得立刻补救，而且历史决策不能撤销。在线 hitting set 就是这个困境的抽象版：地面集是 $U$，到达的是集合序列 $S_1,S_2,\dots$，算法要维护一个单调增长的命中集 $C_t\subseteq U$，保证对每个时刻 $t$ 都有 $C_t\cap S_t\neq\emptyset$。离线最优看完整序列后选最小代价集合，在线算法只看前缀；我们比较二者比值，这就是 competitive ratio。

为什么这个方向一直有人做？因为它是 set cover/hitting set 在“信息不完整”场景下的核心原型，背后对应的是在线监测、增量索引、动态资源部署这类问题：每来一个新需求，你必须立即买一个“保险点”进去，后续只能加不能减。更关键的是，离线世界里我们早就知道结构化 set system 能显著降复杂度，但在线世界长期卡在通用上界附近，这个落差非常刺眼。

研究脉络大概是这样。Alon, Awerbuch, Azar, Buchbinder, Naor (2009) 给出一般 online set cover/hitting set 的 $O(\log n\log m)$ 竞争比，并证明对高效在线算法几乎 tight。与此同时，离线几何/结构化结果一路走高：从 Brönnimann-Goodrich (1994) 到 Even-Rawitz-Shahar (2005)，再到 Varadarajan (2010)、Chan-Grant-Könemann-Sharpe (2012)，核心信息是 VC dimension 和 shallow-cell complexity（SCC）小的时候，离线可以做到远优于通用界，weighted 情况也能被 quasi-uniform sampling 吃下来。在线端的进展则零散得多：Even-Smorodinsky (2014) 在区间、半平面、unit disk 上做到了 unweighted 的 $\Theta(\log n)$；Khan et al. (2023) 把轴对齐正方形也做到紧的对数级，但有明显前提（如格点）。于是关键 open gap 变成：能不能在**加权**且**结构化/几何**的在线 hitting set 里，把双对数现象真正打破？

本文给的答案是肯定的，而且是定量上的实质推进。先是 unweighted：
$$
\mathrm{CR}_{\mathrm{unw}}
= O\!\left(\log n\cdot\bigl(d+\log(d\cdot \phi(d\cdot \mathrm{OPT}\log n,d))\bigr)\right),
$$
其中 $d$ 是 VC dimension，$\phi$ 是 SCC。在线性 SCC（$\phi(\ell,k)=\mathrm{poly}(k)$）且常数 VC 维时，这直接化成 $O(\log n)$。然后是论文最重的一刀：weighted 情况在同样线性 SCC 下做到
$$
\mathrm{CR}_{\mathrm{w}} = O(\log n\log\log n),
$$
把通用 $O(\log n\log m)$ 的第二个对数基本压成了 $\log\log n$，并给出了几何家族上的首批 weighted online bounds。

:::aside[SCC 在文中的角色（形式化）]
对 set system $(U,\mathcal F)$，取任意 $U'\subseteq U, |U'|=\ell$，令 $\mathcal F_{U'}=\{S\cap U':S\in\mathcal F\}$。若其中大小至多 $k$ 的不同子集个数至多 $\ell\,\phi(\ell,k)$，则称 SCC 为 $\phi$。文中还要求 $\phi$ 满足一种“well-behaved”缩放条件：存在 $a\in(1,2),b\ge2$，当 $\ell\ge k\ge b$ 时有 $\phi(\ell,k)\le (\phi(\ell/2,k/2))^a$。线性/多项式型 SCC 都在这个框架里。
:::

技术上，我觉得最聪明的是他们没有“硬把某个离线算法改成在线”，而是先搭了一个统一中间层。给任一在线 fractional hitting set 算法 $\mathcal A$（竞争比 $\rho$），先做一次 uniformization：把每个元素 $e$ 复制成若干 clone $(e,i)$，第 $t$ 个到达集合按当前分数解 $x^t$ 展开成 clone 集合 $T_t$。这个转换（Theorem 3.2）有三个后果几乎决定了整篇论文：每个 $T_t$ 至少有 $B=n$ 个元素；活跃 clone 总数 $N'$ 被 $B(1+\sum_e x_e)$ 控住；最关键的是结构基本不炸，$\mathrm{VCdim}(\mathcal G)\le \mathrm{VCdim}(\mathcal F)$ 且
$$
\phi_{\mathcal G}(\ell,k)\le k\,\phi_{\mathcal F}(\ell,k).
$$
也就是说，在线分数解先把问题“拉平”为一个最小集合规模下界明确、且结构参数可追踪的实例，后面的 rounding 才有地方站。

unweighted 路线沿用 Net-Finder 思想（Mustafa 2019），但要跨过两个在线障碍：$N'$ 不是预先已知（参数要跟着 phase 变），以及 base sampling 本来要一次性看全体元素。文章的处理很干净：前者用 doubling phase，后者用 deferred sampling（元素第一次真正出现时再抽样）。这一步看着是实现细节，实质上是在保持原 ε-net 分析骨架不塌的前提下把“全局先验”改成“前缀可见”。

weighted 路线更难，也更值得细看。作者把 CGKS12 的 quasi-uniform sampling 改成在线分层：
$$
V_0\supseteq V_1\supseteq\cdots\supseteq V_{L^*},\qquad
|V_{\ell+1}|\approx |V_\ell|\Bigl(\tfrac12+h_{\mathcal G}(|V_\ell|,B_\ell)\Bigr),
$$
其中偏置项
$$
h_{\mathcal H}(\ell,k)=\frac{8\ln\bigl(k\,\lceil\log \ell\rceil\,\phi_{\mathcal H}(\ell,k)\bigr)}{k}.
$$
直觉上，$V_{L^*}$ 已经能把单元素被选概率压到约 $1/B$，但还会漏掉“缩得太快”的危险集合，于是每层要给危险集合配 backup 元素。离线里这件事可以全局做；在线里看起来像天生需要未来信息。本文的关键转折是把“找 backup”重写成在线负载均衡：对不同尺度的集合族 $\mathcal H_i$，构造映射 $\gamma_i$，用 Azar-Naor-Rom (1995) 式的在线分配保证每个元素负载只比最优多一个 $\lceil\log|X|\rceil$ 因子。这个额外对数被精确吸收到上面的 bias 里，于是还能证明每个元素作为 backup 被抽中的概率是 $O(1/k^2)$。我最意外的一点就在这里：他们没有试图消灭在线损失，而是把损失做成“可计费项”并嵌回采样参数，分析就闭合了。

最终主定理写成（Theorem 5.7）：
$$
\mathrm{CR}_{\mathrm{w}}
= O\!\left(\rho\cdot\log\bigl(B_{L^*}\log n\cdot \phi_{\mathcal F}(N_{L^*},B_{L^*})\bigr)\right).
$$
在线性 SCC 下再化简成 $O(\rho\log\log n)$，配上经典 fractional online set cover 的 $\rho=O(\log n)$，就得到 $O(\log n\log\log n)$。这不是常数打磨，而是把“结构化在线 weighted hitting set 几乎没有一般性结果”的状态，推进到一个可复用框架：fractional online solver + uniformization + structure-aware online rounding。

论文也很诚实地把边界点出来：连续 piercing setting 还没覆盖；高维几何里若 SCC 对 $\ell$ 有明显多项式依赖，当前分析不会自动给出同等级改进。也正因为如此，这篇工作的价值不只是一个新 bound，而是它明确了下一步该攻哪里：要么改进 fractional 基础竞争比 $\rho$，要么在高维结构里继续压 $\phi(\ell,k)$ 对 $\ell$ 的依赖。和我们知识库里那篇 IID list update 的结果（[arXiv:2603.10244v1](https://arxiv.org/abs/2603.10244v1)）放在一起看，有个共同主题很明显：online 问题里，真正有效的改进通常不是“更强对抗模型下硬拼通用上界”，而是抓住实例结构，把 worst-case 分析里那个多余的对数一层层拆掉。

## Towards Parameterized Hardness on Maintaining Conjunctive Queries [arXiv:2603.14754v1](https://arxiv.org/abs/2603.14754)

这篇 paper 在问一个数据库理论里非常“工程又理论”的问题：你已经在服务里跑着一条固定 SQL（更准确地说是 fixed conjunctive query, CQ），数据不断有单条插入/删除，能不能每次改一点点就把答案继续维护好，而不是每次从头 join 一遍。

如果你做过增量视图维护（incremental view maintenance），这个动机非常直觉：业务关心的是低延迟更新和稳定吞吐，不是“离线重算一次最快”。但在理论上，它变成一个很干净的动态复杂度问题：对固定查询 $Q$，更新代价（amortized update time）$t_u$ 能做到多小？同时还能保证枚举答案时每条输出之间的 delay 很小（理想是常数）。

这条线过去十多年有很清楚的脉络。静态查询侧，free-connex CQ 被证明正好是“线性时间求值/常数 delay 枚举”的分界类；动态侧，Berkholz et al. (2017) [arXiv:1702.06370](https://arxiv.org/abs/1702.06370) 在 OMv 假设（Henzinger et al. (2015), [arXiv:1511.06773](https://arxiv.org/abs/1511.06773)）下给了著名 dichotomy：q-hierarchical 可以做到 $O(1)$ 更新 + $O(1)$ delay，非 q-hierarchical 则做不到这种常数级维护。把它翻成 exponent 语言，就是大家熟悉的“$\sqrt{|D|}$ barrier”：已有条件下界通常只到 $\Omega(|D|^{1/2-\varepsilon})$，而通用上界能做到 $O(|D|)$（例如 Kara et al. (2020) 与后续维护算法），中间隔着一个实打实的区间 $[0.5,1]$。

这篇工作的定位非常明确：它不再问“是不是 q-hierarchical”这种 0/1 分类，而是问“如果同属 free-connex，结构复杂度能不能连续刻画更新指数”。作者引入两个参数：height $k$（更像是 query 在关系超图上的“纵向扩展深度”）和 dimension $d$（更像“每层里可被独立激活的维度数”）。核心结论是参数化下界
$$
\Omega\!\left(|D|^{1-1/\max(k,d)-\varepsilon}\right),
$$
并在 star query 上给出匹配型上界。于是这是我看到的第一篇把 free-connex 动态维护下界稳定推到 $>\sqrt{|D|}$ 的工作。

为了让这个结果的价值更直观，先把对象说清楚。CQ 可以写成
$$
Q(\bar y) \leftarrow R_1(\bar x_1),\ldots,R_m(\bar x_m),
$$
本质上就是若干关系做 natural join，再投影到输出属性 $\bar y$。动态场景里数据库 $D$ 逐条更新，我们要维护 $Q(D)$。free-connex 是 acyclic CQ 里一个关键子类，直觉上它保证“输出变量也能被同一棵 join tree 很好地护住”，所以静态枚举复杂度漂亮；q-hierarchical 则更强，变量包含关系是层级嵌套式的，因此动态维护更容易。

:::aside[height 与 dimension 的形式化核心]
文中 height 通过 relational hypergraph 上的 chordless path / q-chordless path 长度定义：$Q$ 是 height $k$，当且仅当不存在更长于阈值的这两类路径，同时存在长度触发 $k$ 的见证路径。dimension 则对每个 connected subset $V_c$ 看其外部 distinct neighbor set $N$ 与 key set $V_k(N)$，定义局部维度
$$
\mathrm{dim}(V_c)=\max_N\begin{cases}
|V_k(N)|, & V_c\cap \bar y\neq\varnothing\ \text{或}\ V_k(N)\subseteq \bar y,\\
|V_k(N)\cup(\bar y\cap V^\*)|, & \text{otherwise}.
\end{cases}
$$
全局维度是 $\mathrm{dim}(Q)=\max_{V_c}\mathrm{dim}(V_c)$。
:::

有了这两个参数，主定理非常干脆。第一条（Theorem 1.1）是 height-based hardness：若 $Q$ 的 height $k\ge 2$，在 combinatorial $k$-clique 假设下，不存在 update time
$$
O\!\left(|D|^{(k-1)/k-\varepsilon}\right)
$$
且支持常数 delay 枚举的**组合型**动态算法。第二条（Theorem 1.2）是 dimension-based hardness：若 $\mathrm{dim}(Q)=d\ge 2$，在 OuMv$_k$ 假设下（Jin and Xu (2022), [arXiv:2202.11250](https://arxiv.org/abs/2202.11250)），不存在 update time
$$
O\!\left(|D|^{(d-1)/d-\varepsilon}\right)
$$
且支持常数 delay 的动态算法。并且作者给出
$$
Q\ \text{q-hierarchical}\ \Longleftrightarrow\ k=1\ \Longleftrightarrow\ d=1,
$$
把旧 dichotomy 嵌进了新参数层级。

我觉得这两条下界最有意思的地方，不是“都长成 $1-1/t$”，而是它们对应两种本质不同的困难来源。height 的证明是“沿结构链条变长就更难”，技术上把 query 里的 chordless path 抽出来，规约到 odd/even cycle detection 的条件下界；dimension 的证明则是“同样高度下，横向独立维度变多也更难”，通过 star query 作为中间桥梁规约到 OuMv$_k$。也就是说，论文不是把一个 lower bound 反复套皮，而是在“纵向展开”和“横向扩维”两个方向各给了一条可解释的硬度机制。

再说上界部分。作者没有尝试“一步到位”解决任意 free-connex，而是挑了工业里最常见的 star schema（fact table + dimension tables）做到 tight：定义 generalized H-index $h(D)$，其中每个中心属性 $x_i$ 的 $h_i(D)$ 是“至少有 $h_i$ 个值，其频率都至少 $h_i^{d-1}$”的最大整数，整体 $h(D)=\max_i h_i(D)$，并证明
$$
h(D)\le |D|^{1/d}.
$$
然后用 heavy/light 分解维护 $2^d$ 个逻辑分区和对应视图，得到 Theorem 1.4：线性空间 $O(|D|)$，更新
$$
O\!\left(h(D)^{d-1}\right),
$$
枚举 delay
$$
O\!\left(h(D)^{d-j}\right)
$$
（$j$ 是输出属性数）。最坏情形下这退化到 $|D|^{(d-1)/d}$，恰好贴住 dimension 下界；数据偏斜友好时会显著更快。

这一步我个人很买账，因为它没有停在“下界很强，算法以后再说”。它直接给了一个 instance-sensitive 的可执行模板：先承认最坏情况的硬度，再把可利用的偏斜（skew）显式参数化进复杂度。数据库系统里很多维护算法最终都吃这个红利，但这篇是把它和参数化下界严丝合缝地扣在一起了。

当然，论文也留下了两个很具体的边界。其一，height 下界依赖 combinatorial $k$-clique，只覆盖组合算法世界；如果未来有人用 fast matrix multiplication 风格技巧打穿某些实例，这个口子是开着的。其二，star query 之外（尤其 Loomis–Whitney-like 结构）还没有 matching upper bound，所以“free-connex 全类 tight”仍未完成。但和此前“0.5 到 1 的一整段迷雾”相比，这篇至少把地图画成了可导航的：你给我一个查询，我先看它在 $k,d$ 层级中的位置，再判断该期待什么指数级别的 update 代价，而不是只知道它“不属于 q-hierarchical”。

## 未入选但值得关注

### The Compilability Thresholds of 2-CNF to OBDD [arXiv:2603.15463](https://arxiv.org/abs/2603.15463)
论文研究 2-CNF 到 OBDD 的可编译性阈值，核心是把 satisfiability 结构参数与 OBDD 大小相变联系起来。结论很有理论美感，但今天优先入选的是直接推进在线/近似/动态维护主线的结果。

### Rooting Out Entropy: Optimal Tree Extraction for Ultra-Succinct Graphs [arXiv:2603.14649](https://arxiv.org/abs/2603.14649)
这篇提出 ultra-succinct graph 表示中的最优树提取问题，并给出加性不可近似与贪心界。技术扎实，不过影响面更集中在图压缩表示子方向。

### Approximation Algorithms for Action-Reward Query-Commit Matching [arXiv:2603.13487](https://arxiv.org/abs/2603.13487)
文章改进了 action-reward query-commit matching 的近似保证，延续了 stochastic matching 与 sequential pricing 的研究脉络。属于质量很高的常数改进，但相较入选论文的结构性推进，优先级略低。

### Jaguar: A Primal Algorithm for Conjunctive Query Evaluation in Submodular-Width Time [arXiv:2603.13624](https://arxiv.org/abs/2603.13624)
从题目看这是 conjunctive query 求值的重要进展，重点在 submodular-width 时间界下的 primal 算法。由于当前缓存缺少可用摘要且元数据拉取受限，今天无法做足够可靠的深读与取舍判断。
